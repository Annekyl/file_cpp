1. 分治思想概述

1.1 核心范式：Divide - Conquer - Combine

分治法 (Divide and Conquer) 是一种将原问题递归地分解为若干规模更小、相互独立的子问题，分别求解后再合并结果的算法设计范式。

                    ┌─────────────────┐
                    │  原问题 (规模 n)  │
                    └────────┬────────┘
                             │ Divide
                 ┌───────────┴───────────┐
                 ▼                       ▼
        ┌──────────────┐        ┌──────────────┐
        │ 子问题1(n/2)  │        │ 子问题2(n/2)  │
        └──────┬───────┘        └──────┬───────┘
               │ Conquer               │ Conquer
               ▼                       ▼
          解1 (子答案)            解2 (子答案)
               │                       │
               └───────────┬───────────┘
                           │ Combine
                           ▼
                    ┌─────────────┐
                    │ 原问题的解   │
                    └─────────────┘

1.2 分治三要素

分解 (Divide)：将原问题拆分为若干规模更小的子问题。子问题应与原问题结构相同，通常是将区间一分为二。
解决 (Conquer)：递归求解各子问题。当子问题规模足够小时（递归出口），直接求解。
合并 (Combine)：将子问题的解合并为原问题的解。合并的效率往往决定整个算法的复杂度。

1.3 适用条件

  • 原问题可以分解为若干规模更小的同类子问题
  • 子问题之间相互独立，不存在重叠子问题（否则用动态规划更优）
  • 存在递归终止条件（base case）
  • 子问题的合并操作具有高效性

1.4 复杂度分析方法：主定理 (Master Theorem)

对于递推关系 T(n) = a * T(n/b) + O(n^d)，其中 a 为子问题个数，b 为子问题规模缩减倍数，d 为合并的多项式阶数：

条件时间复杂度
d < log_b(a)O(n^(log_b(a)))
d = log_b(a)O(n^d * log n)
d > log_b(a)O(n^d)
提示
归并排序满足 T(n) = 2T(n/2) + O(n)，即 a=2, b=2, d=1，d = log_2(2) = 1，因此复杂度为 O(n log n)。
经典入门题：二分查找

给定有序数组和目标值，每次将搜索范围缩小一半。这是最简单的分治思想应用——每次只需求解一个子问题，另一个直接丢弃。

2. 归并排序 Merge Sort

2.1 算法思想

归并排序是分治思想的经典体现。核心思路：先递归排序左右两半，再将两个有序子数组合并为一个有序数组。

2.2 分治过程可视化

以数组 [38, 27, 43, 3, 9, 82, 10] 为例：

 分解阶段 (Divide):
 ────────────────────────────────────────────────
 [38, 27, 43, 3, 9, 82, 10]
        /                \
 [38, 27, 43, 3]     [9, 82, 10]
    /        \          /      \
 [38, 27] [43, 3]   [9, 82]  [10]   ← base case
  /   \    /   \     /   \
[38] [27] [43] [3] [9] [82]

 合并阶段 (Combine):
 ────────────────────────────────────────────────
 [38] [27] [43] [3] [9] [82]  [10]
  \   /     \   /    \   /      |
 [27, 38] [3, 43]  [9, 82]   [10]
    \       /          \       /
 [3, 27, 38, 43]    [9, 10, 82]
         \               /
    [3, 9, 10, 27, 38, 43, 82]

2.3 Merge 过程详解

合并是归并排序的关键步骤。使用双指针 i、j 分别指向两个有序子数组的起始位置，每次取较小值放入临时数组：

2.4 C++ 代码模板

C++#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;
int a[MAXN], tmp[MAXN];  // 原数组与临时数组

// 合并 [l, mid] 和 [mid+1, r] 两个有序区间
void merge(int l, int mid, int r) {
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j])
            tmp[k++] = a[i++];
        else
            tmp[k++] = a[j++];
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r)   tmp[k++] = a[j++];
    for (int p = l; p <= r; p++)
        a[p] = tmp[p];
}

// 归并排序主函数
void mergeSort(int l, int r) {
    if (l >= r) return;        // 递归出口：区间长度 <= 1
    int mid = l + (r - l) / 2;  // 防止溢出的中点计算
    mergeSort(l, mid);          // 递归排序左半
    mergeSort(mid + 1, r);      // 递归排序右半
    merge(l, mid, r);           // 合并两个有序区间
}

int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];
    mergeSort(0, n - 1);
    for (int i = 0; i < n; i++) cout << a[i] << " ";
    return 0;
}

2.5 复杂度分析

时间复杂度: O(n log n) (最好/最坏/平均)
空间复杂度: O(n) (需要临时数组)
性质说明
稳定性稳定排序 (合并时 a[i] <= a[j] 保证相等元素相对顺序不变)
时间一致性最好/最坏/平均均为 O(n log n)，不受输入数据分布影响
额外空间需要 O(n) 辅助空间，这是相对于快速排序的劣势
适用场景需要稳定排序、链表排序、外部排序、计算逆序对
经典例题：Luogu P1908 逆序对

给定长度为 n 的序列，求逆序对个数。归并排序的合并过程可以自然地统计逆序对数量 (见下一节)。

3. 逆序对计数 Counting Inversions

3.1 逆序对定义

对于序列 a，若存在下标 i < j 且 a[i] > a[j]，则 (i, j) 构成一个逆序对。逆序对的数量衡量了序列的"无序程度"。

3i=0
1i=1
4i=2
2i=3
5i=4

逆序对: (3,1), (4,2) 共 2 个

3.2 利用归并排序统计逆序对

关键观察：在 merge 过程中，当右侧元素 a[j] 小于左侧元素 a[i] 时，说明 a[j] 与左侧 [i, mid] 范围内的所有元素都构成逆序对。

 左半: [3, 27, 38, 43]    右半: [9, 10, 82]
        i=0                    j=0

 Step 1: a[i]=3 < a[j]=9  → 不产生新逆序对, 取 3
 Step 2: a[i]=27 > a[j]=9 → 右侧9比左侧[27,38,43]都小
         新增逆序对: (27,9), (38,9), (43,9) → 共 mid-i+1 = 3 个
         取 9
 Step 3: a[i]=27 > a[j]=10 → 新增逆序对: (27,10), (38,10), (43,10) → 3 个
         取 10
 Step 4: a[i]=27 < a[j]=82 → 不产生, 取 27
 Step 5: a[i]=38 < a[j]=82 → 不产生, 取 38
 Step 6: a[i]=43 < a[j]=82 → 不产生, 取 43
 Step 7: 剩余 82 直接追加

 本层逆序对总数 = 3 + 3 = 6

3.3 C++ 代码模板

C++#include <bits/stdc++.h>
using namespace std;

typedef long long ll;
const int MAXN = 500005;
int a[MAXN], tmp[MAXN];
ll invCnt = 0;  // 逆序对总数，注意用 long long

void merge(int l, int mid, int r) {
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) {
            tmp[k++] = a[i++];
        } else {
            // a[i] > a[j]: 左半 [i..mid] 的所有元素都比 a[j] 大
            invCnt += (mid - i + 1);  // 核心：统计跨左右的逆序对
            tmp[k++] = a[j++];
        }
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r)   tmp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = tmp[p];
}

void mergeSort(int l, int r) {
    if (l >= r) return;
    int mid = l + (r - l) / 2;
    mergeSort(l, mid);
    mergeSort(mid + 1, r);
    merge(l, mid, r);
}

int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];
    mergeSort(0, n - 1);
    cout << invCnt << endl;
    return 0;
}

3.4 复杂度分析

时间复杂度: O(n log n)
空间复杂度: O(n)
注意
逆序对数量可能达到 O(n^2) 级别（例如完全逆序的数组），因此计数变量必须使用 long long，否则会溢出。
另一种方法：树状数组

逆序对也可以用树状数组 (BIT) 在 O(n log n) 时间内统计。对于每个元素，查询有多少比它大的元素已经出现过。详见数据结构篇-树状数组。

经典例题：Luogu P1908 逆序对

4. 快速排序 Quick Sort

4.1 算法思想

快速排序同样是分治算法，但策略与归并排序相反：归并排序是"先分后合"，快速排序是"边分边排"。选取一个基准元素 (pivot)，将数组划分为"小于等于 pivot"和"大于 pivot"两部分，再递归排序两部分。

4.2 Partition 过程详解

以 [3, 6, 2, 7, 4, 1, 5] 为例，选择最后一个元素 5 作为 pivot：

 初始: [3, 6, 2, 7, 4, 1, 5]   pivot = 5
       ↑                    ↑
       i (小于区右界)        pivot

 维护不变量: [ <= pivot | > pivot | 未处理 | pivot ]

 Step 1: a[0]=3 <= 5  → swap(a[0], a[0]), i++
         [3 | 6, 2, 7, 4, 1, 5]
          ↑i

 Step 2: a[1]=6 > 5   → 跳过
         [3 | 6, 2, 7, 4, 1, 5]
          ↑i

 Step 3: a[2]=2 <= 5  → swap(a[2], a[1]), i++
         [3, 2 | 6, 7, 4, 1, 5]
             ↑i

 Step 4: a[3]=7 > 5   → 跳过
 Step 5: a[4]=4 <= 5  → swap(a[4], a[2]), i++
         [3, 2, 4 | 7, 6, 1, 5]
                ↑i

 Step 6: a[5]=1 <= 5  → swap(a[5], a[3]), i++
         [3, 2, 4, 1 | 6, 7, 5]
                   ↑i

 最终: swap(a[i], pivot位置) → [3, 2, 4, 1, 5, 7, 6]
                                    ↑ ≤5   ↑  >5

4.3 C++ 代码模板

C++#include <bits/stdc++.h>
using namespace std;

// 方式一: 双指针 partition (Lomuto 分区)
int partition(int a[], int l, int r) {
    int pivot = a[r];              // 选最后一个元素为基准
    int i = l;                      // i: <= pivot 区域的右边界
    for (int j = l; j < r; j++) {
        if (a[j] <= pivot) {
            swap(a[i], a[j]);
            i++;
        }
    }
    swap(a[i], a[r]);               // 将 pivot 放到正确位置
    return i;                       // 返回 pivot 的最终位置
}

void quickSort(int a[], int l, int r) {
    if (l >= r) return;
    int p = partition(a, l, r);
    quickSort(a, l, p - 1);        // 递归排序左半
    quickSort(a, p + 1, r);        // 递归排序右半
}

// 方式二: 双指针 Hoare 分区 (更高效，交换次数更少)
void quickSortHoare(int a[], int l, int r) {
    if (l >= r) return;
    int pivot = a[l + (r - l) / 2]; // 取中间元素为基准
    int i = l, j = r;
    while (i <= j) {
        while (a[i] < pivot) i++;
        while (a[j] > pivot) j--;
        if (i <= j) {
            swap(a[i], a[j]);
            i++;
            j--;
        }
    }
    if (l < j)  quickSortHoare(a, l, j);
    if (i < r)  quickSortHoare(a, i, r);
}

4.4 随机化快速排序

最坏情况下（数组已有序），朴素快速排序退化为 O(n^2)。通过随机选取 pivot 可以在期望 O(n log n) 时间内完成排序。

C++// 随机化 partition: 随机选择 pivot，避免最坏情况
int randPartition(int a[], int l, int r) {
    // 随机选一个位置与末尾交换
    int randIdx = l + rand() % (r - l + 1);
    swap(a[randIdx], a[r]);
    return partition(a, l, r);  // 复用普通 partition
}

4.5 复杂度分析

情况时间复杂度触发条件
最好O(n log n)每次 pivot 恰好将数组平分
平均O(n log n)随机化后以极高概率达到
最坏O(n^2)每次 pivot 都是极值（如已排序数组选首/尾元素）
平均空间复杂度: O(log n) (递归栈深度)
最坏空间复杂度: O(n)
快速排序 vs 归并排序
  • 快排是原地排序 (O(log n) 栈空间)，归并需要 O(n) 额外空间
  • 快排不稳定，归并稳定
  • 快排平均常数因子更小，实际运行通常更快
  • C++ STL 的 std::sort 使用 Introsort (快排+堆排+插入排序混合)
经典例题：Luogu P1177 【模板】快速排序

使用快速排序对 n 个整数排序。建议使用随机化版本以避免退化。

5. 第 k 大/小元素 QuickSelect

5.1 算法思想

QuickSelect 是快速排序的变体。在 partition 之后，pivot 的位置已经确定。如果 pivot 的位置恰好等于目标位置，直接返回；否则只需递归搜索其中一侧。

 找第 k=3 小的元素 (0-indexed: 第 k-1=2 位)
 数组: [3, 6, 2, 7, 4, 1, 5]

 第一次 partition: pivot=5, p=4 (0-indexed)
   [3, 2, 4, 1] [5] [7, 6]
    ≤ pivot      ^    > pivot
                 p=4

 k=2 < p=4 → 只需递归左侧 [3, 2, 4, 1]

 第二次 partition: pivot=1, p=0
   [1] [3, 2, 4]
    ^   ≤ pivot
    p=0

 k=2 > p=0 → 只需递归右侧 [3, 2, 4]

 第三次 partition: pivot=4, p=3
   [3, 2] [4]
           ^
           p=3

 k=2 < p=3 → 递归左侧 [3, 2]

 第四次 partition: pivot=2, p=1
   [2] [3]
    ^
    p=1

 k=2 > p=1 → 递归右侧 [3], p=2 == k → 答案为 3

5.2 C++ 代码模板

C++#include <bits/stdc++.h>
using namespace std;

// 返回第 k 小的元素 (0-indexed)
// 平均 O(n)，最坏 O(n^2)，随机化后期望 O(n)
int quickSelect(int a[], int l, int r, int k) {
    if (l == r) return a[l];

    // 随机化选 pivot
    int randIdx = l + rand() % (r - l + 1);
    swap(a[randIdx], a[r]);
    int pivot = a[r];
    int i = l;
    for (int j = l; j < r; j++) {
        if (a[j] <= pivot) {
            swap(a[i], a[j]);
            i++;
        }
    }
    swap(a[i], a[r]);

    if (i == k)       return a[i];         // 恰好是第 k 位
    else if (i > k)   return quickSelect(a, l, i - 1, k);  // 在左侧
    else              return quickSelect(a, i + 1, r, k);  // 在右侧
}

// 使用 STL: nth_element (底层为 IntroSelect，保证最坏 O(n))
int kthElementSTL(vector<int>& a, int k) {
    nth_element(a.begin(), a.begin() + k, a.end());
    return a[k];
}

5.3 复杂度分析

平均时间: O(n) -- 每次只需处理一半
最坏时间: O(n^2) -- 随机化后几乎不会出现
空间复杂度: O(1) 原地 (尾递归优化)
关于 STL nth_element

C++ 的 std::nth_element 使用 IntroSelect 算法，结合快速选择和中位数的中位数 (Median of Medians) 策略，保证最坏情况下 O(n) 时间复杂度。比赛中直接使用即可。

经典例题：Luogu P1471 方差 (部分分可用)

给定 n 个数，求第 k 小的数。使用 nth_element 或手写 QuickSelect。

经典例题：LeetCode 215 - Kth Largest Element

求数组中第 k 大的元素。用 QuickSelect 搜索第 (n-k) 小的元素即可。

6. 分治经典应用

6.1 最近点对 (Closest Pair of Points)

问题描述

给定平面上 n 个点，求距离最近的一对点之间的距离。暴力枚举 O(n^2)，分治可优化至 O(n log n)。

分治策略

按 x 坐标排序，取中位线 x = mid 将点集分为左右两半。
递归求解：分别求左半和右半的最近点对距离 d_left 和 d_right，取 d = min(d_left, d_right)。
合并：考虑跨越中线的点对。只需检查与中线 x 距离小于 d 的点（位于带状区域宽度 2d 内），且在 y 方向上距离小于 d 的点。
关键优化：将带状区域内的点按 y 排序后，每个点只需检查其后最多 7 个点（由几何性质保证）。因此该步骤为 O(n)。
         d_left    |    d_right
                   |
    ·    ·        |        ·
       ·          |    ·
    ·       ·     |       ·
                   |
         d         d

 跨越中线的点只需检查带状区域 [mid-d, mid+d] 内的点对:
                   |
         ·         |
    ·    · ·       |  宽度 2d
         · ·       |
                   |

 按 y 排序后，每个点只需与下方距离 < d 的点比较 (最多 7 个)

C++ 代码模板

C++#include <bits/stdc++.h>
using namespace std;

struct Point {
    double x, y;
};

int n;
Point p[100005], tmp[100005];

bool cmpX(const Point& a, const Point& b) {
    return a.x < b.x;
}
bool cmpY(const Point& a, const Point& b) {
    return a.y < b.y;
}

double dist(const Point& a, const Point& b) {
    return hypot(a.x - b.x, a.y - b.y);
}

// 求 p[l..r] 的最近点对距离，要求 p 已按 x 排序
double closest(int l, int r) {
    if (l == r) return 1e18;
    if (l + 1 == r) return dist(p[l], p[r]);

    int mid = l + (r - l) / 2;
    double midX = p[mid].x;
    double d = min(closest(l, mid), closest(mid + 1, r));

    // 收集带状区域内的点，按 y 排序
    int cnt = 0;
    for (int i = l; i <= r; i++) {
        if (fabs(p[i].x - midX) < d)
            tmp[cnt++] = p[i];
    }
    sort(tmp, tmp + cnt, cmpY);

    // 每个点只检查后面最多 7 个点
    for (int i = 0; i < cnt; i++) {
        for (int j = i + 1; j < cnt && tmp[j].y - tmp[i].y < d; j++) {
            d = min(d, dist(tmp[i], tmp[j]));
        }
    }
    return d;
}

int main() {
    cin >> n;
    for (int i = 0; i < n; i++)
        cin >> p[i].x >> p[i].y;
    sort(p, p + n, cmpX);
    printf("%.4f\n", closest(0, n - 1));
    return 0;
}
时间复杂度: O(n log n)
空间复杂度: O(n)
经典例题：Luogu P1429 平面最近点对

6.2 最大子数组和 (Maximum Subarray)

问题描述

给定一个整数数组，找到一个具有最大和的连续子数组，返回其最大和。

方法一：Kadane 算法 (O(n))

虽然这个问题可以用分治解决，但更高效的解法是 Kadane 算法——它本质上是动态规划的思想：

C++// Kadane 算法: O(n) 时间, O(1) 空间
int maxSubarrayKadane(int a[], int n) {
    int maxSum = a[0];
    int curSum = a[0];
    for (int i = 1; i < n; i++) {
        curSum = max(a[i], curSum + a[i]);  // 继续扩展 vs 重新开始
        maxSum = max(maxSum, curSum);
    }
    return maxSum;
}

方法二：分治解法 (O(n log n))

分治解法虽然不如 Kadane 高效，但展示了分治思想的典型应用：

分：将数组一分为二，中点为 mid。
治：递归求解左半最大子数组和 leftMax 与右半最大子数组和 rightMax。
合：最大子数组可能完全在左半、完全在右半，或跨越中线。跨越中线的最大和 = 左侧最大后缀和 + 右侧最大前缀和。
 数组: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
                                mid

 Case 1: 最大子数组完全在左半
         [-2, 1, -3, 4, -1] | [2, 1, -5, 4]

 Case 2: 最大子数组完全在右半
         [-2, 1, -3, 4, -1] | [2, 1, -5, 4]

 Case 3: 最大子数组跨越中线
         [-2, 1, -3, 4, -1] | [2, 1, -5, 4]
                       ──────┤├──────
                        -1+2+1 = 2

 需要计算:
 - 左侧最大后缀和: max(从 mid 向左扩展的最大和)
 - 右侧最大前缀和: max(从 mid+1 向右扩展的最大和)
 - 跨中线最大和 = 左最大后缀 + 右最大前缀

 答案 = max(左递归, 右递归, 跨中线) = max(4, 4, 6) = 6
 子数组 [4, -1, 2, 1]

C++#include <bits/stdc++.h>
using namespace std;

// 分治求最大子数组和
int maxSubarrayDC(int a[], int l, int r) {
    if (l == r) return a[l];

    int mid = l + (r - l) / 2;

    // Case 1 & 2: 递归求左右两半
    int leftMax = maxSubarrayDC(a, l, mid);
    int rightMax = maxSubarrayDC(a, mid + 1, r);

    // Case 3: 跨越中线的最大子数组
    // 左侧最大后缀和
    int leftSuffix = a[mid];
    int sum = a[mid];
    for (int i = mid - 1; i >= l; i--) {
        sum += a[i];
        leftSuffix = max(leftSuffix, sum);
    }
    // 右侧最大前缀和
    int rightPrefix = a[mid + 1];
    sum = a[mid + 1];
    for (int i = mid + 2; i <= r; i++) {
        sum += a[i];
        rightPrefix = max(rightPrefix, sum);
    }
    int crossMax = leftSuffix + rightPrefix;

    return max({leftMax, rightMax, crossMax});
}

6.3 分治方法总结与扩展

分治思维导图
  • 排序类：归并排序、快速排序
  • 选择类：QuickSelect (第 k 大/小)
  • 统计类：逆序对计数、满足条件的数对
  • 几何类：最近点对、凸包 (分治版)
  • 数组类：最大子数组和、Karatsuba 大数乘法
  • 线段树：本质也是分治——递归处理左右子区间
分治 vs 动态规划

分治与动态规划都涉及"将问题分解为子问题"，但有本质区别：

  • 分治：子问题相互独立，无重叠。如归并排序的左右两半互不影响。
  • DP：子问题存在重叠，同一子问题会被多次计算。如斐波那契数列的 f(n-1) 和 f(n-2) 都需要 f(n-3)。
  • 若子问题有重叠却用分治，会导致指数级复杂度；反之用 DP 则过于复杂。
