概述与核心思想

二分查找（Binary Search）是一种在有序或满足单调性的搜索空间中，每次将搜索范围缩小一半，从而以对数时间复杂度找到目标值或满足条件的边界值的算法。

核心思想

每次取搜索区间的中点，根据中点处的值与目标的关系，决定保留左半部分还是右半部分，从而将搜索空间减半。

适用条件

  • 搜索空间具有单调性（如有序数组），或答案空间可以被划分为「不满足条件」和「满足条件」两段
  • 可以通过某个条件 check(mid) 判断当前中点是否满足要求
  • 搜索空间是离散整数或连续实数

基本流程图

  ┌──────────────────────────────────────────────┐
  │            初始化搜索区间 [l, r]              │
  └──────────────────┬───────────────────────────┘
                     ▼
            ┌────────────────┐
            │   l <= r ?     │──── 否 ──→ 结束，返回结果
            └───────┬────────┘
                    │ 是
                    ▼
            ┌────────────────┐
            │ mid = (l+r)/2  │
            └───────┬────────┘
                    ▼
            ┌────────────────┐
            │  check(mid) ?  │
            └───┬────────┬───┘
            是  │        │ 否
                ▼        ▼
        ┌──────────┐ ┌──────────┐
        │ r = mid  │ │ l = mid  │
        │  - 1     │ │  + 1     │
        └────┬─────┘ └────┬─────┘
             └──────┬──────┘
                    ▼
              返回循环顶部
注意

二分查找的前提是搜索空间具有单调性或可以被二分划分。盲目对无序数据使用二分查找是错误的。

整数二分

整数二分是最基础的二分形式，搜索空间为离散的整数区间 [l, r]。根据查找目标的不同，分为两种经典模式：lower_bound（找第一个满足条件的位置）和 upper_bound（找最后一个满足条件的位置）。

模式一：lower_bound — 找第一个 ≥ target 的位置

在有序数组中，找到第一个大于等于 target 的元素的位置。如果不存在，返回数组长度。

图解过程

假设数组为 [1, 3, 3, 3, 5, 7, 9]，查找第一个 ≥ 3 的位置：

10

33
54
75
96

结果为下标 1，即第一个 3 出现的位置。

逐步演示

初始化 l = 0, r = 6，搜索区间为整个数组
mid = 3，a[3] = 3 ≥ 3，满足条件，r = mid = 3（继续往左找更早的）
mid = 1，a[1] = 3 ≥ 3，满足条件，r = mid = 1
mid = 0，a[0] = 1 < 3，不满足条件，l = mid + 1 = 1
l = 1, r = 1，循环结束，返回 l = 1

代码模板

// lower_bound: 找第一个 >= target 的位置
// 返回值范围: [0, n]，n 表示未找到
int lower_bound(int a[], int n, int target) {
    int l = 0, r = n; // 注意: r = n，左闭右开 [0, n)
    while (l < r) {
        int mid = l + (r - l) / 2; // 防溢出写法
        if (a[mid] >= target)
            r = mid;     // mid 可能是答案，保留
        else
            l = mid + 1; // mid 不满足，排除
    }
    return l; // l == r，即为答案
}
// STL 等价函数: std::lower_bound(a, a+n, target) - aC++

模式二：upper_bound — 找第一个 > target 的位置

在有序数组中，找到第一个严格大于 target 的元素的位置。等价于找最后一个 ≤ target 的位置的下一个。

图解过程

同样数组 [1, 3, 3, 3, 5, 7, 9]，查找第一个 > 3 的位置：

10
31
32
33
54
75
96

结果为下标 4，即值为 5 的位置。

代码模板

// upper_bound: 找第一个 > target 的位置
int upper_bound(int a[], int n, int target) {
    int l = 0, r = n;
    while (l < r) {
        int mid = l + (r - l) / 2;
        if (a[mid] > target)
            r = mid;     // mid 可能是答案
        else
            l = mid + 1; // a[mid] <= target，排除
    }
    return l;
}
// STL 等价函数: std::upper_bound(a, a+n, target) - aC++
记忆技巧

lower_bound 用 >= 判断，找到的是「第一个满足 ≥ target」的位置；upper_bound 用 > 判断，找到的是「第一个满足 > target」的位置。两者唯一的区别就在于 check 函数中是否包含等号。

另一种写法：左闭右闭区间

// 左闭右闭 [l, r] 写法的 lower_bound
int lower_bound_closed(int a[], int n, int target) {
    int l = 0, r = n - 1, ans = n; // ans 记录答案，默认 n 表示未找到
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (a[mid] >= target) {
            ans = mid;
            r = mid - 1;  // 继续往左找
        } else {
            l = mid + 1;
        }
    }
    return ans;
}C++
两种写法对比

左闭右开 [l, r)：循环条件 l < r，更新为 r = mid 或 l = mid + 1，返回 l。推荐使用此写法，与 STL 一致。
左闭右闭 [l, r]：循环条件 l <= r，更新为 r = mid - 1 或 l = mid + 1，需要额外变量记录答案。
两种写法等价，选一种熟练掌握即可。比赛中推荐左闭右开写法。

经典例题：查找元素位置

给定有序数组和目标值，求目标值的起始位置和结束位置

等价于求 lower_bound 和 upper_bound - 1。

#include <bits/stdc++.h>
using namespace std;

int main() {
    int a[] = {1, 3, 3, 3, 5, 7, 9};
    int n = 7, target = 3;

    // 方法一: 手写二分
    int first = lower_bound(a, n, target);
    int last  = upper_bound(a, n, target) - 1;

    if (first < n && a[first] == target) {
        cout << "起始位置: " << first << endl;
        cout << "结束位置: " << last << endl;
        cout << "出现次数: " << last - first + 1 << endl;
    } else {
        cout << "未找到" << endl;
    }

    // 方法二: 使用 STL
    auto lo = lower_bound(a, a + n, target);
    auto hi = upper_bound(a, a + n, target);
    cout << "STL 出现次数: " << hi - lo << endl;

    return 0;
}C++

浮点二分

当搜索空间是连续实数时（例如求平方根、求方程的解），使用浮点二分。与整数二分不同，浮点二分的循环终止条件是区间长度足够小（小于精度要求），而非 l == r。

核心区别

对比项 整数二分 浮点二分
搜索空间 离散整数 [l, r] 连续实数 [l, r]
终止条件 l < r（左闭右开） r - l > eps
中点计算 mid = l + (r-l)/2 mid = (l+r) / 2.0
精度控制 无需 需要设置 eps（通常 1e-7 ~ 1e-9）
循环次数 O(log n) 通常固定 100 次迭代更安全

经典问题：求平方根

给定一个非负实数 x，求 sqrt(x) 的值，精确到小数点后 6 位。

  求 sqrt(10) 的过程:

  搜索区间 [0, 10]
  ┌────────────────────────────────────────────────────┐
  │  Step 1: mid = 5.0,   5.0^2 = 25.0  > 10  → r=5.0│
  │  Step 2: mid = 2.5,   2.5^2 = 6.25  < 10  → l=2.5│
  │  Step 3: mid = 3.75,  3.75^2=14.06  > 10  → r=3.75│
  │  Step 4: mid = 3.125, 3.125^2=9.77  < 10  → l=3.125│
  │  ...                                                │
  │  最终收敛到: 3.162277...                             │
  └────────────────────────────────────────────────────┘

代码模板

// 浮点二分求平方根
double my_sqrt(double x) {
    if (x < 1.0) {
        // x < 1 时 sqrt(x) > x，搜索区间为 [x, 1]
        double l = x, r = 1.0;
        for (int i = 0; i < 100; i++) { // 固定迭代次数更安全
            double mid = (l + r) / 2.0;
            if (mid * mid > x) r = mid;
            else l = mid;
        }
        return l;
    }
    double l = 0.0, r = x;
    for (int i = 0; i < 100; i++) {
        double mid = (l + r) / 2.0;
        if (mid * mid > x) r = mid;
        else l = mid;
    }
    return l;
}

// 更通用的浮点二分模板
double binary_search_real(double lo, double hi) {
    for (int i = 0; i < 100; i++) { // 每次精度翻倍，100次足够
        double mid = (lo + hi) / 2.0;
        if (check(mid))
            hi = mid;
        else
            lo = mid;
    }
    return lo;
}C++
推荐做法

浮点二分推荐使用固定迭代次数（如 100 次）代替 r - l > eps 的写法。固定迭代 100 次后精度可达 2^{-100} ≈ 10^{-30}，远超题目要求，且避免了 eps 选择不当导致的死循环风险。

经典例题：求方程的根

求方程 f(x) = x^3 - 5x^2 + 10x - 80 = 0 在 [0, 100] 上的实根

f(x) 是单调递增函数（在该区间上），f(0) < 0, f(100) > 0，由零点定理知必有解。

#include <bits/stdc++.h>
using namespace std;

double f(double x) {
    return x * x * x - 5.0 * x * x + 10.0 * x - 80.0;
}

int main() {
    double l = 0.0, r = 100.0;
    for (int i = 0; i < 100; i++) {
        double mid = (l + r) / 2.0;
        if (f(mid) > 0)
            r = mid;
        else
            l = mid;
    }
    printf("%.6f\n", l);
    return 0;
}
// 输出: 6.103297C++

二分答案

二分答案是二分查找最强大的应用之一。核心思想是：不直接在数组上二分，而是在「答案的取值范围」上二分，每次猜测一个答案 mid，然后验证这个答案是否可行。

适用场景

当问题满足以下条件时可以考虑二分答案：

  • 答案具有单调性：如果 x 可行，则所有比 x 更优的答案也可行（或反之）
  • 答案的取值范围可以确定（有上下界）
  • 对于给定的一个候选答案，可以在合理时间内验证其可行性

模式一：求最小的满足条件的值

当 check(mid) 为 true 时，说明 mid 可行，但可能存在更小的可行解，因此 r = mid；当 check(mid) 为 false 时，mid 不可行，需要更大的值，l = mid + 1。

  答案空间: [L, R]
  check(x) 表示 "x 是否可行"

      false   false   false   true    true    true
  ────┼───────┼───────┼───────┼───────┼───────┼────
      L                                   ↑     R
                                    第一个 true
                                    即为最小可行解

  二分过程:
  mid 可行? → r = mid  (往左找更小的可行解)
  mid 不可行? → l = mid + 1  (往右找)

代码模板

// 二分答案: 求最小的满足条件的值
int bs_min_answer(int L, int R) {
    int l = L, r = R + 1; // 注意 r 的取值
    while (l < r) {
        int mid = l + (r - l) / 2;
        if (check(mid))  // mid 可行
            r = mid;     // 保留 mid，继续往左
        else
            l = mid + 1; // mid 不可行，往右
    }
    return l; // 最小的可行解
}C++

模式二：求最大的满足条件的值

当 check(mid) 为 true 时，mid 可行，可能存在更大的可行解，l = mid（注意: 此时中点计算需用 l + (r - l + 1) / 2 防死循环）；当 false 时，r = mid - 1。

代码模板

// 二分答案: 求最大的满足条件的值
int bs_max_answer(int L, int R) {
    int l = L - 1, r = R; // 注意 l 的取值
    while (l < r) {
        int mid = l + (r - l + 1) / 2; // 上取整，防死循环!
        if (check(mid))  // mid 可行
            l = mid;     // 保留 mid，继续往右
        else
            r = mid - 1; // mid 不可行，往左
    }
    return l; // 最大的可行解
}C++
重要: 防止死循环

在「求最大可行解」的模板中，当 l = mid 时，必须使用 mid = l + (r - l + 1) / 2（上取整）。否则当 r = l + 1 时，mid = l，执行 l = mid 后 l 不变，陷入死循环。

经典例题：分巧克力（NOIP 2017 提高组）

分巧克力

有 N 块巧克力，第 i 块是 H_i x W_i 的矩形。要分给 K 个小朋友，每人分到的必须是同样大小的正方形（边长为整数）。求正方形的最大边长。

分析

  • 答案的单调性：如果边长 x 可以分给 K 个人，那么边长 x-1 也可以（每块巧克力能切出更多小块）
  • check(mid)：对于边长 mid，计算每块巧克力能切出 (H_i/mid) * (W_i/mid) 块，求和后判断是否 ≥ K
  • 二分范围：[1, max(H_i, W_i)]

完整代码

#include <bits/stdc++.h>
using namespace std;

int n, k;
int h[100005], w[100005];

// check: 边长为 x 时，能否切出 >= k 块
bool check(int x) {
    long long cnt = 0;
    for (int i = 0; i < n; i++) {
        cnt += (long long)(h[i] / x) * (w[i] / x);
        if (cnt >= k) return true; // 提前返回，防溢出
    }
    return cnt >= k;
}

int main() {
    cin >> n >> k;
    int maxSide = 0;
    for (int i = 0; i < n; i++) {
        cin >> h[i] >> w[i];
        maxSide = max(maxSide, max(h[i], w[i]));
    }

    // 二分答案: 求最大可行边长
    int l = 1, r = maxSide, ans = 1;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (check(mid)) {
            ans = mid;
            l = mid + 1;   // 可行，尝试更大的
        } else {
            r = mid - 1;  // 不可行，缩小
        }
    }
    cout << ans << endl;
    return 0;
}C++

经典例题：Aggressive Cows（POJ 2456）

Aggressive Cows — 最大化最小距离

有 N 个牛栏排成一排，位置为 x[i]。要把 C 头牛放进去，使得任意两头牛之间的最小距离尽可能大。求这个最大的最小距离。

分析

  • 单调性：如果最小距离 d 可行（能放下 C 头牛），那么 d-1 也可行
  • check(d)：贪心放置，第一头牛放在第一个位置，之后每头牛放在距离前一头至少 d 的最近牛栏
  • 二分范围：[1, x[N-1] - x[0]]
#include <bits/stdc++.h>
using namespace std;

int n, c;
int x[100005];

// check: 最小距离为 d 时，能否放下 c 头牛
bool check(int d) {
    int cnt = 1;          // 第一头牛放在 x[0]
    int last = x[0];
    for (int i = 1; i < n; i++) {
        if (x[i] - last >= d) {
            cnt++;
            last = x[i];
            if (cnt >= c) return true;
        }
    }
    return false;
}

int main() {
    cin >> n >> c;
    for (int i = 0; i < n; i++) cin >> x[i];
    sort(x, x + n);

    int l = 1, r = x[n-1] - x[0], ans = 0;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (check(mid)) {
            ans = mid;
            l = mid + 1;
        } else {
            r = mid - 1;
        }
    }
    cout << ans << endl;
    return 0;
}C++

常见变体

变体一：查找第一个/最后一个等于 target 的位置

这本质上就是 lower_bound 和 upper_bound - 1 的组合，前面整数二分部分已经详细讲解。这里再给出一个简洁的对比写法：

// 查找 target 在有序数组中第一次出现的位置
int find_first(int a[], int n, int target) {
    int pos = lower_bound(a, a + n, target) - a;
    return (pos < n && a[pos] == target) ? pos : -1;
}

// 查找 target 在有序数组中最后一次出现的位置
int find_last(int a[], int n, int target) {
    int pos = upper_bound(a, a + n, target) - a - 1;
    return (pos >= 0 && a[pos] == target) ? pos : -1;
}C++

变体二：旋转数组查找

将一个有序数组在某个位置旋转，例如 [4, 5, 6, 7, 0, 1, 2] 是 [0, 1, 2, 4, 5, 6, 7] 在位置 3 处旋转得到的。在旋转数组中查找目标值。

图解

  原数组:  [0, 1, 2, 4, 5, 6, 7]
  旋转后:  [4, 5, 6, 7, 0, 1, 2]
                   ↑
               旋转点(最小值)

  特点: 数组被分为两段有序子数组
  左半段: [4, 5, 6, 7]  所有值 >= nums[0]
  右半段: [0, 1, 2]     所有值 < nums[0]

思路

取中点 mid 后，[l, mid] 和 [mid, r] 中至少有一段是有序的。判断哪段有序，然后检查 target 是否在有序段内，决定搜索方向。

代码模板

// 在旋转排序数组中查找 target (LeetCode 33)
int search_rotated(int nums[], int n, int target) {
    int l = 0, r = n - 1;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (nums[mid] == target) return mid;

        if (nums[l] <= nums[mid]) {
            // 左半段 [l, mid] 有序
            if (nums[l] <= target && target < nums[mid])
                r = mid - 1;  // target 在左半段
            else
                l = mid + 1;  // target 在右半段
        } else {
            // 右半段 [mid, r] 有序
            if (nums[mid] < target && target <= nums[r])
                l = mid + 1;  // target 在右半段
            else
                r = mid - 1;  // target 在左半段
        }
    }
    return -1;
}C++

变体四：查找峰值

Find Peak Element (LeetCode 162)

找到数组中的一个峰值元素（大于相邻元素），返回其下标。时间复杂度要求 O(log n)。

// 查找峰值: 利用二分，往递增方向走必能找到峰值
int find_peak(int nums[], int n) {
    int l = 0, r = n - 1;
    while (l < r) {
        int mid = l + (r - l) / 2;
        if (nums[mid] > nums[mid + 1])
            r = mid;      // 峰值在 [l, mid]
        else
            l = mid + 1;  // 峰值在 [mid+1, r]
    }
    return l;
}C++

变体五：二分 + 第 K 小/大

在某些问题中，答案是第 K 小（或第 K 大）的值，可以在值域上二分，配合计数函数验证。

K-th Smallest Number in Multiplication Table (LeetCode 668)

给定 m x n 的乘法表，求其中第 k 小的数。

// 乘法表中第 k 小的数
int m, n, k;
// 计算乘法表中有多少个数 <= v
int count_le(int v) {
    int cnt = 0;
    for (int i = 1; i <= m; i++)
        cnt += min(v / i, n);
    return cnt;
}

int find_kth() {
    int l = 1, r = m * n;
    while (l < r) {
        int mid = l + (r - l) / 2;
        if (count_le(mid) >= k)
            r = mid;
        else
            l = mid + 1;
    }
    return l;
}C++

整数二分 O(log n) O(1) n 为搜索区间长度

旋转数组查找 O(log n) O(1) 标准二分的变体
核心复杂度: O(log n)

易错点与技巧

常见错误

错误 1: 整数溢出

mid = (l + r) / 2 当 l 和 r 都很大时会溢出。应使用 mid = l + (r - l) / 2。

错误 2: 死循环

左闭右闭写法中，当 l = mid 且 mid = (l+r)/2 下取整时，若 r = l + 1，则 mid = l，l = mid 不变，死循环。解法：改用上取整 mid = l + (r-l+1)/2。

错误 3: 浮点精度问题

浮点二分不要用 l == r 判断终止，浮点数比较有精度误差。推荐固定迭代次数。

错误 4: check 函数写反

二分答案时，check(mid) 的返回值与更新方向搞混。记住：求最小用 lower_bound 模式，求最大用 upper_bound 模式。先确定好 check 的含义，再写更新逻辑。

实用技巧

技巧 1: STL 二分函数

C++ STL 提供了 lower_bound、upper_bound、binary_search 函数，适用于有序容器（如 vector、set）。在 set 中使用成员函数版本 s.lower_bound(x) 比全局版本更快（O(log n) vs O(n)）。

技巧 2: 二分答案的 check 函数设计

二分答案的关键在于设计 check 函数。常见思路：

  • 贪心验证：如「Aggressive Cows」中贪心放置
  • 计数：统计满足条件的元素个数
  • 模拟：模拟整个过程判断是否可行
技巧 3: 实数二分的通用模板

对于实数二分，推荐以下写法避免精度问题：

// 实数二分通用模板 (固定迭代次数)
double solve() {
    double l = 0.0, r = 1e18;
    for (int i = 0; i < 200; i++) { // 200 次迭代，精度达 2^{-200}
        double mid = (l + r) / 2.0;
        if (check(mid))
            r = mid;
        else
            l = mid;
    }
    return l;
}C++
总结: 二分查找解题框架
  1. 确定搜索空间：答案的取值范围是什么？
  2. 确定单调性：为什么可以用二分？单调性如何？
  3. 设计 check 函数：给定一个候选答案 mid，如何判断它是否可行？
  4. 确定二分模式：求最大还是求最小？用哪个模板？
  5. 注意边界：初始范围是否正确？是否需要防溢出？是否需要防死循环？
