一、容器 (Containers)

STL 容器是竞赛编程中最常用的数据结构。下面按照类别逐一介绍各容器的核心操作、时间复杂度和使用技巧。

1.1 序列容器

vector -- 动态数组

vector 是最常用的容器，底层为连续内存的动态数组。支持 O(1) 随机访问，尾部插入摊还 O(1)。

操作时间复杂度说明
push_back(x)摊还 O(1)尾部添加元素（拷贝/移动构造）
emplace_back(args...)摊还 O(1)尾部原地构造，避免临时对象
pop_back()O(1)删除尾部元素
size() / empty()O(1)元素个数 / 是否为空
resize(n)O(n)调整大小，多删少补（默认值初始化）
reserve(n)O(n)预分配容量，不改变 size
operator[] / at(i)O(1)随机访问，at 有越界检查
insert(it, x)O(n)在迭代器位置前插入
erase(it)O(n)删除迭代器指向的元素
clear()O(n)清空所有元素（不释放内存）
 
#include <vector>
using namespace std;

// 基本用法
vector<int> v;              // 空 vector
vector<int> v2(10);          // 10 个 0
vector<int> v3(10, 5);      // 10 个 5
vector<int> v4 = {1,2,3,4}; // 初始化列表
vector<int> v5(v4.begin()+1, v4.end()); // 拷贝 [1,end)

// 二维 vector（邻接表常用）
vector<vector<int>> adj(n);  // n 行，每行空
vector<vector<int>> mat(n, vector<int>(m, 0)); // n x m 矩阵

// emplace_back vs push_back
vector<pair<int,int>> vp;
vp.push_back({1,2});          // 先构造临时 pair，再移动
vp.emplace_back(1, 2);       // 原地构造，更高效

// reserve 避免多次扩容
vector<int> v;
v.reserve(100000);           // 预分配，减少 realloc
for (int i = 0; i < 100000; i++)
    v.push_back(i);           // 不会触发扩容
迭代器失效规则 (vector)
  • push_back: 如果触发扩容，所有迭代器失效；否则不影响
  • insert: 插入点及之后的所有迭代器失效
  • erase: 被删点及之后的所有迭代器失效
  • clear: 所有迭代器失效
竞赛技巧
  • 已知元素数量时，先 reserve 再 push_back，避免多次扩容
  • v.resize(n) 后可以直接 v[i] 访问，比循环 push_back 更简洁
  • 用 v.data() 获取底层数组指针，传给 C 接口
  • 清空并释放内存: vector<int>().swap(v); 或 v.shrink_to_fit();

deque -- 双端队列

deque 底层为分段连续内存（中控器 + 缓冲区），支持两端 O(1) 插入删除和 O(1) 随机访问。

操作时间复杂度说明
push_back / push_frontO(1)头/尾插入
pop_back / pop_frontO(1)头/尾删除
operator[] / at(i)O(1)随机访问
insert / eraseO(n)中间插入/删除
 
#include <deque>
deque<int> dq;
dq.push_back(1);    // [1]
dq.push_front(2);   // [2, 1]
dq.push_back(3);    // [2, 1, 3]
int a = dq[0];      // 2 (O(1) 随机访问)
dq.pop_front();       // [1, 3]
dq.pop_back();        // [1]

// deque 作为单调队列的底层容器
deque<int> mono;    // 单调队列用 deque 实现滑动窗口
deque vs vector

deque 头部插入 O(1)，但随机访问比 vector 稍慢（非完全连续内存，需二次寻址）。竞赛中 deque 主要用于 BFS 的双端队列优化和单调队列。

list / forward_list -- 链表

list 为双向链表，forward_list 为单向链表。在竞赛中使用较少，因为内存不连续、缓存不友好。

#include <list>
list<int> lst = {1, 2, 3, 4};
auto it = lst.begin();
advance(it, 2);           // it 指向第 3 个元素 (3)
lst.insert(it, 10);        // [1, 2, 10, 3, 4] O(1)
lst.erase(it);              // [1, 2, 10, 4] O(1) (it 仍有效)
lst.splice(lst.begin(), lst, prev(lst.end())); // 把末尾移到头部
lst.sort();                  // O(n log n)，链表自己的 sort
lst.unique();               // 去除相邻重复元素（需先排序）
lst.merge(other_lst);       // 合并两个有序链表
list 插入/删除: O(1) （已有迭代器时）
list 查找: O(n) （不支持随机访问）
竞赛建议

竞赛中几乎不需要 list / forward_list。需要链表时通常手写链式前向星或用 vector + 下标 模拟。仅在需要大量中间插删且不需要随机访问时考虑。

1.2 关联容器 (有序)

基于红黑树实现，元素自动有序排列。

set / multiset -- 有序集合

set 不允许重复元素，multiset 允许重复。支持 O(log n) 插入、删除、查找。

操作时间复杂度说明
insert(x)O(log n)插入元素
erase(x)O(log n)删除值为 x 的所有元素
erase(it)O(1)删除迭代器指向的元素
find(x)O(log n)返回迭代器，找不到返回 end()
count(x)O(log n + k)元素 x 的个数（set 只有 0 或 1）
lower_bound(x)O(log n)>= x 的第一个元素
upper_bound(x)O(log n)> x 的第一个元素
 
#include <set>

// set: 自动去重 + 有序
set<int> s = {3, 1, 4, 1, 5};
// s = {1, 3, 4, 5}  自动排序去重
s.insert(2);          // {1, 2, 3, 4, 5}
s.erase(3);           // {1, 2, 4, 5}

if (s.find(4) != s.end())  // 存在
    puts("found");

auto it = s.lower_bound(3);  // 指向 4（第一个 >= 3）
auto it2 = s.upper_bound(3); // 指向 4（第一个 > 3）

// 遍历（有序）
for (int x : s) printf("%d ", x);
// 输出: 1 2 4 5

// multiset: 允许重复
multiset<int> ms = {1, 2, 2, 3, 3, 3};
ms.insert(2);       // {1, 2, 2, 2, 3, 3, 3}
ms.count(2);        // 返回 3

// 删除 multiset 中某个特定元素（只删一个）
ms.erase(ms.find(2)); // 只删一个 2

// 获取最小/最大值
int mn = *s.begin();         // 最小值
int mx = *s.rbegin();        // 最大值（反向迭代器）
multiset 的 erase 陷阱

ms.erase(x) 会删除 所有 值为 x 的元素！如果只想删一个，必须用 ms.erase(ms.find(x))。但要先确认 x 存在（find 返回 end() 时不要 erase）。

map / multimap -- 有序映射

map 是键值对的有序集合，键唯一。底层红黑树，按键排序。

操作时间复杂度说明
m[key]O(log n)访问/插入键 key（不存在则默认初始化）
m.at(key)O(log n)访问键 key（不存在则抛异常）
m.insert({k,v})O(log n)插入，已存在则不覆盖
m.emplace(k,v)O(log n)原地构造插入
m.count(k)O(log n)键 k 是否存在（0 或 1）
m.find(k)O(log n)查找，返回迭代器
m.erase(k)O(log n)删除键 k
 
#include <map>
#include <string>

map<string, int> mp;

// 插入方式
mp["apple"] = 3;                 // [] 会自动创建
mp.insert({"banana", 5});         // 已存在则无效
mp.emplace("cherry", 7);          // 原地构造

// 访问
int val = mp["apple"];            // 3
int val2 = mp["unknown"];         // 0! 注意：会自动插入

// 安全访问（不自动插入）
if (mp.count("unknown"))          // 不存在，返回 0
    printf("found\n");

auto it = mp.find("apple");       // 返回迭代器
if (it != mp.end())
    printf("%s: %d\n", it->first.c_str(), it->second);

// 遍历（按键的升序）
for (auto& [k, v] : mp)          // C++17 结构化绑定
    printf("%s: %d\n", k.c_str(), v);

// 统计词频的经典写法
map<string, int> freq;
string word;
while (cin >> word)
    freq[word]++;                  // 自动初始化为 0 再 ++
operator[] 的陷阱

m[key] 在 key 不存在时会自动插入一个默认值！如果只想查询，用 m.find(key) 或 m.count(key)。multimap 不支持 [] 操作。

1.3 无序容器 (哈希)

基于哈希表实现，平均 O(1) 操作，但最坏 O(n)。

unordered_set / unordered_map

#include <unordered_set>
#include <unordered_map>

// unordered_set: O(1) 查找/插入
unordered_set<int> us = {3, 1, 4, 1, 5};
us.insert(2);
us.erase(3);
if (us.count(4))  // O(1) 平均
    puts("found");

// unordered_map: O(1) 键值对
unordered_map<string, int> um;
um["hello"] = 42;
um["world"] = 100;
for (auto& [k, v] : um)
    printf("%s: %d\n", k.c_str(), v);

// bucket 相关
printf("bucket_count: %zu\n", us.bucket_count());
printf("load_factor: %.2f\n", us.load_factor());
us.reserve(1000);          // 预分配桶，减少 rehash
us.max_load_factor(0.25);  // 降低负载因子，减少冲突
操作平均最坏说明
insert / erase / findO(1)O(n)哈希冲突严重时退化
遍历O(n + bucket_count)空桶也会遍历
竞赛中 unordered_map 的坑
  • 如果哈希函数被攻击者刻意构造导致大量冲突，复杂度退化为 O(n)。在不确定数据范围时 map 更安全
  • 自定义类型需要提供哈希函数（重载 std::hash 或传入自定义仿函数）
  • 遍历顺序不确定（与插入顺序不同）
  • 竞赛中如需稳定性能，优先用 map（O(log n) 无最坏退化）

1.4 容器适配器

stack -- 栈

#include <stack>
stack<int> st;
st.push(1);      // 入栈
st.push(2);
st.push(3);
int top = st.top(); // 3 (栈顶)
st.pop();           // 弹出栈顶
bool empty = st.empty(); // false
size_t sz = st.size();   // 2
push / pop / top: 均 O(1)

queue -- 队列

#include <queue>
queue<int> q;
q.push(1);       // 入队
q.push(2);
q.push(3);
int front = q.front(); // 1 (队首)
int back  = q.back();  // 3 (队尾)
q.pop();               // 弹出队首
push / pop / front / back: 均 O(1)

priority_queue -- 优先队列（堆）

默认为大顶堆（最大元素优先出队），底层用 vector 实现。

操作时间复杂度说明
push(x)O(log n)插入元素
pop()O(log n)弹出堆顶
top()O(1)查看堆顶
size() / empty()O(1)大小/判空
 
#include <queue>
#include <functional>

// 大顶堆（默认）
priority_queue<int> pq;
pq.push(3); pq.push(1); pq.push(4);
int mx = pq.top(); // 4

// 小顶堆
priority_queue<int, vector<int>, greater<int>> minpq;
minpq.push(3); minpq.push(1); minpq.push(4);
int mn = minpq.top(); // 1

// pair 作为元素（先比较 first，再比较 second）
priority_queue<pair<int,int>> pq2;
pq2.push({1, 100});
pq2.push({2, 50});
pq2.push({2, 200}); // top = {2, 200}

// 自定义比较器（仿函数）
struct Cmp {
    bool operator()(pair<int,int> a, pair<int,int> b) {
        return a.first > b.first; // 小顶堆（按 first）
    }
};
priority_queue<pair<int,int>, vector<pair<int,int>>, Cmp> pq3;
priority_queue vs set/multiset 实现堆

两者都能实现堆。priority_queue 更轻量（仅堆顶访问），但不能删除中间元素。需要删除任意元素时用 multiset（或手写带删除的堆）。

1.5 工具类: pair / tuple

pair

#include <utility>

// 创建
pair<int,int> p1 = {1, 2};
pair<int,int> p2 = make_pair(3, 4);
auto p3 = make_pair(5, 6);

// 访问
int a = p1.first;   // 1
int b = p1.second;  // 2
auto& [x, y] = p1;  // C++17 结构化绑定: x=1, y=2

// 比较：先比 first，再比 second
pair<int,int> a1 = {1, 3};
pair<int,int> a2 = {1, 2};
if (a1 > a2) puts("a1 > a2"); // true: first相同，比second

// 常用于 map 的 insert 和返回多个值
map<int,int> mp;
mp.insert({1, 10});  // pair<int,int>

tuple

#include <tuple>

// 创建
tuple<int, string, double> t1(1, "hello", 3.14);
auto t2 = make_tuple(2, "world", 2.72);

// 访问
int id = get<0>(t1);        // 1
string name = get<1>(t1);   // "hello"

// C++17 结构化绑定
auto& [id2, name2, val2] = t1;

// C++17 类型推断获取
auto& nm = get<string>(t1); // 按类型获取

// tie 解包
int a; string b; double c;
tie(a, b, c) = t1;            // a=1, b="hello", c=3.14
tie(a, ignore, c) = t1;      // 忽略第二个元素

// 比较（按字典序）
auto t3 = make_tuple(1, 2, 3);
auto t4 = make_tuple(1, 2, 4);
if (t3 < t4) puts("t3 < t4"); // true

// tuple_size 获取元素个数
printf("size: %zu\n", tuple_size<decltype(t1)>::value); // 3
pair 与 tuple 的竞赛用途
  • pair: 存储两个值的组合，如坐标 (x,y)、边 (u,v)、键值对
  • tuple: 存储三个及以上值，如图的边 (u,v,w)、Dijkstra 状态 (dist,node,flag)
  • 两者都支持字典序比较，可用于排序、set/map 的键

二、常用算法 (Algorithms)

STL 算法头文件 <algorithm>、<numeric> 提供了大量通用算法。竞赛中掌握这些可以大幅提高编码效率。

2.1 排序算法

sort -- 快速排序 (IntroSort)

底层为 IntroSort（快排 + 堆排 + 插入排序混合），平均 O(n log n)，不稳定。

#include <algorithm>
#include <vector>

vector<int> v = {5, 2, 8, 1, 9, 3};

// 升序排序
sort(v.begin(), v.end());                // [1, 2, 3, 5, 8, 9]

// 降序排序
sort(v.begin(), v.end(), greater<int>()); // [9, 8, 5, 3, 2, 1]

// 自定义比较器
vector<pair<int,int>> vp = {{1,3}, {2,1}, {1,2}};
sort(vp.begin(), vp.end(), [](auto& a, auto& b) {
    if (a.first != b.first) return a.first < b.first;
    return a.second > b.second;
});
// 结果: [{1,3}, {1,2}, {2,1}]

// 数组排序
int arr[] = {5, 2, 8, 1};
sort(arr, arr + 4);                     // [1, 2, 5, 8]
sort(arr, arr + 4, greater<int>());    // 降序

// 字符串排序
string s = "dcba";
sort(s.begin(), s.end());                // "abcd"

// 部分排序（只排序 [first, middle) 部分）
partial_sort(v.begin(), v.begin() + 3, v.end());
// 前 3 个是最小的 3 个，已排序；其余无序
sort: O(n log n)

stable_sort -- 稳定排序

保持相等元素的相对顺序不变。底层为归并排序，O(n log n)，但需要 O(n) 额外空间。

vector<pair<int,string>> students = {
    {90, "Alice"}, {85, "Bob"}, {90, "Charlie"}, {85, "Dave"}
};
// 按成绩排序，相同成绩保持原顺序
stable_sort(students.begin(), students.end(), [](auto& a, auto& b) {
    return a.first > b.first;
});
// 结果: {90,Alice}, {90,Charlie}, {85,Bob}, {85,Dave}
// 注意 Bob 在 Dave 前面（保持了原顺序）

nth_element -- 第 n 小元素

将第 n 小的元素放到正确位置，左边都比它小，右边都比它大（但两边不一定有序）。

vector<int> v = {5, 2, 8, 1, 9, 3, 7, 4, 6};

// 求第 4 小的元素（下标 3）
nth_element(v.begin(), v.begin() + 3, v.end());
int kth = v[3]; // 第 4 小的值

// 求前 k 小的元素（无序）
int k = 5;
nth_element(v.begin(), v.begin() + k, v.end());
// v[0..k-1] 是最小的 k 个元素（无序）

// 求中位数
int n = v.size();
nth_element(v.begin(), v.begin() + n/2, v.end());
int median = v[n/2];
nth_element 平均: O(n) （比 sort 的 O(n log n) 更快）
sort 的比较器要求

比较函数必须满足严格弱序：不能出现 a < a 为 true 的情况。对于浮点数、pair 等类型要特别注意。常见错误：在比较器中使用 <= 而非 <。

2.2 二分查找

lower_bound / upper_bound

在有序序列中进行二分查找。

#include <algorithm>

vector<int> v = {1, 2, 2, 3, 3, 3, 4, 5};

// lower_bound: 第一个 >= x 的迭代器
auto it1 = lower_bound(v.begin(), v.end(), 3);
int idx1 = it1 - v.begin();  // 3 (下标)
int val1 = *it1;              // 3 (值)

// upper_bound: 第一个 > x 的迭代器
auto it2 = upper_bound(v.begin(), v.end(), 3);
int idx2 = it2 - v.begin();  // 6 (下标)
int val2 = *it2;              // 4 (值)

// 统计 x 的个数
int cnt = upper_bound(v.begin(), v.end(), 3)
        - lower_bound(v.begin(), v.end(), 3);
// cnt = 3 (有 3 个 3)

// binary_search: 判断 x 是否存在
bool exists = binary_search(v.begin(), v.end(), 3); // true
bool notfound = binary_search(v.begin(), v.end(), 6); // false

// 在 set/map 中使用（自带成员函数，O(log n)）
set<int> s = {1, 3, 5, 7};
auto it3 = s.lower_bound(4); // 指向 5

// 降序序列需要反转比较器
vector<int> desc = {5, 3, 3, 1};
auto it4 = lower_bound(desc.begin(), desc.end(), 3, greater<int>());
// 第一个 <= 3 的位置
lower_bound / upper_bound / binary_search: O(log n) （随机访问迭代器）
lower_bound / upper_bound 使用注意
  • 序列必须是有序的（升序用默认比较器，降序用 greater）
  • 对非随机访问迭代器（如 list、set），复杂度为 O(n) 而非 O(log n)
  • lower_bound(v.begin(), v.end(), x) 等价于 set::lower_bound(x)
  • 不要对 unordered_set 使用这些函数（无序）

2.3 变换算法

unique -- 去重

去除相邻的重复元素，返回新末尾迭代器。通常先排序再去重。

vector<int> v = {1, 3, 2, 3, 1, 2, 3};
sort(v.begin(), v.end());   // [1, 1, 2, 2, 3, 3, 3]
auto it = unique(v.begin(), v.end());
v.erase(it, v.end());       // 真正删除尾部垃圾: [1, 2, 3]

// 一行去重模板
v.erase(unique(v.begin(), v.end()), v.end());

// 计算去重后元素个数
int n = unique(v.begin(), v.end()) - v.begin();
unique: O(n)

reverse -- 反转

vector<int> v = {1, 2, 3, 4, 5};
reverse(v.begin(), v.end()); // [5, 4, 3, 2, 1]

string s = "abcde";
reverse(s.begin(), s.end()); // "edcba"

// 部分反转
reverse(v.begin(), v.begin() + 3); // 反转前 3 个

rotate -- 循环移位

vector<int> v = {1, 2, 3, 4, 5};
// 左旋: 将 it 指向的元素移到开头
rotate(v.begin(), v.begin() + 2, v.end());
// v = [3, 4, 5, 1, 2]

// 右旋 1 位
rotate(v.begin(), v.end() - 1, v.end());
// v = [2, 3, 4, 5, 1]

next_permutation / prev_permutation -- 全排列

生成下一个/上一个字典序排列。需要先排序才能遍历所有排列。

vector<int> v = {1, 2, 3};

// 枚举所有排列（必须先排序）
do {
    for (int x : v) printf("%d ", x);
    puts("");
} while (next_permutation(v.begin(), v.end()));
// 输出: 123, 132, 213, 231, 312, 321

// 判断是否还有下一个排列
bool has_next = next_permutation(v.begin(), v.end());
// v 已经是 {3,2,1}，没有下一个，返回 false

// 上一个排列
prev_permutation(v.begin(), v.end());
next_permutation / prev_permutation: O(n) （每次调用）

min_element / max_element / minmax_element

vector<int> v = {3, 1, 4, 1, 5, 9};

// 最小值
auto mn = min_element(v.begin(), v.end());
printf("min = %d at pos %d\n", *mn, (int)(mn - v.begin()));

// 最大值
auto mx = max_element(v.begin(), v.end());
printf("max = %d\n", *mx);

// 同时获取最小和最大（只遍历一次）
auto [lo, hi] = minmax_element(v.begin(), v.end());
printf("min = %d, max = %d\n", *lo, *hi);

// 自定义比较器
auto cmp = [](int a, int b) { return abs(a) < abs(b); };
auto it = min_element(v.begin(), v.end(), cmp);
min_element / max_element: O(n)
minmax_element: O(n * 3/2) （比分开调用更高效）

2.4 数值算法

accumulate -- 累加

#include <numeric>

vector<int> v = {1, 2, 3, 4, 5};

// 求和（初始值 0）
int sum = accumulate(v.begin(), v.end(), 0);
// sum = 15

// 求积（初始值 1）
int prod = accumulate(v.begin(), v.end(), 1, multiplies<int>());
// prod = 120

// 拼接字符串
vector<string> vs = {"hello", " ", "world"};
string result = accumulate(vs.begin(), vs.end(), string(""));
// result = "hello world"

// 注意：初始值类型决定返回类型！
// 如果初始值是 int，大数会溢出，用 0LL
long long sum2 = accumulate(v.begin(), v.end(), 0LL);
accumulate 溢出问题

初始值的类型决定了累加结果的类型。如果 vector<int> 的元素很大，必须用 0LL 作为初始值来获得 long long 结果。

partial_sum -- 前缀和

vector<int> v = {1, 2, 3, 4, 5};
vector<int> pre(v.size());

// 计算前缀和
partial_sum(v.begin(), v.end(), pre.begin());
// pre = [1, 3, 6, 10, 15]

// 自定义操作（前缀积）
vector<int> pre2(v.size());
partial_sum(v.begin(), v.end(), pre2.begin(), multiplies<int>());
// pre2 = [1, 2, 6, 24, 120]

// 利用前缀和求区间和: sum[l..r] = pre[r] - pre[l-1]
partial_sum: O(n)

stringstream -- 字符串流

#include <sstream>

// 字符串分割（按空格）

sscanf / sprintf -- C 风格格式化

#include <cstdio>

// sscanf: 从字符串读取
char buf[] = "2023-12-25";
int y, m, d;
sscanf(buf, "%d-%d-%d", &y, &m, &d);
// y=2023, m=12, d=25

char buf2[] = "Name: Alice, Age: 20";
char name[50]; int age;
sscanf(buf2, "Name: %s, Age: %d", name, &age);
// name="Alice", age=20

// sprintf: 格式化写入字符串
char out[100];
sprintf(out, "%04d-%02d-%02d", y, m, d);
// out = "2023-12-25"

// snprintf: 安全版本，限制长度
snprintf(out, sizeof(out), "Result: %d", 42);

// 浮点数格式化
sprintf(out, "%.2f", 3.14159); // "3.14"
sprintf(out, "%10.3f", 3.14);  // "     3.140" (右对齐)
sprintf(out, "%-10.3f", 3.14); // "3.140     " (左对齐)
stringstream vs sscanf/sprintf 选择
  • stringstream: 类型安全，支持 C++ 类型（string 等），但较慢
  • sscanf/sprintf: 速度快，但需手动管理缓冲区，对 string 不友好
  • 竞赛中需要大量字符串解析时，sscanf 更高效
  • 处理含有 string 类型的数据时，用 stringstream

四、输入输出优化

4.1 同步与绑定

默认情况下，C++ 的 cin/cout 与 C 的 scanf/printf 同步，以保证两者可以混用。但这会带来性能开销。

// 关闭同步 + 解绑 cin/cout（竞赛模板必备）
int main() {
    ios::sync_with_stdio(false);  // 关闭 C/C++ 同步
    cin.tie(0);                     // 解绑 cin 和 cout
    // 此后不要混用 cin/cout 和 scanf/printf！

    // 可选：加速 cout
    cout.tie(0);                   // 解绑 cout（默认绑定到 cin）
    // 注意：关闭同步后 cout 不会自动刷新
    // 如需在 cout 后立即输出，手动 cout.flush()
}
各选项的作用
  • ios::sync_with_stdio(false): 关闭 C/C++ 流同步，cin/cout 速度大幅提升
  • cin.tie(0): 解除 cin 与 cout 的绑定。默认每次读入前会刷新 cout
  • cout.tie(0): 解除 cout 的绑定（通常 cout 绑定到 cin）

4.2 scanf/printf vs cin/cout 性能对比

方式读入 10^6 个 int说明
cin（默认）~2000ms非常慢，不推荐
cin + sync(false)~300ms推荐，简洁易用
scanf~400ms标准选择
cin + sync(false) + tie(0)~280ms最快 cin 方案
手写快读 (getchar)~120ms极端优化
 
// 方案一：关闭同步的 cin/cout（推荐）
#include <iostream>
using namespace std;
int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    int n; cin >> n;
    for (int i = 0; i < n; i++) {
        int x; cin >> x;
        cout << x << '\n';  // '\n' 比 endl 快（endl 会刷新）
    }
}

// 方案二：scanf/printf
#include <cstdio>
int main() {
    int n; scanf("%d", &n);
    for (int i = 0; i < n; i++) {
        int x; scanf("%d", &x);
        printf("%d\n", x);
    }
}

// 方案三：手写快读（极端优化）
inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + ch - '0';
        ch = getchar();
    }
    return x * f;
}

inline void write(int x) {
    if (x < 0) { putchar('-'); x = -x; }
    if (x > 9) write(x / 10);
    putchar(x % 10 + '0');
}
endl vs '\n'

endl 除了换行还会刷新缓冲区，性能很差。竞赛中输出大量数据时，务必用 '\n' 代替 endl。关闭同步后更应注意。

5.1 迭代器失效总结

5.2 常见错误模式

经典错误集合
// 错误 1: vector 边遍历边删除
vector<int> v = {1, 2, 3, 4, 5};
// 错误写法:
for (auto it = v.begin(); it != v.end(); it++) {
    if (*it == 3) v.erase(it);  // BUG! erase 后 it 失效
}
// 正确写法:
for (auto it = v.begin(); it != v.end(); ) {
    if (*it == 3) it = v.erase(it);  // erase 返回下一个有效迭代器
    else ++it;
}
// C++20: std::erase(v, 3);

// 错误 2: map 的 [] 自动插入
map<string, int> mp;
if (mp["key"] == 0)  // "key" 不存在时自动插入值 0
    puts("not found"); // 会输出！因为 0==0
// 正确: 用 mp.count("key") 或 mp.find("key") != mp.end()

// 错误 3: multiset erase 删除全部
multiset<int> ms = {1, 2, 2, 2, 3};
ms.erase(2);  // 删除所有 2！只剩 {1, 3}
// 只删一个: ms.erase(ms.find(2));

// 错误 4: 未排序就用 lower_bound
vector<int> v2 = {5, 2, 8, 1};
auto it2 = lower_bound(v2.begin(), v2.end(), 3); // 未排序！结果不确定

// 错误 5: accumulate 溢出
vector<int> v3(1000000, 1000000);
int sum = accumulate(v3.begin(), v3.end(), 0); // 溢出！int 范围
long long sum2 = accumulate(v3.begin(), v3.end(), 0LL); // 正确

5.3 竞赛最佳实践

STL 使用口诀
  • 能用 STL 就用 STL：少手写，减少 bug
  • reserve 先行：已知大小就 reserve，减少扩容
  • emplace 优先：比 push 更高效（原地构造）
  • '\\n' 代替 endl：避免不必要的刷新
  • lower_bound 代替手写二分：STL 的二分已高度优化
  • pair 当 key：多关键字排序用 vector<pair> + sort 或 set<pair>
  • tie 解包返回值：函数返回多个值时用 tie(a,b,c) = func()
  • 结构化绑定 (C++17)：auto [a, b] = make_pair(1, 2);
