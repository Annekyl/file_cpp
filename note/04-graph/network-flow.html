1. 网络流概述

1.1 什么是网络流

网络流（Network Flow）是图论中一类重要的模型，用于描述"物质"在网络中的流动。想象一个水管网络：水从源头（source）流出，经过各段管道（每段管道有最大流量限制），最终汇入汇点（sink）。网络流要解决的核心问题是：从源点到汇点，最大能输送多少流量？

  网络流模型示意：

      s ──(10)──→ A ──(8)──→ t
      │           ↑           ↑
      └──(8)──→ B ──(10)──→ C
                  └──(5)──→ t

  s = 源点 (Source), t = 汇点 (Sink)
  边上数字 = 容量 (Capacity)

  问题：从 s 到 t 的最大流量是多少？

1.2 网络流的应用领域

  • 物流运输：交通网络中的最大运力
  • 二分图匹配：最大匹配可转化为最大流
  • 项目选择：最小割模型解决收益最大化问题
  • 图像分割：计算机视觉中的前景/背景分离
  • 棒球联赛淘汰问题：判断某队是否有夺冠可能

2. 基本概念

2.1 定义

术语 定义
流网络 (Flow Network) 有向图 G = (V, E)，每条边 (u,v) 有容量 c(u,v) >= 0
源点 (Source) s 只有出边的顶点，流的起点
汇点 (Sink) t 只有入边的顶点，流的终点
可行流 f 满足容量约束和流量守恒的函数 f: E → R+
流量 |f| 从 s 流出的净流量 = 流入 t 的净流量
残量网络 (Residual Graph) 在当前流 f 下，包含所有残量 > 0 的边构成的图
增广路 (Augmenting Path) 残量网络中从 s 到 t 的路径

2.2 两个约束条件

流的基本约束
  1. 容量约束：对每条边 (u,v)，0 <= f(u,v) <= c(u,v)
  2. 流量守恒：对每个非源非汇的顶点 v，流入量 = 流出量

2.3 残量网络与反向边

残量网络是网络流算法的核心概念。对于原图中的每条边 (u, v) 容量为 c：

  • 正向残量：c - f(u,v)（还可以增加的流量）
  • 反向残量：f(u,v)（可以退回的流量）

实现时，每条边存为两条：正向边（容量 c）和反向边（容量 0）。增广时正向边减少流量，反向边增加流量。

  反向边的作用：

  原图:  A ──(10)──→ B

  实际存储（链式前向星）:
  边0: A → B, cap = 10  (正向)
  边1: B → A, cap = 0   (反向)

  流过 6 单位后:
  边0: A → B, cap = 4   (还剩 4)
  边1: B → A, cap = 6   (可退回 6)

  反向边使得算法能够"反悔"之前的决策

3. Ford-Fulkerson 方法

3.1 基本思想

Ford-Fulkerson 是求最大流的方法论框架，核心思想是：反复在残量网络中寻找增广路，沿增广路增加流量，直到不存在增广路为止。

初始化：所有边的流量为 0
在残量网络中寻找一条从 s 到 t 的增广路
找到增广路后，计算路径上的瓶颈容量（最小残量）
沿增广路增加瓶颈容量的流量，更新残量网络
重复步骤 2-4，直到不存在增广路
此时的流量即为最大流
Ford-Fulkerson 的局限

若增广路的选择不加限制（如用 DFS 随意搜索），时间复杂度为 O(E * |f*|)，其中 |f*| 是最大流值。当容量为浮点数或极大整数时，可能非常慢甚至不终止。

4. Edmonds-Karp 算法

4.1 改进思路

Edmonds-Karp 是 Ford-Fulkerson 的具体实现：使用 BFS 寻找增广路（选择最短增广路），保证了多项式时间复杂度。

4.2 复杂度分析

时间复杂度 O(VE^2)
空间复杂度 O(V + E)

为什么是 O(VE^2)？关键观察：

  • BFS 找到的最短增广路长度随增广次数单调不减
  • 某条边 (u,v) 成为关键边（瓶颈）时，下次再成为关键边时 s→u 和 v→t 的最短路至少各增加 2
  • 每条边最多成为关键边 O(V) 次
  • 总共 O(VE) 次增广，每次 BFS 花费 O(E)
适用场景

Edmonds-Karp 适合中小规模网络（V, E 在几千以内），代码简洁，不易出错。大规模网络应使用 Dinic 算法。

5. Dinic 算法详解

5.1 算法思想

Dinic 算法是目前竞赛中最常用的最大流算法，通过引入分层图（Level Graph）和阻塞流（Blocking Flow）的概念，将每次 BFS + 多次 DFS 结合，大幅提高了效率。

5.2 核心概念

  • 分层图（Level Graph）：用 BFS 从 s 出发计算每个顶点到 s 的最短距离（层数），只保留从第 i 层到第 i+1 层的边
  • 阻塞流（Blocking Flow）：在分层图上，从 s 到 t 的所有路径上都已经有流经过的流
  • 当前弧优化（Current Arc Optimization）：对每个顶点记录上次尝试到哪条边，避免重复搜索已经确定无法增广的边

5.3 算法流程

BFS 构建分层图：从 s 出发 BFS，计算每个顶点的层数 level[v]。若无法到达 t，算法结束。
DFS 寻找阻塞流：从 s 出发 DFS，只沿着 level[v] = level[u] + 1 的边前进（沿分层图），寻找多条增广路，直到从 s 出发无法再到达 t。
当前弧优化：DFS 时，对每个顶点 u 记录当前搜到第几条边 cur[u]，避免重复搜索已确定无法增广的边。
重复步骤 1-3，直到 BFS 无法到达 t。
  Dinic 算法执行过程：

  第 1 轮 BFS: s(层0) → A(层1) → B(层1) → t(层2)
  分层图: s→A, s→B, A→t, B→t

  第 1 轮 DFS: 在分层图上反复找增广路
  增广路1: s→A→t, 瓶颈=5
  增广路2: s→B→t, 瓶颈=3
  ...直到无路可走（阻塞流完成）

  第 2 轮 BFS: 重新计算分层
  ...重复直到 t 不可达

  ┌─────────────────────────────┐
  │  每轮: BFS建分层 + DFS找阻塞流  │
  │  最多 O(V) 轮，每轮 O(VE)     │
  │  总复杂度: O(V^2 * E)         │
  └─────────────────────────────┘

5.4 为什么分层图有效

正确性论证

分层图保证每次 DFS 只走最短增广路（边数最少）。当 BFS 发现 t 不可达时，意味着残量网络中不存在 s-t 路径，由最大流最小割定理，此时流达到最大。

5.5 当前弧优化的重要性

没有当前弧优化时，Dinic 的复杂度为 O(V^2 * E * log C)。加入当前弧优化后：

  • 每条边至多被考虑一次（在某一轮的 DFS 中）
  • 每轮 DFS 的总代价为 O(VE)
  • 总共 O(V) 轮
  • 总复杂度：O(V^2 * E)
Dinic 在特殊图上的表现
  • 单位容量图（所有边容量为 1）：O(min(V^(2/3), E^(1/2)) * E)
  • 二分图（s→U→V→t 结构）：O(V^(1/2) * E)
  • 因此 Dinic 求二分图最大匹配的效率优于匈牙利算法

5.6 复杂度对比

算法 时间复杂度 适用场景 难度
Ford-Fulkerson (DFS) O(E * |f*|) 小规模、整数容量 简单
Edmonds-Karp (BFS) O(VE^2) 中小规模 简单
Dinic O(V^2 * E) 竞赛首选 中等
ISAP O(V^2 * E) 实践中常数较小 较难
HLPP O(V^2 * sqrt(E)) 理论最优 困难

6. 最小割定理

6.1 割的定义

流网络 G = (V, E) 的一个 s-t 割 (S, T) 是将顶点集 V 分为两个集合 S 和 T，使得 s 属于 S，t 属于 T。

  • 割的容量：c(S, T) = sum of c(u,v) for all u in S, v in T
  • 最小割：容量最小的 s-t 割

6.2 最大流最小割定理

最大流最小割定理（Max-Flow Min-Cut Theorem）

以下三个命题等价：

  1. f 是最大流
  2. 残量网络中不存在增广路
  3. 存在一个 s-t 割 (S, T)，使得 |f| = c(S, T)

核心结论：最大流 = 最小割

6.3 最小割的构造

在最大流算法完成后，从 s 出发在残量网络中 BFS/DFS 能到达的所有顶点构成集合 S，其余顶点构成集合 T。割边就是从 S 到 T 的所有满流边。

运行最大流算法，得到残量网络
在残量网络中，从 s 出发 BFS/DFS，标记所有可达顶点
S = 被标记的顶点集合，T = 未被标记的顶点集合
最小割的割边 = 原图中从 S 到 T 且残量为 0 的边

6.4 最小割的应用

很多看起来与网络流无关的问题可以建模为最小割问题：

  • 项目选择问题：选择一些项目使收益最大（扣除依赖成本）
  • 图像分割：将像素分为前景和背景
  • 最大权闭合子图：选择一些顶点使得选中的集合是闭合的且权值和最大

7. 最小费用最大流（MCMF）

7.1 问题定义

在流网络中，每条边除了容量 c(u,v) 外，还有一个单位费用 w(u,v)。问题：在最大流的前提下，使总费用 sum(f(u,v) * w(u,v)) 最小。

7.2 核心思想

将最大流算法中的 BFS 替换为最短路算法（SPFA 或 Dijkstra），每次沿费用最小的增广路增广。

SPFA 增广法

每次用 SPFA 在残量网络中求从 s 到各点的最短路（以费用为边权），然后沿 s→t 的最短路增广。直到无法增广或最短路费用 >= 0（若要求最小费用最大流则直到无法增广）。

7.3 费用流的反向边

与最大流类似，费用流也需要反向边。反向边的费用是正向边费用的相反数：

  • 正向边 (u, v)：容量 c，费用 w
  • 反向边 (v, u)：容量 0，费用 -w
  • 增广时：正向边容量减少，反向边容量增加，费用取反

7.4 SPFA 的负环问题

由于反向边的费用为负，残量网络中可能存在负权边。因此不能直接用 Dijkstra（除非配合 Johnson 势函数），通常使用 SPFA。

SPFA 被卡的应对

SPFA 最坏复杂度 O(VE)，在构造数据上可能退化。应对策略：

  • 使用 SLF（Small Label First）优化：双端队列
  • 使用 Dijkstra + Johnson 势函数（Primal-Dual 方法），复杂度 O(F * E * log V)
  • 实践中 SPFA 通常很快，多数 OJ 题目可通过

7.5 复杂度分析

算法 时间复杂度 备注
SPFA 增广 O(F * VE) F 为最大流值，最坏较慢
Dijkstra + 势函数 O(F * E * log V) Primal-Dual，更快更稳
Capacity Scaling O(E^2 * log U) U 为最大容量

8. 应用与建模

8.1 二分图匹配转化为最大流

将二分图最大匹配转化为最大流问题是网络流最经典的应用之一。

  二分图匹配 → 最大流建模：

  原二分图:
  U = {u1, u2, u3}     V = {v1, v2, v3}
  边: u1-v1, u1-v2, u2-v1, u3-v2

  添加超级源 s 和超级汇 t：
      s ──(1)──→ u1 ──(1)──→ v1 ──(1)──→ t
      s ──(1)──→ u2 ──(1)──→ v2 ──(1)──→ t
      s ──(1)──→ u3
                    u1 ──(1)──→ v2
                    u2 ──(1)──→ v1

  所有新增边容量为 1
  最大流 = 最大匹配数

8.2 项目选择问题（最大权闭合子图）

有 n 个项目，每个项目有收益（正或负）。选择某些项目时，必须同时选择其依赖的项目。求最大总收益。

建模方法
  1. 源 s 向所有正收益项目连边，容量 = 收益值
  2. 所有负收益项目向汇 t 连边，容量 = |收益值|
  3. 项目 i 依赖项目 j 时，连边 i → j，容量 = +∞
  4. 最大权闭合子图的权值 = 正收益总和 - 最小割

8.3 最大权闭合子图构造

在最小割 (S, T) 中：

  • S ∩ {正收益项目} = 选中的正收益项目
  • T ∩ {负收益项目} = 不选的负收益项目
  • 闭合子图 = S 中除 s 外的所有顶点

8.4 棒球联赛淘汰问题

n 支队伍，已知每支队伍已胜场数和剩余比赛安排。判断某支队伍是否有可能成为冠军（或并列冠军）。

建模：每场未进行的比赛建一个顶点（源连向它），每支其他队伍建一个顶点（它连向汇），容量约束使得每场比赛的胜利分配不超过队伍的胜场上限。

8.5 方格取数问题

在 n*m 的方格中，每个格子有数值，取出一些格子使得取出的格子互不相邻，且数值和最大。转化为最小割（最大权独立集）。

  最小割建模通用框架：

  ┌─────────────────────────────────────┐
  │                                       │
  │  选择类问题 → 每个元素选/不选 → S/T 割 │
  │                                       │
  │  收益类 → s 连正权, 负权连 t           │
  │  依赖类 → 依赖边容量 +∞               │
  │  互斥类 → 互斥边容量 +∞               │
  │                                       │
  │  答案 = 总正收益 - 最小割              │
  └─────────────────────────────────────┘

9. C++ 完整模板

9.1 Dinic 最大流模板

#include <bits/stdc++.h>
using namespace std;

struct Dinic {
    struct Edge {
        int to, rev;
        long long cap;
    };
    vector<vector<Edge>> g;
    vector<int> level, cur;
    int n;

    void init(int _n) {
        n = _n;
        g.assign(n + 1, {});
        level.resize(n + 1);
        cur.resize(n + 1);
    }

    void addEdge(int u, int v, long long cap) {
        g[u].push_back({v, (int)g[v].size(), cap});
        g[v].push_back({u, (int)g[u].size() - 1, 0});
    }

    // BFS 构建分层图
    bool bfs(int s, int t) {
        fill(level.begin(), level.end(), -1);
        level[s] = 0;
        queue<int> q;
        q.push(s);
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (auto& e : g[u]) {
                if (e.cap > 0 && level[e.to] == -1) {
                    level[e.to] = level[u] + 1;
                    q.push(e.to);
                }
            }
        }
        return level[t] != -1;
    }

    // DFS 寻找阻塞流（带当前弧优化）
    long long dfs(int u, int t, long long flow) {
        if (u == t) return flow;
        long long total = 0;
        for (int& i = cur[u]; i < (int)g[u].size(); i++) {
            auto& e = g[u][i];
            if (e.cap > 0 && level[e.to] == level[u] + 1) {
                long long f = dfs(e.to, t, min(flow, e.cap));
                if (f > 0) {
                    e.cap -= f;
                    g[e.to][e.rev].cap += f;
                    total += f;
                    flow -= f;
                    if (flow == 0) break;
                }
            }
        }
        return total;
    }

    // 求 s 到 t 的最大流
    long long maxFlow(int s, int t) {
        long long total = 0;
        while (bfs(s, t)) {
            fill(cur.begin(), cur.end(), 0);
            total += dfs(s, t, LLONG_MAX);
        }
        return total;
    }

    // 获取最小割集合 S（从 s 可达的顶点）
    vector<bool> minCut(int s) {
        vector<bool> vis(n + 1, false);
        queue<int> q;
        q.push(s);
        vis[s] = true;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (auto& e : g[u])
                if (e.cap > 0 && !vis[e.to]) {
                    vis[e.to] = true;
                    q.push(e.to);
                }
        }
        return vis;
    }
};

9.2 最小费用最大流模板（SPFA 版）

struct MCMF {
    struct Edge {
        int to, rev;
        long long cap, cost;
    };
    vector<vector<Edge>> g;
    vector<long long> dist;
    vector<int> preu, pree; // 记录增广路上每个顶点的前驱
    vector<bool> inq;
    int n;

    void init(int _n) {
        n = _n;
        g.assign(n + 1, {});
        dist.resize(n + 1);
        preu.resize(n + 1);
        pree.resize(n + 1);
        inq.resize(n + 1);
    }

    void addEdge(int u, int v, long long cap, long long cost) {
        g[u].push_back({v, (int)g[v].size(), cap, cost});
        g[v].push_back({u, (int)g[u].size() - 1, 0, -cost});
    }

    // SPFA 求最短路（以费用为边权）
    bool spfa(int s, int t) {
        fill(dist.begin(), dist.end(), LLONG_MAX);
        fill(inq.begin(), inq.end(), false);
        dist[s] = 0;
        queue<int> q;
        q.push(s);
        inq[s] = true;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            inq[u] = false;
            for (int i = 0; i < (int)g[u].size(); i++) {
                auto& e = g[u][i];
                if (e.cap > 0 && dist[u] + e.cost < dist[e.to]) {
                    dist[e.to] = dist[u] + e.cost;
                    preu[e.to] = u;
                    pree[e.to] = i;
                    if (!inq[e.to]) {
                        q.push(e.to);
                        inq[e.to] = true;
                    }
                }
            }
        }
        return dist[t] != LLONG_MAX;
    }

    // 返回 {最大流, 最小费用}
    pair<long long, long long> minCostMaxFlow(int s, int t) {
        long long flow = 0, cost = 0;
        while (spfa(s, t)) {
            // 沿最短路增广
            long long f = LLONG_MAX;
            for (int v = t; v != s; v = preu[v])
                f = min(f, g[preu[v]][pree[v]].cap);
            for (int v = t; v != s; v = preu[v]) {
                auto& e = g[preu[v]][pree[v]];
                e.cap -= f;
                g[v][e.rev].cap += f;
            }
            flow += f;
            cost += f * dist[t];
        }
        return {flow, cost};
    }
};

9.3 Edmonds-Karp 模板（简洁版）

struct EdmondsKarp {
    int n;
    long long cap[N][N]; // 邻接矩阵存容量
    int pre[N];

    void init(int _n) {
        n = _n;
        memset(cap, 0, sizeof(cap));
    }

    void addEdge(int u, int v, long long c) {
        cap[u][v] += c; // 注意：+= 处理重边
    }

    long long bfs(int s, int t) {
        memset(pre, -1, sizeof(pre));
        queue<pair<int, long long>> q;
        q.push({s, LLONG_MAX});
        pre[s] = s;
        while (!q.empty()) {
            auto [u, f] = q.front(); q.pop();
            for (int v = 1; v <= n; v++) {
                if (pre[v] == -1 && cap[u][v] > 0) {
                    pre[v] = u;
                    long long nf = min(f, cap[u][v]);
                    if (v == t) return nf;
                    q.push({v, nf});
                }
            }
        }
        return 0;
    }

    long long maxFlow(int s, int t) {
        long long total = 0, f;
        while ((f = bfs(s, t)) > 0) {
            total += f;
            for (int v = t; v != s; v = pre[v]) {
                cap[pre[v]][v] -= f;
                cap[v][pre[v]] += f;
            }
        }
        return total;
    }
};
