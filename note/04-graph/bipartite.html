1. 二分图概述

1.1 定义

二分图（Bipartite Graph）是顶点集可以分割为两个互不相交的子集 U 和 V，使得图中每条边的两个端点分别属于 U 和 V 的图。换句话说，同一集合内部没有边相连。

  二分图结构示意：

  集合 U (左部)          集合 V (右部)
  ┌─────────┐            ┌─────────┐
  │  u1 ──────────────── v1       │
  │  u2 ──────┬───────── v2       │
  │  u3 ──────┼───────── v3       │
  │  u4 ──────┘          v4       │
  └─────────┘            └─────────┘

  特征：所有边都是 "跨集合" 的，同集合内无边

1.2 核心性质

  • 无向图 G 是二分图 当且仅当 G 不含奇数长度的环（奇环）
  • 树一定是二分图（无环，自然无奇环）
  • 二分图的最大独立集 = 总顶点数 - 最大匹配数
  • 二分图的最小顶点覆盖 = 最大匹配数（König 定理）
核心概念

二分图的研究核心围绕三个经典问题：判定（是否为二分图）、匹配（最大匹配）、覆盖（最小顶点覆盖）。这三者通过 König 定理紧密联系。

2. 二分图判定（染色法）

2.1 原理

使用 BFS 或 DFS 对图进行 2-染色（黑白染色）。若存在一种染色方案使得相邻顶点颜色不同，则图为二分图。若染色过程中发现相邻顶点同色，则图含奇环，不是二分图。

2.2 BFS 染色

任选一个未染色的顶点，染为颜色 0（白色），加入队列
取出队首顶点 u，遍历其所有邻居 v
若 v 未染色：染为与 u 不同的颜色，入队
若 v 已染色且与 u 同色：存在奇环，不是二分图
队列为空则该连通分量染色成功；继续处理下一个未染色分量

2.3 BFS 染色代码

#include <bits/stdc++.h>
using namespace std;

const int N = 100010;
vector<int> g[N];
int color[N]; // 0=未染色, 1=白色, -1=黑色

bool bfs(int start) {
    queue<int> q;
    color[start] = 1;
    q.push(start);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : g[u]) {
            if (color[v] == 0) {
                color[v] = -color[u]; // 染相反颜色
                q.push(v);
            } else if (color[v] == color[u]) {
                return false; // 同色冲突，存在奇环
            }
        }
    }
    return true;
}

bool isBipartite(int n) {
    memset(color, 0, sizeof(color));
    for (int i = 1; i <= n; i++)
        if (color[i] == 0 && !bfs(i))
            return false;
    return true;
}
时间复杂度 O(V + E)
空间复杂度 O(V)

3. 匈牙利算法（Hungarian Algorithm）

3.1 问题定义

二分图最大匹配：给定二分图 G = (U, V, E)，找到一个边集 M，使得 M 中任意两条边没有公共端点，且 |M| 最大。

3.2 核心概念

  • 匹配（Matching）：边集 M 中没有两条边共享同一个顶点
  • 匹配点：被匹配边覆盖的顶点
  • 非匹配点：未被匹配边覆盖的顶点
  • 增广路（Augmenting Path）：从 U 中非匹配点出发，交替经过非匹配边和匹配边，到达 V 中非匹配点的路径
  • 交错路：匹配边和非匹配边交替出现的路径

3.3 增广路定理（Berge's Theorem）

Berge 增广路定理

一个匹配 M 是最大匹配，当且仅当图中不存在关于 M 的增广路。

找到一条增广路后，将路径上匹配边和非匹配边互换，匹配数 +1。

3.4 算法流程详解

匈牙利算法的核心思想：对每个左部顶点 u，尝试为其寻找匹配。若目标顶点 v 已被匹配，则递归尝试为 v 的原匹配对象寻找新的匹配（"让路"操作）。

  匈牙利算法 —— 增广路示意

  第 1 轮：u1 匹配 v1         第 2 轮：u2 想匹配 v1（已占用）
  u1 ═══ v1  (匹配)           u2 ─── v1  (尝试)
                               u1 ─── v2  (给 u1 找新匹配)
                               u2 ═══ v1, u1 ═══ v2  (增广!)

  交替路径: u2 → v1 → u1 → v2
  匹配边与非匹配边互换，匹配数 +1
初始化匹配数组 matchV[] = -1（表示未匹配）
遍历每个左部顶点 u
对 u 执行 DFS 寻找增广路：遍历 u 的邻居 v
若 v 未被匹配：直接匹配 u-v，返回成功
若 v 已被匹配：尝试为 matchV[v]（即 u'）找一个新匹配（递归 DFS），标记 v 为已访问避免重复
若递归成功：更新 matchV[v] = u，返回成功
若所有邻居都尝试失败：u 暂时无法匹配，返回失败
最终 matchV[] 中非 -1 的数量即为最大匹配数

3.5 逐步模拟

以如下二分图为例：U = {1, 2, 3}, V = {a, b, c}，边为 1-a, 1-b, 2-b, 2-c, 3-a

轮次 当前顶点 u 操作 匹配状态
1 u1 尝试 a：a 未匹配 → 直接匹配 1-a
2 u2 尝试 b：b 未匹配 → 直接匹配 1-a, 2-b
3 u3 尝试 a：a 已匹配给 u1 → 尝试为 u1 找新匹配 → u1 尝试 b：b 已匹配给 u2 → 尝试为 u2 找新匹配 → u2 尝试 c：c 未匹配 → 成功！回溯更新 1-b, 2-c, 3-a
时间复杂度 O(VE)
空间复杂度 O(V + E)
注意

匈牙利算法中每次 DFS 前需要重置 vis[] 数组。vis[] 的作用是标记本轮搜索中已经访问过的右部顶点，防止同一轮中重复访问。

4. König 定理

4.1 定理陈述

König 定理（1931）

在二分图中，最大匹配数 = 最小顶点覆盖数。

4.2 基本概念

  • 顶点覆盖：选取一些顶点，使得图中每条边至少有一个端点被选中
  • 最小顶点覆盖：包含顶点数最少的顶点覆盖
  • 独立集：选取一些顶点，使得任意两个被选中的顶点之间没有边相连
  • 最大独立集：包含顶点数最多的独立集

4.3 推论

对于二分图，有如下重要等式关系：

概念 公式
最大匹配数 = 最小顶点覆盖数（König 定理）
最大独立集 = 总顶点数 - 最小顶点覆盖数 = 总顶点数 - 最大匹配数
最小边覆盖 = 总顶点数 - 最大匹配数（前提：无孤立顶点）
DAG 最小路径覆盖 = 总顶点数 - 拆点二分图最大匹配数

4.4 最小顶点覆盖的构造

在匈牙利算法求出最大匹配后，可以通过以下步骤构造最小顶点覆盖：

从左部所有非匹配点出发，沿交错路（非匹配边→匹配边→...）做 DFS/BFS 标记
最小顶点覆盖 = (左部未标记的点) ∪ (右部已标记的点)
  最小顶点覆盖构造示意：

  最大匹配: u1-v1, u2-v2, u3-v3

  交错路搜索（从左部非匹配点出发）：
  u4 →(非匹配边) v1 →(匹配边) u1 →(非匹配边) v3 →(匹配边) u3

  标记结果: 左部标记={u4,u1,u3}, 右部标记={v1,v3}

  最小顶点覆盖 = 左部未标记 ∪ 右部已标记
               = {u2} ∪ {v1, v3}
               = {u2, v1, v3}  (大小 = 3 = 最大匹配数)

5. Hall 定理（婚姻定理）

5.1 定理陈述

Hall 定理（Philip Hall, 1935）

二分图 G = (U, V, E) 存在完美匹配（即 |U| 个左部顶点全部匹配）当且仅当：对于 U 的任意子集 S，有 |N(S)| >= |S|，其中 N(S) 是 S 中所有顶点的邻居集合。

5.2 直观理解

Hall 定理的本质是：任意 k 个左部顶点，至少要有 k 个不同的右部邻居可供选择。如果存在某个子集 S，其邻居数量少于 |S|，则 S 中必然有顶点无法匹配。

5.3 推广形式

对于不要求完美匹配的一般情况，Hall 定理可推广为：

最大匹配数 = min(|U|, min over S⊆U of (|U| - |S| + |N(S)|))

即最大匹配数 = |U| - max over S⊆U of (|S| - |N(S)|)

5.4 应用场景

  • 棋盘覆盖问题：用 1x2 骨牌覆盖棋盘，判断能否完全覆盖
  • 任务分配：判断 n 个工人能否各分配一个不同任务
  • 集合系统：判断是否存在 SDR（不同代表系）

6. 应用与建模

6.1 指派问题（Assignment Problem）

n 个工人，n 项任务，每个工人能胜任若干任务，求最大分配方案数。直接建模为二分图最大匹配问题。

6.2 最小路径覆盖（DAG）

在 DAG 上，用最少的不相交路径覆盖所有顶点。

建模方法

将每个顶点 v 拆成两个：v_in（左部）和 v_out（右部）。对于原图中的每条边 u→v，在二分图中连边 u_out → v_in。则最小路径覆盖 = n - 最大匹配数。

6.3 二分图最小点覆盖

选择最少的顶点，使得每条边至少有一个端点被选中。根据 König 定理，直接等于最大匹配数。

6.4 二分图最大独立集

选择最多的顶点，使得任意两个被选中的顶点之间没有边。最大独立集 = 总顶点数 - 最大匹配数。

6.5 字符串匹配问题

给定若干字符串，若两个字符串在某一位不同（汉明距离为 1），则建边。问题转化为二分图相关问题。

  问题建模思路总结：

  ┌──────────────────┐     ┌──────────────────────┐
  │  最大匹配         │ ←── │  匈牙利算法 / 网络流   │
  └──────┬───────────┘     └──────────────────────┘
         │
    ┌────┴────┐
    │ König    │
    │ 定理     │
    └────┬────┘
         │
  ┌──────┴──────────────────────────────┐
  │                                      │
  ▼                                      ▼
  最小顶点覆盖                          最大独立集
  = 最大匹配数                         = 总顶点数 - 最大匹配数

7. C++ 完整模板

7.1 匈牙利算法模板（DFS 版）

#include <bits/stdc++.h>
using namespace std;

const int N = 510, M = 100010;

struct Hungarian {
    vector<int> g[N];   // g[u]: 左部顶点 u 的邻接表（右部顶点）
    int matchV[N];       // matchV[v]: 右部顶点 v 匹配的左部顶点，-1 表示未匹配
    bool vis[N];         // 本轮 DFS 中已访问的右部顶点
    int n, m;            // n: 左部顶点数, m: 右部顶点数

    void init(int _n, int _m) {
        n = _n; m = _m;
        for (int i = 1; i <= n; i++) g[i].clear();
        memset(matchV, -1, sizeof(matchV));
    }

    void addEdge(int u, int v) {
        g[u].push_back(v);
    }

    // 为左部顶点 u 寻找增广路
    bool dfs(int u) {
        for (int v : g[u]) {
            if (vis[v]) continue;
            vis[v] = true;
            // v 未匹配，或者 v 的原匹配对象能找到新匹配
            if (matchV[v] == -1 || dfs(matchV[v])) {
                matchV[v] = u;
                return true;
            }
        }
        return false;
    }

    // 求最大匹配数
    int maxMatching() {
        int res = 0;
        for (int u = 1; u <= n; u++) {
            memset(vis, false, sizeof(vis));
            if (dfs(u)) res++;
        }
        return res;
    }

    // 获取匹配方案（左部 u 匹配的右部顶点）
    int getMatch(int u) {
        for (int v = 1; v <= m; v++)
            if (matchV[v] == u) return v;
        return -1;
    }
} hung;

7.2 二分图判定模板

struct BipartiteCheck {
    vector<int> g[N];
    int color[N]; // 0=未染色, 1=白色, -1=黑色
    int n;

    void init(int _n) {
        n = _n;
        for (int i = 1; i <= n; i++) g[i].clear();
        memset(color, 0, sizeof(color));
    }

    bool bfs(int start) {
        queue<int> q;
        color[start] = 1;
        q.push(start);
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : g[u]) {
                if (color[v] == 0) {
                    color[v] = -color[u];
                    q.push(v);
                } else if (color[v] == color[u]) {
                    return false;
                }
            }
        }
        return true;
    }

    bool check() {
        for (int i = 1; i <= n; i++)
            if (color[i] == 0 && !bfs(i))
                return false;
        return true;
    }
};

7.3 DAG 最小路径覆盖模板

// 将 DAG 上最小路径覆盖转化为二分图最大匹配
// 建图：对于 DAG 中的边 u→v，连 u(左部) → v(右部)
// 结果：最小路径覆盖 = n - 最大匹配数
int minPathCover(int n, vector<pair<int,int>>& edges) {
    Hungarian h;
    h.init(n, n);
    for (auto& [u, v] : edges)
        h.addEdge(u, v); // u(左部) → v(右部)
    int match = h.maxMatching();
    return n - match;
}
