1. 拓扑排序概述

拓扑排序 (Topological Sort) 是对有向无环图 (DAG, Directed Acyclic Graph) 的所有顶点进行线性排序，使得对于每条有向边 (u, v)，顶点 u 在排序中都出现在 v 之前。

核心性质
  • 拓扑排序 仅适用于 DAG。若图中存在有向环，则不存在拓扑序。
  • 一个 DAG 的拓扑排序结果不一定唯一，取决于选择顶点的顺序。
  • 拓扑排序可以用来判断图中是否有环。

直观理解

可以把拓扑排序想象成"课程安排"问题：某些课程有先修要求（有向边），我们需要找到一种合法的上课顺序，使得每门课的先修课都排在它前面。

  DAG 示例:                 拓扑序之一:    拓扑序之二:

    5 → 2                    5 4 2 3 1 0   4 5 0 2 3 1
    5 → 0
    4 → 0                    验证:          验证:
    4 → 1                    5→2 ✓ 5→0 ✓   4→0 ✓ 4→1 ✓
    2 → 3                    4→0 ✓ 4→1 ✓   5→0 ✓ 5→2 ✓
    3 → 1                    2→3 ✓ 3→1 ✓   2→3 ✓ 3→1 ✓

拓扑排序存在的充要条件

条件说明
图是有向图无向图没有拓扑排序的概念（可视为双向边形成环）
图中无环有向环中每个点都互相可达，无法确定先后顺序
等价于 DAG有向 + 无环 = DAG，拓扑排序等价于 DAG 的线性扩展

2. Kahn 算法 (BFS-based)

Kahn 算法基于贪心思想：每次选择一个入度为 0 的顶点加入拓扑序列，然后删去该顶点及其出边（即将其邻居的入度减 1），重复此过程。

算法步骤

计算所有顶点的入度 indeg[v]。
将所有入度为 0 的顶点加入队列 q。
当队列非空时，取出队首顶点 u，加入拓扑序列。
遍历 u 的所有邻居 v，令 indeg[v]--。若 indeg[v] == 0 则将 v 入队。
重复步骤 3-4 直到队列为空。
若拓扑序列长度等于顶点数，则排序成功；否则图中存在环。
  Kahn 算法过程演示 (入度标注在括号中):

  初始:        5(0) → 2(1)    5入队
               5(0) → 0(2)    4(0) 入队
               4(0) → 0       队列: [5, 4]
               4(0) → 1
               2(1) → 3(1)
               3(1) → 1

  取出5:       序列=[5]       2.indeg: 1→0, 0.indeg: 2→1
                              队列: [4, 2]

  取出4:       序列=[5,4]     0.indeg: 1→0, 1.indeg: 2→1
                              队列: [2, 0]

  取出2:       序列=[5,4,2]   3.indeg: 1→0
                              队列: [0, 3]

  取出0:       序列=[5,4,2,0] 无出边
                              队列: [3]

  取出3:       序列=[5,4,2,0,3] 1.indeg: 1→0
                                队列: [1]

  取出1:       序列=[5,4,2,0,3,1]  完成! |序列|=6=|V|

Kahn 算法模板

// Kahn 算法 — BFS 拓扑排序
// 返回拓扑序列; 若存在环则返回空 vector
vector<int> topo_sort_kahn(int n, const vector<vector<int>>& adj) {
    vector<int> indeg(n, 0);
    for (int u = 0; u < n; u++)
        for (int v : adj[u])
            indeg[v]++;

    queue<int> q;
    for (int i = 0; i < n; i++)
        if (indeg[i] == 0) q.push(i);

    vector<int> topo;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        topo.push_back(u);
        for (int v : adj[u]) {
            if (--indeg[v] == 0)
                q.push(v);
        }
    }
    return topo.size() == n ? topo : vector<int>();
}
字典序最小拓扑序

将 Kahn 算法中的 queue 换成 priority_queue（小根堆），即可在多个入度为 0 的顶点中优先选择编号最小的，从而得到字典序最小的拓扑排序。

3. DFS 实现拓扑排序

DFS 拓扑排序的核心思想：对图进行 DFS，当某个顶点的所有后代都访问完毕后（即 DFS 回溯时），将该顶点加入序列。最终将序列反转即为拓扑序。

原理说明

DFS 的后序遍历保证：若存在边 u → v，则 v 比 u 先完成（先回溯），因此在后序中 v 排在 u 前面。反转后 u 排在 v 前面，满足拓扑序要求。

  DFS 拓扑排序过程:

  图:  5→2→3→1
       5→0
       4→0
       4→1

  DFS(5): → DFS(2): → DFS(3): → DFS(1): 完成
                3完成, 2完成
          → DFS(0): 完成
    5完成

  DFS(4): → DFS(0): 已访问
          → DFS(1): 已访问
    4完成

  后序: [1, 3, 2, 0, 5, 4]
  反转: [4, 5, 0, 2, 3, 1]  ← 拓扑序

DFS 拓扑排序模板

// DFS 拓扑排序
vector<vector<int>> adj;
vector<int> topo;
vector<int> vis;  // 0=未访问, 1=在栈中, 2=已完成

bool dfs_topo(int u) {
    vis[u] = 1;  // 标记为"在递归栈中"
    for (int v : adj[u]) {
        if (vis[v] == 1) return false;  // 存在环
        if (vis[v] == 0 && !dfs_topo(v)) return false;
    }
    vis[u] = 2;  // 标记为"已完成"
    topo.push_back(u);
    return true;
}

bool topo_sort_dfs(int n) {
    vis.assign(n, 0);
    topo.clear();
    for (int i = 0; i < n; i++)
        if (vis[i] == 0 && !dfs_topo(i))
            return false;  // 存在环
    reverse(topo.begin(), topo.end());
    return true;
}
注意

DFS 版本的拓扑排序无法轻易控制字典序。若需要字典序最小的拓扑排序，应使用 Kahn 算法配合优先队列。

Kahn vs DFS 对比

特性Kahn (BFS)DFS
时间复杂度O(V + E)O(V + E)
判环拓扑序列长度 < V遇到"在栈中"的节点
字典序最小用优先队列即可不易实现
层次关系天然按层处理不保证
代码复杂度较简单需三色标记

4. 判环 — 利用拓扑排序检测环

拓扑排序是检测有向图中是否存在环的最直观方法之一。

Kahn 算法判环

若 Kahn 算法结束后，拓扑序列中的顶点数 小于图的总顶点数，则说明剩余顶点的入度始终不为 0，即图中存在环。

DFS 判环

在 DFS 过程中，用三色标记法：

  • 白色 (0)：未访问
  • 灰色 (1)：正在递归栈中（正在访问其后代）
  • 黑色 (2)：已完成

若 DFS 遇到一个灰色顶点，说明存在后向边，即图中有环。

  环检测示例:

  有环: 0→1→2→0     DFS(0): → DFS(1): → DFS(2): → 0(灰色!) → 环!
                  Kahn: 序列长度 0 < 3 → 有环

  无环: 0→1→2→3     DFS: 全部变为黑色, 无灰色冲突
                  Kahn: 序列长度 4 = 4 → 无环
与 DFS 判环的区别

拓扑排序判环适用于需要同时获取拓扑序的场景。若仅需判环，也可以直接使用 DFS 三色标记法，不需要最后反转序列。

5. DAG 最长路 / 最短路 (DP on DAG)

在 DAG 上，利用拓扑排序的顺序进行动态规划，可以在 O(V + E) 时间内求出最短路和最长路，比 Dijkstra 更高效（无需堆）。

核心思想

拓扑排序保证：当处理顶点 u 时，所有能到达 u 的顶点都已经被处理完毕，因此可以直接进行状态转移。

DAG 最长路

// DAG 最长路 — 按拓扑序 DP
// dist[i] 表示从源点 s 到 i 的最长距离
vector<int> dag_longest_path(int n, int s,
    const vector<vector<pair<int,int>>>& adj) {

    vector<int> topo = topo_sort_kahn(n, /*邻接表(无权)*/);
    vector<int> dist(n, -1e9);
    dist[s] = 0;

    for (int u : topo) {
        if (dist[u] == -1e9) continue;  // 不可达
        for (auto& [v, w] : adj[u]) {
            dist[v] = max(dist[v], dist[u] + w);
        }
    }
    return dist;
}

DAG 最短路

// DAG 最短路 — 按拓扑序 DP
vector<int> dag_shortest_path(int n, int s,
    const vector<vector<pair<int,int>>>& adj) {

    vector<int> topo = topo_sort_kahn(n, /*邻接表(无权)*/);
    vector<int> dist(n, 1e9);
    dist[s] = 0;

    for (int u : topo) {
        if (dist[u] == 1e9) continue;
        for (auto& [v, w] : adj[u]) {
            dist[v] = min(dist[v], dist[u] + w);
        }
    }
    return dist;
}
为什么不用 Dijkstra?

DAG 上拓扑排序 DP 的时间复杂度为 O(V + E)，优于 Dijkstra 的 O(E log V)。且可以同时处理最长路（Dijkstra 不支持负权/最长路）。但此方法仅适用于 DAG。

6. 应用场景

6.1 任务调度

给定一组任务及其依赖关系，拓扑排序给出一种合法的执行顺序。例如：编译系统中源文件的编译顺序、构建系统的任务编排。

6.2 课程安排

某些课程有先修要求。拓扑排序可以判断是否能完成所有课程（无环），并给出一种可行的选课顺序。

6.3 关键路径 (Critical Path)

在项目管理中，利用拓扑排序求 DAG 最长路，可以确定项目的最短完成时间和关键活动（关键路径上的边）。

// 关键路径: 拓扑排序 + DAG 最长路
// 1. 正向拓扑序求最早开始时间 es[]
// 2. 逆向拓扑序求最晚开始时间 ls[]
// 3. 若 es[u] == ls[u], 则 u 在关键路径上

6.4 其他应用

  • 表达式求值：编译器中的依赖分析
  • Spreadsheet 计算：单元格公式之间的依赖关系
  • 死锁检测：资源分配图中的环检测
  • 符号链接解析：文件系统中的循环引用检测

7. 完整 C++ 模板

7.1 链式前向星 + Kahn

#include <bits/stdc++.h>
using namespace std;

const int N = 100010, M = 200010;

struct Edge { int to, nxt; } e[M];
int head[N], cnt;
int indeg[N], n, m;

void add(int u, int v) {
    e[++cnt] = {v, head[u]};
    head[u] = cnt;
}

vector<int> topo() {
    queue<int> q;
    for (int i = 1; i <= n; i++)
        if (!indeg[i]) q.push(i);

    vector<int> res;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        res.push_back(u);
        for (int i = head[u]; i; i = e[i].nxt) {
            int v = e[i].to;
            if (--indeg[v] == 0) q.push(v);
        }
    }
    return res;
}

int main() {
    scanf("%d%d", &n, &m);
    for (int i = 0; i < m; i++) {
        int u, v; scanf("%d%d", &u, &v);
        add(u, v); indeg[v]++;
    }
    vector<int> ans = topo();
    if ((int)ans.size() == n)
        for (int x : ans) printf("%d ", x);
    else
        puts("有环");
    return 0;
}

7.2 DAG 最长路完整模板

// DAG 最长路 (带权边)
#include <bits/stdc++.h>
using namespace std;

const int N = 100010;
vector<pair<int,int>> adj[N];  // {to, weight}
int indeg[N], dist[N], n;

void dag_longest(int s) {
    memset(dist, -0x3f, sizeof(dist));
    dist[s] = 0;

    queue<int> q;
    for (int i = 1; i <= n; i++)
        if (!indeg[i]) q.push(i);

    vector<int> topo;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        topo.push_back(u);
        for (auto& [v, w] : adj[u])
            if (--indeg[v] == 0) q.push(v);
    }

    for (int u : topo) {
        if (dist[u] < -1e9) continue;
        for (auto& [v, w] : adj[u])
            dist[v] = max(dist[v], dist[u] + w);
    }
}

7.3 字典序最小拓扑排序

// 字典序最小拓扑排序 (priority_queue)
vector<int> topo_lex_min(int n,
    const vector<vector<int>>& adj) {

    vector<int> indeg(n, 0);
    for (int u = 0; u < n; u++)
        for (int v : adj[u]) indeg[v]++;

    priority_queue<int, vector<int>, greater<int>> pq;
    for (int i = 0; i < n; i++)
        if (!indeg[i]) pq.push(i);

    vector<int> topo;
    while (!pq.empty()) {
        int u = pq.top(); pq.pop();
        topo.push_back(u);
        for (int v : adj[u])
            if (--indeg[v] == 0) pq.push(v);
    }
    return topo.size() == n ? topo : vector<int>();
}
