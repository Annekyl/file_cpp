最短路算法概述

最短路问题是图论中最经典的问题之一：给定一个带权图，求从源点到目标点（或所有点对之间）权值之和最小的路径。根据问题规模和图的特性，有不同的算法可供选择。

问题分类

  • 单源最短路 (SSSP)：从一个源点出发，到所有其他点的最短距离。代表算法：Dijkstra、Bellman-Ford、SPFA。
  • 全源最短路 (APSP)：任意两点之间的最短距离。代表算法：Floyd-Warshall、Johnson。

算法总览对比表

算法 类型 时间复杂度 负权边 负环检测 适用场景
Dijkstra (朴素) SSSP O(V2) 不支持 否 稠密图
Dijkstra (堆优化) SSSP O((V+E) log V) 不支持 否 稀疏图，无负权
Bellman-Ford SSSP O(VE) 支持 是 含负权边
SPFA SSSP 均摊 O(E)，最坏 O(VE) 支持 是 含负权边，稀疏图
Floyd-Warshall APSP O(V3) 支持 是 小规模图 (V ≤ 500)
Johnson APSP O(V·E·log V) 支持 是 稀疏图全源最短路

松弛操作 (Relaxation)

几乎所有最短路算法都基于松弛操作。对于一条边 (u, v, w)，如果经过 u 到达 v 的路径更短，则更新：

// 松弛操作：核心思想
if (dist[u] + w < dist[v]) {
    dist[v] = dist[u] + w;  // 更新最短距离
    pre[v] = u;             // 记录前驱（用于路径还原）
}

Dijkstra 算法

算法思想

Dijkstra 是一种贪心 + 优先队列的单源最短路算法。核心思想：每次从尚未确定最短路的顶点中，选择距离源点最近的顶点，标记为"已确定"，然后用它去松弛相邻的边。

  • 维护一个距离数组 dist[]，dist[i] 表示源点到顶点 i 的当前最短距离估计值。
  • 维护一个集合 S，存放已确定最短路的顶点。
  • 每次从 S 的补集中取出 dist 最小的顶点 u，加入 S，并对 u 的所有出边执行松弛。

正确性证明（直觉）

假设所有边权非负。设 u 是当前未确定顶点中 dist 最小的点，假设存在一条更短的路径 P 到达 u。则 P 上必定存在某个未确定的中间顶点 v，而 dist[v] ≤ dist[u]（因为 v 先于 u 被经过，且边权非负），这与 u 是最小的假设矛盾。因此 dist[u] 就是最短路。

关键性质

Dijkstra 要求所有边权非负。因为贪心策略依赖于"已经确定的顶点不会被更优路径更新"，负权边会破坏这个性质。

为什么不能处理负权边？

考虑一个反例：A→B (w=3)，A→C (w=5)，C→B (w=-4)。Dijkstra 先确定 B（dist=3），但实际最短路是 A→C→B = 5+(-4) = 1。已标记的顶点 B 的距离不会被修正，因此结果错误。

算法流程（堆优化版）

初始化：dist[source] = 0，其余 dist[] = +∞，将 (0, source) 压入优先队列。
从优先队列中弹出距离最小的 (d, u)。若 d > dist[u]，说明已被更优值更新过，跳过。
对 u 的每条出边 (u, v, w)，执行松弛：若 dist[u] + w < dist[v]，更新 dist[v] 并将 (dist[v], v) 压入队列。
重复步骤 2-3 直到队列为空。

复杂度分析

时间: O((V + E) log V)
空间: O(V + E)

每个顶点最多入队一次（不考虑重复），每次出队后遍历其所有出边。优先队列操作为 O(log V)，总计 O((V+E) log V)。使用斐波那契堆可优化到 O(V log V + E)，但实践中意义不大。

C++ 代码模板

#include <bits/stdc++.h>
using namespace std;

typedef pair<int, int> pii;  // (距离, 顶点)

const int INF = 0x3f3f3f3f;
const int MAXN = 100005;

vector<pii> adj[MAXN];   // adj[u] = {(v, w), ...}
int dist[MAXN];
bool vis[MAXN];
int pre[MAXN];            // 前驱数组，用于路径还原

void dijkstra(int src, int n) {
    memset(dist, 0x3f, sizeof(dist));
    memset(vis, 0, sizeof(vis));
    memset(pre, -1, sizeof(pre));

    dist[src] = 0;
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    pq.push({0, src});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (vis[u]) continue;  // 跳过已松弛的旧条目
        vis[u] = true;

        for (auto& [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pre[v] = u;
                pq.push({dist[v], v});
            }
        }
    }
}

// 路径还原：从终点回溯到源点
vector<int> get_path(int src, int dst) {
    vector<int> path;
    for (int v = dst; v != -1; v = pre[v])
        path.push_back(v);
    reverse(path.begin(), path.end());
    return path;
}

int main() {
    int n, m, src;
    scanf("%d%d%d", &n, &m, &src);
    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d%d%d", &u, &v, &w);
        adj[u].push_back({v, w});
        // adj[v].push_back({u, w}); // 无向图取消注释
    }
    dijkstra(src, n);

    for (int i = 1; i <= n; i++)
        printf("%d ", dist[i]);
    return 0;
}
实现技巧
  • 使用 vis[] 数组标记已确定顶点，而不是在队列中删除旧条目（懒删除）。
  • 距离初始化用 0x3f3f3f3f 而非 INT_MAX，因为前者相加不会溢出。
  • 路径还原需要维护 pre[] 前驱数组。

Bellman-Ford 算法

算法思想

Bellman-Ford 是一种能处理负权边的单源最短路算法。其核心思想是：对图中所有边执行 V-1 轮松弛操作。经过 k 轮松弛后，可以确定从源点经过最多 k 条边到达各点的最短距离。

算法流程

初始化：dist[source] = 0，其余 dist[] = +∞。
执行 V-1 轮循环，每轮遍历所有边 (u, v, w)，对每条边执行松弛操作。
负环检测：再执行第 V 轮松弛，若仍有更新，说明存在负权环。
正确性

不含负环的图中，最短路最多经过 V-1 条边。第 i 轮松弛保证了经过最多 i 条边的最短路已确定。因此 V-1 轮后所有最短路都被正确计算。

执行过程追踪

同样以示例图为例，源点 1：

第 1 轮：遍历所有边。1→2(w=1)：dist[2]=1；1→5(w=10)：dist[5]=10；2→4(w=2)：dist[4]=3；2→3(w=3)：dist[3]=4；3→4(w=1)：3→5(w=2)：dist[5]=6。
第 2 轮：无更新。
第 3 轮：无更新。
第 4 轮：无更新。V-1=4 轮完成，无负环。

复杂度分析

时间: O(V · E)
空间: O(V + E)

C++ 代码模板

struct Edge {
    int u, v, w;
};

vector<Edge> edges;
int dist[MAXN];

// 返回 true 表示存在负环
bool bellman_ford(int src, int n) {
    memset(dist, 0x3f, sizeof(dist));
    dist[src] = 0;

    for (int i = 1; i < n; i++) { // V-1 轮
        bool updated = false;
        for (auto& e : edges) {
            if (dist[e.u] != INF && dist[e.u] + e.w < dist[e.v]) {
                dist[e.v] = dist[e.u] + e.w;
                updated = true;
            }
        }
        if (!updated) break; // 提前终止优化
    }

    // 第 V 轮：检测负环
    for (auto& e : edges) {
        if (dist[e.u] != INF && dist[e.u] + e.w < dist[e.v]) {
            return true; // 存在负环
        }
    }
    return false;
}

SPFA 算法

算法思想

SPFA (Shortest Path Faster Algorithm) 是 Bellman-Ford 的队列优化版本。核心观察：在 Bellman-Ford 中，只有上一轮 dist 发生变化的顶点，其出边才可能产生新的松弛。因此，用一个队列维护"需要松弛的顶点"，避免无效遍历。

算法流程

初始化：dist[source] = 0，将 source 加入队列，标记 in_queue[source] = true。
取出队首顶点 u，标记 in_queue[u] = false。
遍历 u 的所有出边 (u, v, w)。若 dist[u] + w < dist[v]，更新 dist[v]。若 v 不在队列中，将 v 入队。
重复步骤 2-3 直到队列为空。
负环检测：记录每个顶点的入队次数，若某个顶点入队超过 V 次，说明存在负环。

与 Bellman-Ford 的关系

SPFA 本质上等价于 Bellman-Ford，只是用队列优化了松弛顺序。在最坏情况下（如网格图），SPFA 的时间复杂度退化为 O(VE)，与 Bellman-Ford 相同。

注意：SPFA 已被卡

在 ACM 竞赛中，出题人可以通过构造特殊数据（如网格图、菊花图等）使 SPFA 退化到 O(VE)。对于无负权边的问题，应优先使用 Dijkstra。SPFA 在最坏情况下会被 TLE。

SPFA vs Dijkstra 选择指南

条件 推荐算法 原因
无负权边 Dijkstra 稳定 O((V+E)logV)，不会被卡
有负权边，无负环 SPFA 平均 O(E)，比 Bellman-Ford 快
需要检测负环 SPFA / Bellman-Ford Dijkstra 无法处理
差分约束系统 SPFA 可将约束转化为带负权的最短路

执行过程追踪

以示例图为例（假设存在负权边 2→3(w=-1)）：

初始化：dist[1]=0，其余 ∞。队列：[1]
弹出 1：松弛 1→2：dist[2]=1，入队 2；松弛 1→5：dist[5]=10，入队 5。队列：[2, 5]
弹出 2：松弛 2→4：dist[4]=3，入队 4；松弛 2→3(w=-1)：dist[3]=0，入队 3。队列：[5, 4, 3]
弹出 5：松弛 5→1：不更新。队列：[4, 3]
弹出 4：无出边。队列：[3]
弹出 3：松弛 3→4：不更新；松弛 3→5：dist[5]=2，入队 5。队列：[5]
弹出 5：松弛 5→1：不更新。队列为空，结束。

复杂度分析

平均时间: O(E)
最坏时间: O(V · E)
空间: O(V + E)

C++ 代码模板

vector<pii> adj[MAXN]; // (v, w)
int dist[MAXN];
bool in_queue[MAXN];
int cnt[MAXN];  // 入队次数，用于判负环

// 返回 true 表示存在负环
bool spfa(int src, int n) {
    memset(dist, 0x3f, sizeof(dist));
    memset(in_queue, false, sizeof(in_queue));
    memset(cnt, 0, sizeof(cnt));

    queue<int> q;
    dist[src] = 0;
    q.push(src);
    in_queue[src] = true;
    cnt[src] = 1;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        in_queue[u] = false;

        for (auto& [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (!in_queue[v]) {
                    q.push(v);
                    in_queue[v] = true;
                    cnt[v]++;
                    if (cnt[v] > n) return true; // 负环
                }
            }
        }
    }
    return false;
}

// SLF 优化（Small Label First）：加速常数
// 如果 dist[new_node] < dist[q.front()]，插入队首而非队尾
// 使用 deque 替代 queue 实现
SPFA 优化技巧
  • SLF 优化：如果新节点的 dist 小于队首节点的 dist，插入队首（用 deque）。
  • LLL 优化：队列中 dist 低于平均值的节点优先处理。
  • 栈代替队列：某些场景下用栈代替队列效果更好（DFS 序）。

Floyd-Warshall 算法

算法思想

Floyd-Warshall 是一种求解全源最短路（所有顶点对之间的最短距离）的动态规划算法。核心思想：设 dp[k][i][j] 表示只经过前 k 个顶点作为中间节点时，从 i 到 j 的最短距离。

状态转移方程：

dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])

即：从 i 到 j 经过前 k 个节点的最短路，要么不经过 k（等于经过前 k-1 个节点的结果），要么经过 k（等于 i→k 的最短路 + k→j 的最短路）。

由于 dp[k] 只依赖 dp[k-1]，可以用滚动数组优化为二维 dp[i][j]，就得到了经典的三重循环写法。

算法流程

初始化 dp[i][j]：若 i==j 则 0，若有边 i→j 则为边权，否则 +∞。
三重循环：最外层枚举中间节点 k，内两层枚举起点 i 和终点 j。
执行松弛：dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j])。
循环结束后，dp[i][j] 即为 i 到 j 的最短距离。
循环顺序至关重要

k 必须在最外层！如果把 i 或 j 放在最外层，dp 值可能被错误地用还未初始化的状态更新，导致结果错误。

负环检测

Floyd 执行完毕后，检查对角线 dp[i][i]。若 dp[i][i] < 0，说明顶点 i 在某个负环上。

路径还原

维护一个 nxt[i][j] 数组，记录 i 到 j 最短路上 i 的下一个顶点。

// 初始化
nxt[i][j] = j;  // 若有边 i→j

// Floyd 过程中更新
if (dp[i][k] + dp[k][j] < dp[i][j]) {
    dp[i][j] = dp[i][k] + dp[k][j];
    nxt[i][j] = nxt[i][k];  // 经过 k，i 的下一跳与 i→k 相同
}

// 还原路径
vector<int> get_path(int i, int j) {
    vector<int> path = {i};
    while (i != j) {
        i = nxt[i][j];
        path.push_back(i);
    }
    return path;
}

初始距离矩阵：

复杂度分析

时间: O(V3)
空间: O(V2)

三重循环，每层 V 次。适合 V ≤ 500 左右的规模。V=1000 时约 109 操作，通常会 TLE。

C++ 代码模板

const int INF = 0x3f3f3f3f;
int dp[MAXN][MAXN];  // dp[i][j] = i到j的最短距离
int nxt[MAXN][MAXN]; // 路径还原

void floyd(int n) {
    // 初始化已在读入时完成
    for (int k = 1; k <= n; k++)
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                if (dp[i][k] + dp[k][j] < dp[i][j]) {
                    dp[i][j] = dp[i][k] + dp[k][j];
                    nxt[i][j] = nxt[i][k];
                }
}

// 检测负环
bool has_negative_cycle(int n) {
    for (int i = 1; i <= n; i++)
        if (dp[i][i] < 0) return true;
    return false;
}

int main() {
    int n, m;
    scanf("%d%d", &n, &m);

    // 初始化
    memset(dp, 0x3f, sizeof(dp));
    for (int i = 1; i <= n; i++) {
        dp[i][i] = 0;
        nxt[i][i] = i;
    }

    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d%d%d", &u, &v, &w);
        dp[u][v] = min(dp[u][v], w); // 重边取最小
        nxt[u][v] = v;
    }

    floyd(n);

    // 输出所有点对最短路
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= n; j++)
            printf("%d%c", dp[i][j] == INF ? -1 : dp[i][j],
                   j == n ? '\n' : ' ');
    return 0;
}
Floyd 的其他用途
  • 传递闭包：将边权设为 1（或用 bool），可求任意两点是否连通。
  • 最小环：在 Floyd 过程中，dp[i][i] + dp[i][k] + dp[k][i] 可以求最小环。
  • 字典序最小路径：配合 nxt 数组还原路径。

Johnson 算法

问题背景

对于含负权边的图，要求全源最短路：

  • Floyd 的 O(V3) 对大规模稀疏图太慢。
  • 不能直接对每个顶点跑 Dijkstra（负权边）。
  • 对每个顶点跑 SPFA 最坏 O(V2E)，也不够快。

Johnson 算法通过势函数重赋权消除负权边，然后对每个顶点跑 Dijkstra。

算法原理

势函数 (Potential Function)

引入一个虚拟源点 s，连接到所有顶点（边权为 0）。从 s 跑 Bellman-Ford（或 SPFA）得到 h[v] = s 到 v 的最短距离。

定义新边权：w'(u,v) = w(u,v) + h[u] - h[v]

关键性质
  • 非负性：对于原图中的每条边 (u,v,w)，由三角不等式 h[v] ≤ h[u] + w，得到 w' = w + h[u] - h[v] ≥ 0。所有新边权非负！
  • 最短路等价：新图中 u 到 v 的最短路与原图中仅相差一个常数 h[v]-h[u]。即 dist'[u][v] = dist[u][v] + h[v] - h[u]。

算法流程

添加虚拟源点 0，向所有顶点连边权为 0 的边。
从 0 跑 Bellman-Ford/SPFA 得到势函数 h[]。若存在负环则报告。
对每条边 (u, v, w)，重新赋权为 w' = w + h[u] - h[v]。
对每个顶点 1~V 跑 Dijkstra（新图无负权边）。
将结果还原：dist[i][j] = dist'[i][j] - h[j] + h[i]。

复杂度分析

时间: O(V·E·log V)
空间: O(V + E)

Bellman-Ford/SPFA 阶段：O(VE)。Dijkstra 阶段：V 次 Dijkstra = O(V · (V+E) log V)。对于稀疏图 (E ~ V)，总体约 O(V2 log V)。

Johnson vs Floyd
  • 稀疏图（E ~ V）：Johnson O(V2 log V) 远优于 Floyd O(V3)。
  • 稠密图（E ~ V2）：Johnson O(V3 log V) 反而比 Floyd 慢。
  • 实际竞赛中，V ≤ 500 一般用 Floyd；V ≤ 3000 且为稀疏图用 Johnson。

C++ 代码模板

const int INF = 0x3f3f3f3f;
const int MAXN = 3005;

struct Edge {
    int v, w;
};

vector<Edge> adj[MAXN]; // 原图
vector<Edge> adj_new[MAXN]; // 重赋权后的图
int h[MAXN];  // 势函数
int dist_all[MAXN][MAXN]; // 全源最短路结果

// SPFA 求势函数
bool compute_potential(int n) {
    memset(h, 0x3f, sizeof(h));
    bool inq[MAXN] = {};
    int cnt[MAXN] = {};
    queue<int> q;

    // 虚拟源点 0 到所有点距离为 0
    h[0] = 0;
    q.push(0);
    inq[0] = true;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        inq[u] = false;

        for (auto& [v, w] : adj[u]) {
            if (h[u] + w < h[v]) {
                h[v] = h[u] + w;
                if (!inq[v]) {
                    q.push(v);
                    inq[v] = true;
                    cnt[v]++;
                    if (cnt[v] > n) return false; // 负环
                }
            }
        }
    }
    return true;
}

// Dijkstra（堆优化）
void dijkstra(int src, int n) {
    int& out = dist_all[src]; // 输出到全局数组
    memset(out, 0x3f, (n + 1) * sizeof(int));
    out[src] = 0;
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    pq.push({0, src});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();
        if (d > out[u]) continue;
        for (auto& [v, w] : adj_new[u]) {
            if (out[u] + w < out[v]) {
                out[v] = out[u] + w;
                pq.push({out[v], v});
            }
        }
    }
}

void johnson(int n) {
    // 1. 添加虚拟源点
    for (int i = 1; i <= n; i++)
        adj[0].push_back({i, 0});

    // 2. 计算势函数
    if (!compute_potential(n)) {
        printf("Negative cycle detected!\n");
        return;
    }

    // 3. 重赋权
    for (int u = 1; u <= n; u++)
        for (auto& [v, w] : adj[u])
            adj_new[u].push_back({v, w + h[u] - h[v]});

    // 4. 对每个顶点跑 Dijkstra
    for (int i = 1; i <= n; i++)
        dijkstra(i, n);

    // 5. 还原真实距离
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= n; j++)
            if (dist_all[i][j] != INF)
                dist_all[i][j] += h[j] - h[i];
}

DAG 最短路（附赠）

对于有向无环图 (DAG)，可以先拓扑排序，然后按拓扑序依次松弛。时间复杂度 O(V+E)，是最优解法。

vector<pii> adj[MAXN]; // (v, w)
int indeg[MAXN];
int dist[MAXN];

void dag_shortest(int src, int n) {
    memset(dist, 0x3f, sizeof(dist));
    dist[src] = 0;

    queue<int> q;
    for (int i = 1; i <= n; i++)
        if (indeg[i] == 0) q.push(i);

    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (auto& [v, w] : adj[u]) {
            if (dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
            if (--indeg[v] == 0)
                q.push(v);
        }
    }
}

常见陷阱与注意事项

竞赛中的常见错误
  • 多测不清空：多组测试数据时忘记清空邻接表、距离数组等。建议每次 memset 或使用 vector.clear()。
  • 重边处理：邻接表自然支持重边；邻接矩阵需取 min。
  • 自环：正权自环无影响，负权自环构成负环。
  • 溢出：边权和可能超过 INT_MAX，注意使用 long long。
  • INF 设置：0x3f3f3f3f (约 109) 可以相加不溢出。若用 long long 则用 0x3f3f3f3f3f3f3f3f。
  • SPFA 被卡：无负权边时坚决用 Dijkstra。

进阶主题

  • 0-1 BFS：边权只有 0 和 1 时，用双端队列 BFS 实现 O(V+E)。
  • 分层图最短路：将图复制 k 层，层间连特定边，用于"最多经过 k 次免费边"等问题。
  • 最短路计数：统计最短路条数，在松弛时额外维护 cnt 数组。
  • 次短路：维护 dist1 和 dist2 两组距离，松弛时同时更新。
  • 差分约束：将不等式约束转化为图论模型，用 SPFA 求解。
