1. MST 概述与基本性质

1.1 什么是最小生成树

给定一个连通无向加权图 G = (V, E)，其最小生成树 (Minimum Spanning Tree) 是一棵包含图中所有顶点的树，且树中所有边的权值之和最小。

  • MST 必须恰好包含 |V| - 1 条边
  • 如果图中存在相同权值的边，MST 可能不唯一，但总权值唯一
  • 若图不连通，则可以求最小生成森林 (Minimum Spanning Forest)
  示例图:                     MST (总权值 = 16):

  (A)---4---(B)              (A)---4---(B)
   | \       /|               |           |
   8   3   5  2               8           2
   |     \ /  |               |           |
  (D)---9---(C)              (D)       (C)
                                       /
                                      1
                                     /
                                   (E)

  边集:                       选中边:
  A-B: 4                      A-B: 4  (保留)
  A-D: 8                      A-D: 8  (保留)
  A-C: 3                      B-C: 2  (保留)
  B-C: 2                      C-E: 1  (保留)
  B-D: 5                      未选:
  C-D: 9                      A-C: 3  (会形成环)
  C-E: 1                      B-D: 5  (会形成环)
                              C-D: 9  (权值过大)

1.2 切割性质 (Cut Property)

切割性质

对于图 G 的任意一个切割 (S, V-S)，如果边 (u, v) 是横跨该切割的所有边中权值最小的边（可以是之一），那么 (u, v) 必定存在于某棵 MST 中。

直觉理解：把顶点分成两组，连接两组的最短边一定可以被安全地选入 MST。因为如果不选它，就不得不选更长的边来连接这两个部分。

    切割 {A, D} | {B, C, E}

    {A, D}          {B, C, E}
     A ---3---*--- B
     |             /|
     8      *5*   2 |      * = 横跨切割的边
     |       /     \|
     D ---9---*--- C
                  /
                 1
                  \
                   E

    横跨切割的边: A-C(3), B-D(5), A-B(4), D-C(9)
    最小边: A-C(3) --> 一定在某棵 MST 中

1.3 环性质 (Cycle Property)

环性质

对于图中的任意一个环 (cycle)，如果环中某条边的权值严格大于环中其他所有边的权值，那么这条边一定不在任何 MST 中。

直觉理解：环中最贵的边永远不可能出现在 MST 里，因为删掉它不会使图不连通，反而能减少总权值。

1.4 MST 存在性

  • 充分必要条件：连通无向图一定存在 MST
  • 若图不连通，则存在最小生成森林（每棵连通分量各一棵 MST）
  • 所有 MST 的总权值相同
  • 如果所有权值互不相同，则 MST 唯一

2. Kruskal 算法

2.1 算法思想

Kruskal 算法是一种全局贪心策略：将所有边按权值从小到大排序，依次考虑每条边，如果这条边连接的两个顶点还不在同一个连通分量中（即加入后不会形成环），就将这条边加入 MST。

2.2 算法流程

将所有边按权值从小到大排序
初始化并查集，每个顶点各自为一个集合
按排序顺序依次遍历每条边 (u, v, w)
若 find(u) != find(v)，则选中该边，执行 union(u, v)
若已选中 V - 1 条边，算法结束；否则继续

2.4 并查集 (DSU) 回顾

Kruskal 算法的核心依赖是并查集，用于高效判断两个顶点是否在同一连通分量。

并查集优化

路径压缩 + 按秩合并同时使用时，单次操作的均摊时间复杂度为 O(alpha(n))，其中 alpha 是反阿克曼函数，实际可视为常数。

2.5 复杂度分析

时间: O(E log E)
空间: O(V + E)
  • 排序边集: O(E log E)
  • 并查集操作: O(E * alpha(V))，近似 O(E)
  • 总时间: O(E log E)，瓶颈在排序
  • 注意: O(E log E) = O(E log V)，因为 E ≤ V2
适用场景

Kruskal 算法特别适合稀疏图（边数较少），因为它的时间复杂度主要取决于边数。对于稠密图，Prim 算法可能更优。

3. Prim 算法

3.1 算法思想

Prim 算法是一种局部扩展策略：从某个起始顶点开始，维护一个已选顶点集合 S，每次从连接 S 和 V-S 的边中选择权值最小的边，将其对应的顶点加入 S。这个过程和 Dijkstra 算法非常相似。

3.2 优先队列版本 (稀疏图优化)

使用最小堆维护从已选集合到未选顶点的最小边权。

选择任意起始顶点 s，将其 dist 设为 0，其余为 INF
将 (0, s) 加入优先队列
从优先队列中取出权值最小的顶点 u
若 u 已在 MST 中，跳过；否则将 u 加入 MST
遍历 u 的所有邻接边 (u, v, w)，若 v 不在 MST 中且 w < dist[v]，更新 dist[v] 并入队
重复直到所有顶点都被加入 MST 或队列为空

MST 总权值 = 0 + 3 + 2 + 5 + 8 + 7 = 25

3.4 邻接矩阵版本 (稠密图优化)

对于稠密图 (E 接近 V2)，不使用优先队列，而是每次线性扫描找最小 dist，时间复杂度为 O(V2)，比优先队列版本更优。

选择起始顶点，dist[] 初始化
循环 V 次：找到 dist 最小且未加入 MST 的顶点 u
将 u 加入 MST，用 u 的邻接边更新 dist[]

3.5 Prim vs Dijkstra

Prim 与 Dijkstra 的区别

两者的代码结构几乎相同，核心区别在于更新方式：

  • Dijkstra：dist[v] = min(dist[v], dist[u] + w)，累加路径权值
  • Prim：dist[v] = min(dist[v], w)，直接取边权（因为 dist[v] 表示的是从 MST 到 v 的最短边，而非路径长度）

3.6 复杂度分析

实现方式 时间复杂度 适用场景
优先队列 (邻接表) O(E log V) 稀疏图
邻接矩阵 (线性扫描) O(V2) 稠密图
Fibonacci 堆 (理论) O(E + V log V) 理论最优，实际不常用

4. Boruvka 算法

4.1 算法思想

Boruvka 算法是最古老的 MST 算法（1926年），其核心思想是多路扩展：在每一轮中，为每个连通分量找到连接到其他分量的最小权边，然后将所有这些边同时加入 MST。每轮过后连通分量数量至少减半。

4.2 算法流程

初始化：每个顶点为一个独立的连通分量
对每个连通分量，遍历其所有边，找到连接到其他分量的最小权边
将找到的所有最小边加入 MST（合并连通分量）
若连通分量数大于 1，回到步骤 2

4.3 复杂度

时间: O(E log V)

每轮连通分量至少减半，最多 log V 轮，每轮遍历所有边，因此总时间为 O(E log V)。

适用场景

Boruvka 算法在并行计算中有天然优势（每轮各分量独立计算），也适合处理边以分布式方式给出的场景。在竞赛中较少使用，但了解其思想有助于理解 MST 的本质。

6. MST 进阶应用

6.1 次小生成树 (Second MST)

次小生成树是指权值总和严格大于 MST 的生成树中权值最小的那个。求法如下：

先求出一棵 MST，记其边集为 T
枚举不在 T 中的每条边 e = (u, v, w)
将 e 加入 T 会形成一个环，在这个环中找到 T 里权值最大的边 e_max
用 e 替换 e_max 得到一棵新生成树，其权值为 MST + w - w_max
在所有替换方案中取权值最小的，即为次小生成树
如何快速找到环中最大边？

在 MST 上预处理 倍增数组，记录树上两点路径间权值最大的边。对于每条非树边 (u, v, w)，用 LCA + 倍增在 O(log V) 时间内找到 u-v 路径上的最大边权。

6.2 最大生成树

只需将边按权值从大到小排序，其余步骤与最小生成树完全相同。或者将所有边权取反后求 MST。

6.3 MST 验证

给定一棵生成树 T，如何验证它是否为 MST？

对 T 中的每条边 e，检查它是否满足切割性质
等价地：对于每条非树边 (u, v, w)，检查 u-v 在 T 上的路径中所有边权是否都不超过 w
若所有非树边都满足该条件，则 T 是 MST

6.4 最小瓶颈路

最小瓶颈生成树是使得最大边权最小的生成树。有趣的是，MST 本身就是最小瓶颈生成树。因此，MST 上两点间路径的最大边权即为这两点间所有路径中瓶颈值最小的。

6.5 Kruskal 重构树

在 Kruskal 算法执行过程中，每合并两个连通分量时创建一个新节点，权值为合并边的权值。形成的树称为 Kruskal 重构树，具有如下性质：

  • 是一棵有根二叉树（大根堆性质）
  • 原图两点间路径的最大边权 = 重构树上两点 LCA 的权值
  • 可用于回答 "两点间所有路径的最大边权的最小值" 类问题

8. 完整代码模板

8.1 Kruskal + 并查集

#include <bits/stdc++.h>
using namespace std;

// ===== 并查集 =====
struct DSU {
    vector<int> fa, rk;
    DSU(int n) : fa(n + 1), rk(n + 1, 0) {
        iota(fa.begin(), fa.end(), 0);
    }
    int find(int x) {
        return fa[x] == x ? x : fa[x] = find(fa[x]);
    }
    bool merge(int x, int y) {
        x = find(x), y = find(y);
        if (x == y) return false;
        if (rk[x] < rk[y]) swap(x, y);
        fa[y] = x;
        if (rk[x] == rk[y]) rk[x]++;
        return true;
    }
};

// ===== 边结构体 =====
struct Edge {
    int u, v, w;
    bool operator<(const Edge& o) const {
        return w < o.w;
    }
};

// ===== Kruskal 算法 =====
// 返回 MST 总权值，若图不连通返回 -1
long long kruskal(int n, vector<Edge>& edges) {
    sort(edges.begin(), edges.end());
    DSU dsu(n);
    long long total = 0;
    int cnt = 0;  // 已选边数

    for (auto& e : edges) {
        if (dsu.merge(e.u, e.v)) {
            total += e.w;
            cnt++;
            if (cnt == n - 1) break;
        }
    }

    return cnt == n - 1 ? total : -1;
}

int main() {
    int n, m;
    scanf("%d %d", &n, &m);
    vector<Edge> edges(m);
    for (int i = 0; i < m; i++) {
        scanf("%d %d %d", &edges[i].u, &edges[i].v, &edges[i].w);
    }
    long long ans = kruskal(n, edges);
    if (ans == -1) puts("orz");
    else printf("%lld\n", ans);
    return 0;
}

8.2 Prim + 优先队列 (邻接表)

#include <bits/stdc++.h>
using namespace std;

typedef pair<int,int> pii;

// ===== Prim 算法 (优先队列版) =====
// graph[u] = {(v, w), ...}
// 返回 MST 总权值，若图不连通返回 -1
long long prim(int n, vector<vector<pii>>& graph) {
    vector<bool> vis(n + 1, false);
    vector<int> dist(n + 1, INT_MAX);
    priority_queue<pii, vector<pii>, greater<pii>> pq;

    dist[1] = 0;
    pq.push({0, 1});

    long long total = 0;
    int cnt = 0;  // 已加入 MST 的顶点数

    while (!pq.empty()) {
        auto [w, u] = pq.top(); pq.pop();
        if (vis[u]) continue;
        vis[u] = true;
        total += w;
        cnt++;

        for (auto& [v, wt] : graph[u]) {
            if (!vis[v] && wt < dist[v]) {
                dist[v] = wt;
                pq.push({wt, v});
            }
        }
    }

    return cnt == n ? total : -1;
}

int main() {
    int n, m;
    scanf("%d %d", &n, &m);
    vector<vector<pii>> graph(n + 1);
    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d %d %d", &u, &v, &w);
        graph[u].push_back({v, w});
        graph[v].push_back({u, w});
    }
    long long ans = prim(n, graph);
    if (ans == -1) puts("orz");
    else printf("%lld\n", ans);
    return 0;
}

8.3 Prim 邻接矩阵版本 (稠密图)

#include <bits/stdc++.h>
using namespace std;

const int INF = 0x3f3f3f3f;
int g[505][505];
int dist[505];
bool vis[505];

long long prim(int n) {
    memset(dist, 0x3f, sizeof(dist));
    memset(vis, 0, sizeof(vis));
    dist[1] = 0;
    long long total = 0;

    for (int i = 1; i <= n; i++) {
        // 找 dist 最小的未访问顶点
        int u = -1;
        for (int j = 1; j <= n; j++)
            if (!vis[j] && (u == -1 || dist[j] < dist[u]))
                u = j;

        if (dist[u] == INF) return -1;  // 不连通
        vis[u] = true;
        total += dist[u];

        // 用 u 更新邻接顶点
        for (int v = 1; v <= n; v++)
            if (!vis[v] && g[u][v] < dist[v])
                dist[v] = g[u][v];
    }
    return total;
}

int main() {
    int n, m;
    scanf("%d %d", &n, &m);
    memset(g, 0x3f, sizeof(g));
    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d %d %d", &u, &v, &w);
        g[u][v] = g[v][u] = min(g[u][v], w);  // 处理重边
    }
    long long ans = prim(n);
    if (ans == -1) puts("orz");
    else printf("%lld\n", ans);
    return 0;
}

8.4 次小生成树 (倍增法)

#include <bits/stdc++.h>
using namespace std;

typedef long long ll;
typedef pair<int,int> pii;

const int N = 100005, LOG = 17;

struct Edge { int u, v, w; bool used; };

// 并查集
int fa[N];
int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }

// 树上倍增
vector<pii> tree[N];  // tree[u] = {(v, w), ...}
int dep[N], fa_node[N][LOG + 1];
int mx[N][LOG + 1];    // 路径最大边权
int mx2[N][LOG + 1];   // 路径次大边权

void dfs(int u, int p, int d) {
    dep[u] = d;
    fa_node[u][0] = p;
    for (int k = 1; k <= LOG; k++) {
        fa_node[u][k] = fa_node[fa_node[u][k-1]][k-1];
        // 合并两段路径的最大和次大边权
        int a = mx[u][k-1], b = mx[fa_node[u][k-1]][k-1];
        int a2 = mx2[u][k-1], b2 = mx2[fa_node[u][k-1]][k-1];
        if (a > b) {
            mx[u][k] = a;
            mx2[u][k] = max(a2, b);
        } else if (a < b) {
            mx[u][k] = b;
            mx2[u][k] = max(a, b2);
        } else {
            mx[u][k] = a;
            mx2[u][k] = max(a2, b2);
        }
    }
    for (auto& [v, w] : tree[u]) {
        if (v == p) continue;
        mx[v][0] = w;
        mx2[v][0] = -1;
        dfs(v, u, d + 1);
    }
}

// 查询 u-v 路径上的最大边权 (严格小于 val 时可替换)
pii query(int u, int v) {
    int ret1 = 0, ret2 = 0;  // 最大和次大
    if (dep[u] < dep[v]) swap(u, v);
    for (int k = LOG; k >= 0; k--)
        if (dep[fa_node[u][k]] >= dep[v]) {
            // 合并到答案中
            if (mx[u][k] > ret1) { ret2 = ret1; ret1 = mx[u][k]; }
            else if (mx[u][k] < ret1) ret2 = max(ret2, mx[u][k]);
            ret2 = max(ret2, mx2[u][k]);
            u = fa_node[u][k];
        }
    if (u == v) return {ret1, ret2};
    for (int k = LOG; k >= 0; k--)
        if (fa_node[u][k] != fa_node[v][k]) {
            // 合并 u 和 v 两段
            int vals[] = {ret1, ret2, mx[u][k], mx2[u][k], mx[v][k], mx2[v][k]};
            ret1 = ret2 = 0;
            for (int x : vals) {
                if (x > ret1) { ret2 = ret1; ret1 = x; }
                else if (x < ret1) ret2 = max(ret2, x);
            }
            u = fa_node[u][k]; v = fa_node[v][k];
        }
    // 最后一步到 LCA
    int vals[] = {ret1, ret2, mx[u][0], mx2[u][0], mx[v][0], mx2[v][0]};
    ret1 = ret2 = 0;
    for (int x : vals) {
        if (x > ret1) { ret2 = ret1; ret1 = x; }
        else if (x < ret1) ret2 = max(ret2, x);
    }
    return {ret1, ret2};
}

int main() {
    int n, m;
    scanf("%d %d", &n, &m);
    vector<Edge> edges(m);
    for (int i = 0; i < m; i++)
        scanf("%d %d %d", &edges[i].u, &edges[i].v, &edges[i].w);

    // Kruskal 求 MST
    sort(edges.begin(), edges.end(),
         [](const Edge& a, const Edge& b) { return a.w < b.w; });
    for (int i = 1; i <= n; i++) fa[i] = i;
    ll mst_val = 0;
    for (auto& e : edges) {
        int fu = find(e.u), fv = find(e.v);
        if (fu != fv) {
            fa[fu] = fv;
            mst_val += e.w;
            e.used = true;
            tree[e.u].push_back({e.v, e.w});
            tree[e.v].push_back({e.u, e.w});
        }
    }

    // 倍增预处理
    dfs(1, 0, 1);

    // 枚举非树边求次小生成树
    ll ans = 1e18;
    for (auto& e : edges) {
        if (e.used) continue;
        auto [mx1, mx2v] = query(e.u, e.v);
        // 严格次小: 替换严格小于 e.w 的最大边
        if (e.w > mx1)
            ans = min(ans, mst_val + e.w - mx1);
        else if (e.w > mx2v)
            ans = min(ans, mst_val + e.w - mx2v);
    }
    printf("%lld\n", ans);
    return 0;
}

8.5 常见变体技巧

问题类型 转化方法
部分边已确定 已确定的边权值设为 0，或预先 union
最大生成树 边权取反，或排序改为从大到小
第 k 小/大生成树 需要搜索/枚举，通常结合状压或搜索剪枝
虚拟源点建站问题 新建虚拟源点 0，向各点连边表示建站成本
平面图 MST 边数 O(V)，直接 Kruskal 即可
曼哈顿距离 MST 只保留 O(V) 条候选边，再 Kruskal

总结

MST 核心要点
  • MST 的理论基础是切割性质和环性质
  • Kruskal = 排序 + 并查集，适合稀疏图，代码简洁
  • Prim (PQ) = 优先队列 + 贪心扩展，适合稀疏图
  • Prim (矩阵) = 线性扫描，适合稠密图
  • 竞赛中 Kruskal 使用频率最高，建议优先掌握
  • 次小生成树 = MST + 倍增 + 枚举非树边替换
  • Kruskal 重构树是处理 "瓶颈路径" 类问题的利器
