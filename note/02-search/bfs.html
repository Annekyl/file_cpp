1. BFS 基础 — 队列与层序遍历

1.1 核心思想

BFS（广度优先搜索）是一种从起点出发，逐层扩展的搜索策略。 它使用队列（queue）来维护待访问的节点：每次从队首取出一个节点， 将其所有未访问的邻居加入队尾，如此反复直到队列为空。

BFS 的关键性质
  • BFS 天然按照"距离起点的步数"分层访问节点
  • 第一次到达某个节点时，一定是经过最少步数的路径
  • BFS 适用于无权图或边权相同的图的最短路问题

1.2 BFS 执行流程图解

以一个 4×4 网格迷宫为例，起点为 S，终点为 E，# 为障碍物，. 为可通行区域：

初始网格:                    第1层扩展:                  第2层扩展:
  S . # .                      S1. # .                      S1. # .
  . # . .                      1# . .                      22# . .
  . . . #                      . . . #                      3. . . #
  . . . E                      . . . E                      4. . . E

队列: [(0,0,0)]              队列: [(1,0,1),(0,1,1)]     队列: [(2,0,2),(1,2,2),(3,0,2),(0,2,2)]
dist[0][0] = 0               dist 更新到 1               dist 更新到 2

第3层扩展:                    第4层扩展:                   最终到达 E:
  S1. # .                      S1. # .                      S1. # .
  22# . .                      22# . .                      22# . .
  33. . #                      333. #                      333. #
  4. . . E                      44. . E                      444. E*
dist 更新到 3               dist 更新到 4               最短距离 = 4

1.3 BFS 基本代码模板

// 通用 BFS 模板（网格/图均适用）
#include <bits/stdc++.h>
using namespace std;

const int dx[] = {0, 0, 1, -1};
const int dy[] = {1, -1, 0, 0};
int n, m;
char grid[1005][1005];      // 网格地图
int dist[1005][1005];        // 距离数组，-1 表示未访问

int bfs(int sx, int sy, int ex, int ey) {
    memset(dist, -1, sizeof(dist));
    queue<pair<int,int>> q;
    q.push({sx, sy});
    dist[sx][sy] = 0;

    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();

        // 到达终点，返回最短距离
        if (x == ex && y == ey) return dist[x][y];

        // 遍历四个方向
        for (int i = 0; i < 4; i++) {
            int nx = x + dx[i], ny = y + dy[i];
            // 边界检查 + 可通行检查 + 未访问检查
            if (nx >= 0 && nx < n && ny >= 0 && ny < m
                && grid[nx][ny] != '#' && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
    return -1;  // 无法到达终点
}

1.4 层序遍历（按层处理）

有时我们需要区分每一层的节点（例如二叉树的层序遍历），可以用以下两种方式：

方法一：记录队列大小

// 按层 BFS：每次处理当前层的所有节点
int level = 0;
while (!q.empty()) {
    int sz = q.size();  // 当前层的节点数
    while (sz--) {
        auto cur = q.front(); q.pop();
        // 处理当前节点...
        // 将邻居加入队列
    }
    level++;  // 进入下一层
}

方法二：存储 (节点, 层数) 对

// 使用 pair 存储节点和层数
queue<pair<Node, int>> q;
q.push({start, 0});
while (!q.empty()) {
    auto [cur, level] = q.front(); q.pop();
    // level 即为当前层数
}

2. BFS 求最短路 — 无权图最短路径

2.1 为什么 BFS 能求最短路？

在无权图中（或所有边权相同的图），BFS 从起点开始逐层扩展。 第 k 层的所有节点恰好是距离起点为 k 的所有节点。由于 BFS 按层访问， 每个节点第一次被访问时，一定走的是最短路径。

关键证明

假设节点 v 第一次被访问时经过的路径长度为 d，如果存在一条更短路径长度 d' < d， 那么在 BFS 扩展到第 d' 层时就会访问到 v，与"第一次在第 d 层被访问"矛盾。因此第一次访问即为最短路。

BFS 逐层扩展示意图：

            [S]              第 0 层：起点 S
           / | \
         [A] [B] [C]         第 1 层：距离起点 1 的所有节点
         /|    |    \
       [D] [E] [F] [G]       第 2 层：距离起点 2 的所有节点
          \  |      |
          [H] [I]  [J]       第 3 层：距离起点 3 的所有节点

队列变化过程：
初始:  [S]
       [A, B, C]              <- 出 S，入 A B C
       [B, C, D, E]           <- 出 A，入 D E
       [C, D, E, F]           <- 出 B，入 F
       [D, E, F, G]           <- 出 C，入 G
       [E, F, G, H]           <- 出 D，入 H
       ...                    逐层扩展，直到目标节点出队

2.2 图的 BFS 最短路模板

// 邻接表存储图，求单源最短路（无权图）
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5;
vector<int> g[N];           // 邻接表
int dist[N];                 // dist[i] = 起点到 i 的最短距离

void bfs_shortest(int s, int n) {
    memset(dist, -1, sizeof(int) * (n + 1));
    queue<int> q;
    q.push(s);
    dist[s] = 0;

    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : g[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

int main() {
    int n, m;
    scanf("%d%d", &n, &m);
    while (m--) {
        int u, v;
        scanf("%d%d", &u, &v);
        g[u].push_back(v);
        g[v].push_back(u);  // 无向图
    }
    bfs_shortest(1, n);
    printf("到节点 %d 的最短距离: %d\n", n, dist[n]);
    return 0;
}

2.3 经典例题：迷宫最短路径

迷宫最短路径（典型网格 BFS）

题意：给定 n×m 的网格迷宫，S 为起点，E 为终点，# 为墙壁，. 为通道。 每次可以向上下左右移动一步，求从 S 到 E 的最短步数。

思路：直接对网格进行 BFS，每个格子为一个节点，相邻可通行格子之间有边。

复杂度：时间 O(n×m)，空间 O(n×m)

// 洛谷 P1162 / LeetCode 1091 - 像此类网格最短路径问题
#include <bits/stdc++.h>
using namespace std;

int n, m;
char a[1005][1005];
int d[1005][1005];
const int dx[] = {0,0,1,-1};
const int dy[] = {1,-1,0,0};

int main() {
    scanf("%d%d", &n, &m);
    int sx, sy, ex, ey;
    for (int i = 0; i < n; i++) {
        scanf("%s", a[i]);
        for (int j = 0; j < m; j++) {
            if (a[i][j] == 'S') sx = i, sy = j;
            if (a[i][j] == 'E') ex = i, ey = j;
        }
    }

    memset(d, -1, sizeof(d));
    queue<pair<int,int>> q;
    q.push({sx, sy});
    d[sx][sy] = 0;

    while (!q.empty()) {
        auto [x,y] = q.front(); q.pop();
        for (int k = 0; k < 4; k++) {
            int nx = x+dx[k], ny = y+dy[k];
            if (nx>=0 && nx<n && ny>=0 && ny<m
                && a[nx][ny]!='#' && d[nx][ny]==-1) {
                d[nx][ny] = d[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
    printf("%d\n", d[ex][ey]);
    return 0;
}

3. 多源 BFS — 多起点同时扩展

3.1 什么是多源 BFS？

多源 BFS 是指同时从多个起点开始进行 BFS。将所有起点同时加入队列， 它们在第一层一起扩展，第二层继续一起扩展...每个节点的距离是它到最近起点的最短距离。

适用场景
  • 求每个点到最近的某个特殊点的距离（如到最近的 1 的距离）
  • 多个火源同时蔓延，问人能否逃出（多源 BFS 的典型变形）
  • 多个人从不同位置出发，问最短的汇合时间

3.2 多源 BFS 流程图解

多源 BFS 示例：3 个源点同时扩展

初始状态 (第0层):              第1层扩展:                   第2层扩展:
  1 . . . .                      1 1 . . .                    1 1 2 . .
  . . . . .                      1 . . . .                    1 2 . . .
  . . 1 . .          -->         . 1 1 . .        -->         2 1 1 2 .
  . . . . .                      . 1 . 1 .                    . 1 2 . .
  . . . . 1                      . . . . 1                    . . . . 1

源点: (0,0), (2,2), (4,4)     每个源点各扩展一层          各区域"汇合"

队列初始化: [(0,0,0), (2,2,0), (4,4,0)]  -- 所有源点同时入队

3.3 多源 BFS 代码模板

// 多源 BFS：求每个格子到最近源点的距离
#include <bits/stdc++.h>
using namespace std;

int n, m;
int a[1005][1005];     // 原始地图，1 表示源点
int dist[1005][1005];   // 到最近源点的距离
const int dx[] = {0,0,1,-1};
const int dy[] = {1,-1,0,0};

void multi_source_bfs() {
    memset(dist, -1, sizeof(dist));
    queue<pair<int,int>> q;

    // 步骤1: 将所有源点同时加入队列
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (a[i][j] == 1) {
                q.push({i, j});
                dist[i][j] = 0;
            }

    // 步骤2: 正常 BFS，所有源点一起扩展
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int k = 0; k < 4; k++) {
            int nx = x + dx[k], ny = y + dy[k];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m
                && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
}

3.4 经典例题：LeetCode 542 - 01 矩阵

LeetCode 542: 01 矩阵

题意：给定一个由 0 和 1 组成的矩阵，求每个元素到最近的 0 的距离。

思路：将所有值为 0 的格子作为源点，进行多源 BFS，dist 数组即为答案。

// LeetCode 542 - 01 矩阵 (多源 BFS)
class Solution {
public:
    vector<vector<int>> updateMatrix(vector<vector<int>>& mat) {
        int n = mat.size(), m = mat[0].size();
        vector<vector<int>> dist(n, vector<int>(m, -1));
        queue<pair<int,int>> q;

        // 所有 0 入队作为源点
        for (int i = 0; i < n; i++)
            for (int j = 0; j < m; j++)
                if (mat[i][j] == 0) {
                    q.push({i, j});
                    dist[i][j] = 0;
                }

        int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};
        while (!q.empty()) {
            auto [x,y] = q.front(); q.pop();
            for (int k = 0; k < 4; k++) {
                int nx = x+dx[k], ny = y+dy[k];
                if (nx>=0 && nx<n && ny>=0 && ny<m
                    && dist[nx][ny]==-1) {
                    dist[nx][ny] = dist[x][y] + 1;
                    q.push({nx, ny});
                }
            }
        }
        return dist;
    }
};

3.5 复杂度分析

时间： O(n × m)
空间： O(n × m)

多源 BFS 与单源 BFS 的时间复杂度完全相同，因为每个节点仍然只入队一次。 多个源点只是在初始化时同时入队，并不增加总遍历次数。

4. 双端队列 BFS (0-1 BFS)

4.1 什么是 0-1 BFS？

当图的边权只有 0 和 1 两种时，可以使用双端队列（deque）来优化 BFS：

  • 如果经过权为 0 的边到达邻居节点，将该邻居插入队首（相当于"免费"扩展）
  • 如果经过权为 1 的边到达邻居节点，将该邻居插入队尾（正常扩展）

这样保证了队列中节点按距离单调不减排列，每次从队首取出的节点就是当前距离最小的。

与 Dijkstra 的关系

0-1 BFS 本质上是 Dijkstra 算法在边权只有 0/1 时的特化。Dijkstra 用优先队列维护最小距离， 而 0-1 BFS 用双端队列替代，因为只有 0/1 两种边权，所以可以用队首/队尾区分。

4.2 0-1 BFS 流程图解

示例图：边权为 0 或 1

    0       1
[S] ---> [A] ---> [T]
 |                   ^
 |   1       0       |
 +---> [B] ---> [C]--+
       (权1)   (权0)

执行过程：
步骤  双端队列 (左=队首)        出队节点   dist[] 更新
 0    [S]                       --         dist[S]=0
 1    [A, B]                    S          dist[A]=0(权0,队首), dist[B]=1(权1,队尾)
 2    [B, T]                    A          dist[T]=1(权1,队尾)
 3    [C, T]                    B          dist[C]=1+0=1(权0,队首)
 4    [T]                       C          (T 已在队列中, dist[T]=1 已是最优)
 5    []                        T          完成! dist[T]=1

关键: A 通过权 0 边到达，插队首; B 通过权 1 边到达，排队尾

4.3 0-1 BFS 代码模板

// 0-1 BFS 双端队列模板
#include <bits/stdc++.h>
using namespace std;

const int INF = 1e9;
const int N = 1e5 + 5;

struct Edge { int to, w; };
vector<Edge> g[N];
int dist[N];

void bfs_01(int s, int n) {
    fill(dist, dist + n + 1, INF);
    deque<int> dq;
    dq.push_front(s);
    dist[s] = 0;

    while (!dq.empty()) {
        int u = dq.front(); dq.pop_front();

        for (auto& [v, w] : g[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (w == 0)
                    dq.push_front(v);  // 权 0: 插队首
                else
                    dq.push_back(v);   // 权 1: 排队尾
            }
        }
    }
}

4.4 网格 0-1 BFS：带代价的移动

在网格问题中，有时候移动到某个格子有不同代价。例如：在 LeetCode 1368 中， 不同方向的移动代价不同（0 或 1），就可以用 0-1 BFS。

// 网格 0-1 BFS 模板
int bfs_grid_01(vector<vector<int>>& grid) {
    int n = grid.size(), m = grid[0].size();
    vector<vector<int>> dist(n, vector<int>(m, INF));
    // 四个方向: 右(1), 左(2), 下(3), 上(4)
    int dx[] = {0,0,1,-1};
    int dy[] = {1,-1,0,0};

    deque<pair<int,int>> dq;
    dq.push_front({0,0});
    dist[0][0] = 0;

    while (!dq.empty()) {
        auto [x,y] = dq.front(); dq.pop_front();
        for (int k = 0; k < 4; k++) {
            int nx = x+dx[k], ny = y+dy[k];
            if (nx<0||nx>=n||ny<0||ny>=m) continue;
            // 当前方向编号 k+1，与格子标记比较得出代价
            int cost = (grid[x][y] == k+1) ? 0 : 1;
            if (dist[x][y] + cost < dist[nx][ny]) {
                dist[nx][ny] = dist[x][y] + cost;
                if (cost == 0) dq.push_front({nx,ny});
                else dq.push_back({nx,ny});
            }
        }
    }
    return dist[n-1][m-1];
}

5. BFS 状态搜索 — 状态空间搜索

5.1 什么是状态搜索？

在很多问题中，BFS 搜索的对象不是网格或图上的节点，而是问题的状态。 每个状态可以看作图中的一个节点，从一个状态通过某种操作转移到另一个状态就是一条边。 用 BFS 搜索状态空间，可以找到最少操作次数使初始状态变为目标状态。

状态搜索的关键步骤
  1. 定义状态：用什么数据结构表示一个状态（整数编码、字符串、结构体等）
  2. 状态转移：从当前状态可以进行哪些操作，产生哪些新状态
  3. 判重：用 set、map 或标记数组记录已访问的状态，避免重复搜索
  4. 终止条件：找到目标状态时返回步数

5.2 经典例题：八数码问题

八数码 / 滑动拼图（LeetCode 773）

题意：3×3 的网格中有 8 个数字块和一个空格，每次可以将空格与相邻数字交换。 求从初始状态到达目标状态的最少步数。

思路：将棋盘状态编码为字符串，BFS 搜索所有可达状态。

八数码状态空间搜索示意：

初始状态:           操作: 交换空格与相邻数字        目标状态:
+---+---+---+                                   +---+---+---+
| 1 | 2 | 3 |      1 2 3      1 2 3             | 1 | 2 | 3 |
+---+---+---+      4   5  --> 4 5                +---+---+---+
| 4 |   | 5 |      6 7 8      6 7 8             | 4 | 5 |   |
+---+---+---+                                   +---+---+---+
| 6 | 7 | 8 |      空格下移    空格右移          | 6 | 7 | 8 |
+---+---+---+                                   +---+---+---+

状态编码: "123456780" (0 表示空格)
状态转移: 每次移动空格，产生最多 4 个新状态
判重: 使用 unordered_set 记录已访问状态
// 八数码 BFS 状态搜索
#include <bits/stdc++.h>
using namespace std;

int slidingPuzzle(vector<vector<int>>& board) {
    string target = "123450";  // 目标状态
    string start = "";
    for (auto& row : board)
        for (int x : row)
            start += to_string(x);

    if (start == target) return 0;

    // 预计算每个位置的邻居（2x3 网格）
    vector<vector<int>> neighbors = {
        {1,3}, {0,2,4}, {1,5},
        {0,4}, {1,3,5}, {2,4}
    };

    queue<pair<string,int>> q;  // (状态, 步数)
    unordered_set<string> vis;
    q.push({start, 0});
    vis.insert(start);

    while (!q.empty()) {
        auto [state, steps] = q.front(); q.pop();
        int pos = state.find('0');  // 找空格位置

        for (int next : neighbors[pos]) {
            swap(state[pos], state[next]);
            if (state == target) return steps + 1;
            if (!vis.count(state)) {
                vis.insert(state);
                q.push({state, steps + 1});
            }
            swap(state[pos], state[next]);  // 恢复状态
        }
    }
    return -1;
}

6. BFS 与 DFS 对比

6.1 核心区别

BFS (广度优先)                       DFS (深度优先)

     [S]                                  [S]
    / | \                                 /
  [A] [B] [C]                          [A]
  /|   |    \                           /
[D] [E] [F] [G]                      [D]
                                      |
                                    [E]  ... 回溯
                                       \
                                       [B]
                                        |
                                       [F]  ... 继续深入

队列: 先进先出 (FIFO)                栈/递归: 后进先出 (LIFO)
逐层扩展，像"水波扩散"                沿一条路走到底，再回溯

6.2 详细对比表

对比维度BFS（广度优先）DFS（深度优先）
数据结构队列 (queue)栈 (stack) / 递归
搜索顺序逐层扩展，先广后深沿路径深入，先深后广
最短路天然保证无权图最短路不能保证最短路
空间复杂度O(分支因子^深度)，较大O(深度)，较小
完备性有解一定找到（有限图）可能陷入无限深分支
最优性无权图中最优不保证最优
实现方式循环 + 队列递归 / 循环 + 栈
适用问题最短路、层序遍历、状态搜索连通性、枚举方案、回溯
内存消耗大（需存整层节点）小（只需存当前路径）
递归深度无限制受递归栈深度限制

6.3 什么时候用 BFS？什么时候用 DFS？

选择 BFS 的场景
  • 需要求最短路径/最少步数（无权图）
  • 需要层序处理（如二叉树层序遍历）
  • 图的深度很大而宽度有限（避免栈溢出）
  • 需要找到第一个满足条件的解（最优性搜索）
  • 多源同时扩展的问题
选择 DFS 的场景
  • 需要枚举所有方案（如全排列、子集）
  • 需要判断连通性（不需要最短路）
  • 搜索空间树的深度较小而分支很多
  • 需要回溯的场景（如 N 皇后、数独）
  • 拓扑排序、强连通分量等图算法的基础

6.4 BFS 与 DFS 的典型对比示例

问题：在一个无限大的二叉树中找目标节点

BFS: 逐层搜索              DFS: 沿左子树一路深入
层 0: [1]                  1 -> 2 -> 4 -> 8 -> 16 -> ...
层 1: [2, 3]               如果目标在右子树深处，需要
层 2: [4, 5, 6, 7]         很久才能回溯找到
层 3: [8,9,10,11,12,13,14,15]

如果目标在 15: BFS 用 O(2^d) 空间
              DFS 用 O(d) 空间，但可能先遍历完整棵左子树

结论:
- 目标离根近 -> BFS 更快
- 目标离根远 -> DFS 省空间，但可能慢
- 无限图中 -> BFS 保证找到最短路径
