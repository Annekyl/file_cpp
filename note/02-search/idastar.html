1. 迭代加深搜索 (IDS)

迭代加深搜索（Iterative Deepening Search，简称 IDS）是一种结合了 DFS 的空间效率和 BFS 的最优性的搜索策略。其核心思想是：不断增大搜索深度上限，从 1 开始逐步递增，每次在该深度限制下执行 DFS。

1.1 基本思想

想象你在迷宫中寻找出口。普通 DFS 会一头扎进某条死胡同到底再回来；BFS 则逐层搜索但需要大量内存。IDS 的做法是：先只搜 1 步能到达的所有位置，没找到就搜 2 步，再找不到搜 3 步......如此反复。

  深度限制 = 1          深度限制 = 2          深度限制 = 3
      A                     A                     A
      |                   / | \                 / | \
      B                  B  C  D              B  C  D
                                               /|\  |
                                              E F G H
  只搜索深度 1       搜索深度 1~2         搜索深度 1~3

1.2 算法框架

// 迭代加深搜索通用框架
bool found;
int maxDepth;

void dfs(int u, int depth) {
    if (found) return;           // 已找到答案，提前退出
    if (depth > maxDepth) return; // 超过当前深度限制
    if (isGoal(u)) {
        found = true;
        return;
    }
    for (auto& v : neighbors(u)) {
        dfs(v, depth + 1);
    }
}

void iterativeDeepening(int start) {
    for (maxDepth = 0; ; maxDepth++) {
        found = false;
        dfs(start, 0);
        if (found) break;  // 找到答案
    }
}

1.3 为什么对树搜索是最优的？

表面上看，IDS 反复搜索同一节点似乎很浪费。但关键观察是：在分支因子为 b 的树中，绝大多数节点集中在最底层。具体来说，最底层节点占总节点数的 (b-1)/b。

关键结论

对于分支因子为 b、目标在深度 d 的搜索树，IDS 扩展的节点总数约为 BFS 的 b/(b-1) 倍。当 b=10 时，IDS 仅比 BFS 多搜索约 11% 的节点，却节省了指数级的内存。

时间复杂度 O(b^d) 与 BFS 同阶
空间复杂度 O(d) 仅需存储当前路径

1.4 IDS 的优势总结

  • 空间效率：与 DFS 相同，仅 O(d) 的栈空间
  • 最优性：与 BFS 相同，保证找到最短路径
  • 完备性：在有限分支的树中一定能找到解
  • 无需预估状态数：不像 DFS 需要设定最大深度

2. IDA* 算法

IDA*（Iterative Deepening A*）是 IDS 与启发式函数的结合。它用一个估价函数 f(n) = g(n) + h(n) 来替代简单的深度限制，其中 g(n) 是从起点到当前状态的实际代价，h(n) 是从当前状态到目标的启发式估计。

2.1 核心思想

IDA* 将每一层的深度限制替换为 f 值的阈值。在每次迭代中，只探索 f(n) ≤ threshold 的节点。如果当前迭代没有找到解，就将阈值更新为下一次迭代中所有被剪枝节点的最小 f 值，然后重新搜索。

  IDA* 搜索过程：

  第 1 轮: threshold = h(start) = 4
  ┌──────────────────────────┐
  │  f(start)=4              │
  │  ├── f=5 (剪枝)          │
  │  └── f=6 (剪枝)          │
  │  未找到解 → 新阈值 = 5   │
  └──────────────────────────┘

  第 2 轮: threshold = 5
  ┌──────────────────────────┐
  │  f(start)=4              │
  │  ├── f=5 → 深入...       │
  │  │   └── f=6 (剪枝)     │
  │  └── f=6 (剪枝)          │
  │  未找到解 → 新阈值 = 6   │
  └──────────────────────────┘

  第 3 轮: threshold = 6
  ┌──────────────────────────┐
  │  f(start)=4              │
  │  ├── f=5 → f=6 → 找到!  │
  │  返回最优解              │
  └──────────────────────────┘

2.2 算法框架

// IDA* 通用框架
int threshold;      // 当前 f 值阈值
int minNextThreshold; // 下一轮的最小阈值
bool found;

// h(state): 启发式函数，估计从 state 到目标的最小代价
// 要求 h 是可采纳的（admissible），即不高估真实代价

void dfs_idastar(State state, int g) {
    if (found) return;
    int f = g + h(state);
    if (f > threshold) {
        minNextThreshold = min(minNextThreshold, f);
        return;
    }
    if (isGoal(state)) {
        found = true;
        return;
    }
    for (auto& next : neighbors(state)) {
        dfs_idastar(next, g + cost(state, next));
    }
}

void ida_star(State start) {
    threshold = h(start);
    while (!found) {
        minNextThreshold = INT_MAX;
        dfs_idastar(start, 0);
        if (found) break;
        if (minNextThreshold == INT_MAX) break; // 无解
        threshold = minNextThreshold;
    }
}

2.3 可采纳启发函数 (Admissible Heuristic)

IDA* 保证最优性的前提是启发函数 h(n) 是可采纳的，即对于任意节点 n，h(n) 不超过从 n 到目标的实际最短距离。

可采纳性的含义

如果 h(n) 总是低估或恰好等于真实代价，那么 IDA* 一定不会错过最优解。反之，如果 h(n) 有时高估，可能会剪掉包含最优解的分支。

与 A* 的区别

A* 使用优先队列维护所有待扩展节点，需要 O(b^d) 的空间。IDA* 通过迭代加深避免了这一开销，空间仅 O(d)，代价是可能重复扩展部分节点。当状态空间很大（如 15-puzzle）时，IDA* 往往更实用。

2.4 什么时候用 IDA*？

  • 状态空间太大，A* 的优先队列会爆内存
  • 存在好的可采纳启发函数
  • 搜索树的分支因子不太大
  • 需要保证找到最优解

4. 经典问题 — 15-Puzzle

15-Puzzle（数字华容道）是 IDA* 最经典的应用场景。在一个 4×4 的棋盘上有 15 个数字方块和 1 个空格，每次只能将与空格相邻的方块滑入空格，目标是将初始排列变为有序排列。

4.1 问题分析

15-puzzle 的状态空间约为 16! / 2 ≈ 10^13，BFS 完全不可行。但使用 IDA* + Manhattan 距离启发函数，可以在合理时间内求解。

4.2 Manhattan 距离启发函数

对于棋盘上每个数字方块，计算它当前位置到目标位置的 Manhattan 距离（横向距离 + 纵向距离），所有方块的距离之和就是一个可采纳的启发值。

  当前状态:            目标状态:
  ┌──┬──┬──┬──┐       ┌──┬──┬──┬──┐
  │ 1│ 2│ 3│ 4│       │ 1│ 2│ 3│ 4│
  ├──┼──┼──┼──┤       ├──┼──┼──┼──┤
  │ 5│ 6│  │ 8│       │ 5│ 6│ 7│ 8│
  ├──┼──┼──┼──┤       ├──┼──┼──┼──┤
  │ 9│10│ 7│11│       │ 9│10│11│12│
  ├──┼──┼──┼──┤       ├──┼──┼──┼──┤
  │13│14│15│12│       │13│14│15│  │
  └──┴──┴──┴──┘       └──┴──┴──┴──┘

  7: (2,1)→(1,2) 距离 = |2-1|+|1-2| = 2
  8: (1,3)→(1,3) 距离 = 0
  11: (2,3)→(2,2) 距离 = 1
  12: (3,3)→(2,3) 距离 = 2
  h = 2+0+1+2 = 5  (其余方块距离为 0)

4.3 IDA* 求解代码

#include <cstdio>
#include <cstdlib>
#include <algorithm>
using namespace std;

const int N = 4;
int board[N][N];
int threshold, minNext;
bool solved;

// 方向: 上右下左
const int dx[] = {-1, 0, 1, 0};
const int dy[] = {0, 1, 0, -1};
const char dir[] = {'U', 'R', 'D', 'L'};

// Manhattan 距离启发函数
int heuristic() {
    int h = 0;
    for (int i = 0; i < N; i++)
        for (int j = 0; j < N; j++)
            if (board[i][j] != 0) {
                int val = board[i][j] - 1;
                h += abs(i - val / N) + abs(j - val % N);
            }
    return h;
}

// IDA* DFS
void dfs(int x, int y, int g, int lastDir) {
    if (solved) return;
    int h = heuristic();
    int f = g + h;
    if (f > threshold) {
        minNext = min(minNext, f);
        return;
    }
    if (h == 0) { solved = true; return; }

    for (int d = 0; d < 4; d++) {
        // 避免立即反向移动
        if (d == (lastDir ^ 2)) continue;
        int nx = x + dx[d], ny = y + dy[d];
        if (nx < 0 || nx >= N || ny < 0 || ny >= N) continue;

        swap(board[x][y], board[nx][ny]);
        dfs(nx, ny, g + 1, d);
        swap(board[x][y], board[nx][ny]);
        if (solved) return;
    }
}

int solve(int sx, int sy) {
    threshold = heuristic();
    while (!solved) {
        minNext = INT_MAX;
        dfs(sx, sy, 0, -1);
        if (solved) return threshold;
        if (minNext == INT_MAX) return -1;
        threshold = minNext;
    }
    return threshold;
}
优化技巧

避免反向移动：记录上一步方向，跳过反向操作可减少约 25% 的节点扩展。 增量更新 h 值：每次移动只影响一个方块的 Manhattan 距离，不必重新计算整个棋盘。

5. 经典问题 — 八皇后 IDA* 求解

八皇后问题要求在 8×8 棋盘上放置 8 个皇后，使得任意两个皇后不能互相攻击（不在同一行、同一列、同一对角线）。虽然这个问题通常用回溯法解决，但 IDA* 也可以应用于此。

5.1 启发函数设计

对于部分放置的棋盘，启发函数可以估计"至少还需要多少行才能放完所有皇后"。最简单的可采纳启发是：剩余未放置的皇后数（即 n - 已放置行数）。

5.2 IDA* 实现

#include <cstdio>
#include <cstring>
#include <algorithm>
using namespace std;

const int N = 8;
int queens[N];       // queens[i] = j 表示第 i 行皇后放在第 j 列
bool col[N], diag1[2*N], diag2[2*N];
int threshold, minNext;
bool found;

// 启发函数：已放置的皇后之间冲突对数
int conflicts() {
    int cnt = 0;
    for (int i = 0; i < N; i++)
        for (int j = i + 1; j < N; j++)
            if (queens[i] >= 0 && queens[j] >= 0)
                if (abs(i-j) == abs(queens[i]-queens[j]))
                    cnt++;
    return cnt;
}

void dfs(int row, int placed) {
    if (found) return;
    if (row == N) { found = true; return; }

    // f = 已用行数 + 剩余行数(至少每行一个) + 当前冲突数
    int h = (N - row) + conflicts();
    if (placed + h > threshold) {
        minNext = min(minNext, placed + h);
        return;
    }

    for (int c = 0; c < N; c++) {
        if (col[c] || diag1[row - c + N] || diag2[row + c]) continue;
        queens[row] = c;
        col[c] = diag1[row - c + N] = diag2[row + c] = true;
        dfs(row + 1, placed + 1);
        col[c] = diag1[row - c + N] = diag2[row + c] = false;
        queens[row] = -1;
        if (found) return;
    }
}

void solve_queens() {
    memset(queens, -1, sizeof(queens));
    threshold = N;  // 初始阈值: 至少需要 N 行
    while (!found) {
        minNext = INT_MAX;
        dfs(0, 0);
        if (!found) threshold = minNext;
    }
}
说明

八皇后问题状态空间较小（仅 92 种解），用普通回溯法已经足够快。这里用 IDA* 演示主要是为了理解算法框架。在实际竞赛中，IDA* 更适合状态空间极大、启发函数有效的场景。

6. 完整 C++ 模板

6.1 IDS 模板

// 迭代加深搜索模板 — 适用于树上最短路、博弈搜索等
#include <bits/stdc++.h>
using namespace std;

const int MAXD = 30;  // 最大深度限制

bool found;
int maxDepth;
int path[MAXD];      // 记录搜索路径

// state: 当前状态, depth: 当前深度
void dfs_ids(State state, int depth) {
    if (found) return;
    if (depth > maxDepth) return;
    if (isGoal(state)) {
        found = true;
        // 输出/记录答案
        return;
    }
    for (auto& next : getNextStates(state)) {
        path[depth] = next.action;
        dfs_ids(next, depth + 1);
    }
}

int ids_solve(State start) {
    for (maxDepth = 0; maxDepth <= MAXD; maxDepth++) {
        found = false;
        dfs_ids(start, 0);
        if (found) return maxDepth;
    }
    return -1;  // 未找到
}

6.2 IDA* 模板

// IDA* 模板 — 适用于大规模状态空间最优解搜索
#include <bits/stdc++.h>
using namespace std;

int threshold, minNext;
bool found;

// 启发函数: 必须可采纳 (不高估真实代价)
int heuristic(const State& s);

// 从 state 到 next 的移动代价
int stepCost(const State& state, const State& next);

void dfs_idastar(State state, int g) {
    if (found) return;
    int f = g + heuristic(state);
    if (f > threshold) {
        minNext = min(minNext, f);
        return;
    }
    if (isGoal(state)) {
        found = true;
        return;
    }
    for (auto& next : getNextStates(state)) {
        dfs_idastar(next, g + stepCost(state, next));
        if (found) return;
    }
}

int ida_star_solve(State start) {
    threshold = heuristic(start);
    while (!found) {
        minNext = INT_MAX;
        dfs_idastar(start, 0);
        if (found) return threshold;
        if (minNext == INT_MAX) return -1; // 无解
        threshold = minNext;
    }
    return -1;
}

7. 复杂度分析

7.1 IDS 复杂度

设分支因子为 b，目标在深度 d。IDS 在第 i 轮搜索深度 ≤ i 的所有节点，节点数为 O(b^i)。

总扩展节点数 b^d + b^(d-1) + ... + b = O(b^d)
空间复杂度 O(d) 仅存储当前路径

最底层节点占总数的 (b-1)/b，所以重复搜索的代价很小。当 b=2 时，IDS 扩展的节点数恰好是 BFS 的 2 倍；当 b=10 时，仅多约 11%。

7.2 IDA* 复杂度

IDA* 的复杂度取决于启发函数的质量。设最优解代价为 C*，启发函数的平均误差为 ε。

最好情况 O(d) h(n) = h*(n) 完美启发
最坏情况 O(b^d) h(n) = 0 退化为 IDS
迭代轮数

IDA* 的迭代轮数等于 threshold 的不同取值个数，通常远小于解的深度 d。每一轮的节点扩展数随阈值增大而指数增长，最后一轮的扩展数占主导地位。

7.3 IDA* vs A* 空间对比

算法 空间复杂度 15-Puzzle 约需内存
A* O(b^d) 约数十 GB
IDA* O(d) 约几 KB
