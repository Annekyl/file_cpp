1. DP 基础概念

1.1 什么是动态规划

动态规划 (Dynamic Programming, DP) 是一种通过将复杂问题分解为更小的子问题来求解的算法思想。与分治不同，DP 的子问题之间存在重叠，因此需要将每个子问题的解保存下来，避免重复计算。

1.2 两个核心性质

最优子结构 (Optimal Substructure)

问题的最优解包含其子问题的最优解。即：可以通过子问题的最优解，组合得到原问题的最优解。这是 DP 能够工作的根本前提。

重叠子问题 (Overlapping Subproblems)

在递归求解过程中，同一个子问题会被多次计算。DP 通过记忆化（自顶向下）或填表（自底向上）来消除重复计算。

1.3 三个关键要素

要素说明示例 (LIS)
状态定义 用什么变量描述子问题？选择维度是 DP 设计中最关键的一步 dp[i]：以第 i 个元素结尾的最长递增子序列长度
转移方程 当前状态如何由已知状态推导而来？描述状态之间的依赖关系 dp[i] = max(dp[j] + 1)，其中 j < i 且 a[j] < a[i]
初始条件与边界 递推的起点是什么？边界情况如何处理？ dp[0] = 1，每个元素自身构成长度为 1 的子序列

1.4 DP 的两种实现方式

  自顶向下 (记忆化搜索)              自底向上 (递推填表)
  ┌───────────────────┐           ┌───────────────────┐
  │   solve(n)        │           │   dp[0] = ...     │
  │     ├─ solve(n-1) │           │   dp[1] = ...     │
  │     ├─ solve(n-2) │           │   dp[2] = ...     │
  │     └─ ...        │           │   ...             │
  │   用 memo[] 缓存   │           │   dp[n] = answer  │
  └───────────────────┘           └───────────────────┘
  优点: 只算需要的子问题               优点: 无递归开销, 常数更小
  缺点: 递归栈开销                    缺点: 可能计算不需要的状态

1.5 线性 DP 的特点

线性 DP 是最基础的一类 DP，其状态通常沿着一个线性维度（如数组下标）排列。转移时，当前状态只依赖于前面若干个已计算的状态。常见模型包括：LIS、LCS、编辑距离等。

2. 最长递增子序列 (LIS)

LeetCode 300. 最长递增子序列

给定一个整数数组 nums，找到其中最长严格递增子序列的长度。

示例：输入 [10, 9, 2, 5, 3, 7, 101, 18]，输出 4（子序列为 [2, 3, 7, 101]）

2.1 O(n^2) 动态规划

状态定义

dp[i] 表示以 a[i] 结尾的最长递增子序列的长度。必须以 a[i] 结尾是关键约束，它保证了我们可以从更短的子序列转移过来。

转移方程

// 遍历 i 之前的所有位置 j
for (int j = 0; j < i; j++) {
    if (a[j] < a[i])
        dp[i] = max(dp[i], dp[j] + 1);
}

含义：如果 a[j] < a[i]，说明可以把 a[i] 接在以 a[j] 结尾的递增子序列后面，长度加 1。在所有合法的 j 中取最大值。

时间复杂度 O(n^2)
空间复杂度 O(n) 
C++#include <bits/stdc++.h>
using namespace std;

int lis(vector<int>& a) {
    int n = a.size();
    vector<int> dp(n, 1);   // 每个元素自身构成长度为 1 的子序列
    int ans = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (a[j] < a[i]) {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }
        ans = max(ans, dp[i]);
    }
    return ans;
}

2.2 O(n log n) 贪心 + 二分优化

核心思想

维护一个辅助数组 tail[]，其中 tail[len] 表示长度为 len 的递增子序列的最小末尾元素。这个数组天然有序（递增），因此可以用二分查找来加速。

贪心策略

对于相同的子序列长度，末尾元素越小越好——因为更小的末尾更容易让后面的元素接上来，从而产生更长的子序列。

算法流程

初始化空数组 tail[]
遍历原数组每个元素 a[i]
若 a[i] 大于 tail 末尾，直接追加（子序列长度 +1）
否则，在 tail 中二分查找第一个 ≥ a[i] 的位置并替换（保持最优末尾）
tail 的长度即为 LIS 长度

填表过程演示

以 [10, 9, 2, 5, 3, 7, 101, 18] 为例：

注意：tail 数组 [2, 3, 7, 18] 并不是实际的 LIS，它只用于计算长度。实际 LIS 是 [2, 3, 7, 101]。

时间复杂度 O(n log n)
空间复杂度 O(n) 
C++#include <bits/stdc++.h>
using namespace std;

int lis_fast(vector<int>& a) {
    vector<int> tail;  // tail[i]: 长度为 i+1 的递增子序列的最小末尾

    for (int x : a) {
        // lower_bound 找到第一个 >= x 的位置
        auto it = lower_bound(tail.begin(), tail.end(), x);
        if (it == tail.end()) {
            tail.push_back(x);      // x 比所有末尾都大, 延长子序列
        } else {
            *it = x;                  // 替换, 保持最优末尾
        }
    }
    return tail.size();
}

2.3 LIS 方案还原

在 O(n^2) DP 的基础上，记录每个状态的前驱 pre[i]，最后从 dp 值最大的位置沿前驱回溯即可。

C++#include <bits/stdc++.h>
using namespace std;

vector<int> lis_reconstruct(vector<int>& a) {
    int n = a.size();
    vector<int> dp(n, 1), pre(n, -1);
    int best = 0;  // dp 值最大的位置

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (a[j] < a[i] && dp[j] + 1 > dp[i]) {
                dp[i] = dp[j] + 1;
                pre[i] = j;       // 记录前驱
            }
        }
        if (dp[i] > dp[best]) best = i;
    }

    // 回溯还原路径
    vector<int> seq;
    for (int i = best; i != -1; i = pre[i]) {
        seq.push_back(a[i]);
    }
    reverse(seq.begin(), seq.end());
    return seq;
}
注意

O(n log n) 方法的 tail 数组不直接对应实际 LIS 序列。如需在 O(n log n) 时间内还原方案，需要额外维护每个元素在 tail 中的位置索引，并在替换时记录信息，实现较为复杂。

3. 最长公共子序列 (LCS)

LeetCode 1143. 最长公共子序列

给定两个字符串 text1 和 text2，返回这两个字符串的最长公共子序列的长度。

示例：text1 = "abcde", text2 = "ace"，输出 3（LCS 为 "ace"）

3.1 O(nm) 二维 DP

状态定义

dp[i][j] 表示字符串 A 的前 i 个字符与字符串 B 的前 j 个字符的最长公共子序列长度。

转移方程

if (A[i-1] == B[j-1])
    dp[i][j] = dp[i-1][j-1] + 1;         // 字符匹配, LCS 长度 +1
else
    dp[i][j] = max(dp[i-1][j], dp[i][j-1]); // 不匹配, 取两种跳过方案的较大值

转移的直观理解：

  • A[i-1] == B[j-1]：当前字符可以匹配，LCS 长度在 dp[i-1][j-1] 基础上加 1
  • A[i-1] != B[j-1]：当前字符不匹配，要么跳过 A 的第 i 个字符（dp[i-1][j]），要么跳过 B 的第 j 个字符（dp[i][j-1]），取较大值

时间复杂度 O(nm)
空间复杂度 O(nm) 
C++#include <bits/stdc++.h>
using namespace std;

int lcs(string& A, string& B) {
    int n = A.size(), m = B.size();
    // dp[i][j]: A 前 i 个字符 与 B 前 j 个字符的 LCS 长度
    vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));

    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            if (A[i - 1] == B[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }
    return dp[n][m];
}

3.2 空间优化：滚动数组 O(min(n,m))

观察转移方程，dp[i][j] 只依赖于当前行和上一行，因此可以用两行交替（滚动数组）将空间降为 O(m)。进一步，我们可以只用一行，但需要从右向左填充以避免覆盖未使用的值。

C++int lcs_optimized(string& A, string& B) {
    // 确保 B 是较短的字符串, 减少空间
    if (A.size() < B.size()) swap(A, B);
    int n = A.size(), m = B.size();
    vector<int> dp(m + 1, 0);

    for (int i = 1; i <= n; i++) {
        int prev = 0;  // 保存 dp[i-1][j-1] 的值
        for (int j = 1; j <= m; j++) {
            int temp = dp[j];  // 暂存 dp[i-1][j] (下一轮变成 prev)
            if (A[i - 1] == B[j - 1]) {
                dp[j] = prev + 1;
            } else {
                dp[j] = max(dp[j], dp[j - 1]);
            }
            prev = temp;
        }
    }
    return dp[m];
}
时间复杂度 O(nm)
空间复杂度 O(min(n, m))

3.3 LCS 方案还原

从 dp[n][m] 开始回溯：

  • 若 A[i-1] == B[j-1]，该字符属于 LCS，加入结果，移动到 (i-1, j-1)
  • 否则，向 dp 值较大的方向移动（(i-1,j) 或 (i,j-1)）
C++string lcs_reconstruct(string& A, string& B) {
    int n = A.size(), m = B.size();
    vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));

    // 填表 (同标准 LCS)
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            dp[i][j] = (A[i-1] == B[j-1])
                ? dp[i-1][j-1] + 1
                : max(dp[i-1][j], dp[i][j-1]);

    // 回溯还原
    string res;
    int i = n, j = m;
    while (i > 0 && j > 0) {
        if (A[i-1] == B[j-1]) {
            res += A[i-1];  // 该字符属于 LCS
            i--; j--;
        } else if (dp[i-1][j] > dp[i][j-1]) {
            i--;   // 向上走
        } else {
            j--;   // 向左走
        }
    }
    reverse(res.begin(), res.end());
    return res;
}
经典变体

最长公共子串 (Longest Common Substring)：子串要求连续，转移方程变为 dp[i][j] = (A[i-1]==B[j-1]) ? dp[i-1][j-1]+1 : 0，答案为表中所有值的最大值。

LeetCode 583. 两个字符串的删除操作：答案为 n + m - 2 * LCS(A, B)。

4. 编辑距离 (Edit Distance)

LeetCode 72. 编辑距离

给定两个单词 word1 和 word2，计算将 word1 转换成 word2 所需的最少操作数。允许的操作：插入一个字符、删除一个字符、替换一个字符。

示例：word1 = "horse", word2 = "ros"，输出 3

4.1 状态定义与转移

状态定义

dp[i][j] 表示将 word1 的前 i 个字符转换成 word2 的前 j 个字符所需的最少操作数。

转移方程

if (word1[i-1] == word2[j-1])
    dp[i][j] = dp[i-1][j-1];               // 字符相同, 无需操作
else
    dp[i][j] = 1 + min({
        dp[i-1][j],                            // 删除 word1[i]: 从 dp[i-1][j] 转移
        dp[i][j-1],                            // 插入 word2[j]: 从 dp[i][j-1] 转移
        dp[i-1][j-1]                           // 替换 word1[i] 为 word2[j]
    });
  三种操作的转移方向:
  ┌────────────┐
  │ dp[i-1][j] │ ──删除──> dp[i][j]
  └────────────┘

  ┌────────────┐
  │ dp[i][j-1] │ ──插入──> dp[i][j]
  └────────────┘

  ┌──────────────┐
  │ dp[i-1][j-1] │ ──替换──> dp[i][j]
  └──────────────┘

初始条件

  • dp[i][0] = i：将 word1 的前 i 个字符全部删除，需要 i 次操作
  • dp[0][j] = j：在空串基础上插入 j 个字符，需要 j 次操作

DP 表填表示例

答案为 dp[5][3] = 3。一种最优方案：horse → rorse（替换 h→r）→ rose（删除 r）→ ros（删除 e）。

时间复杂度 O(nm)
空间复杂度 O(nm) 或 O(min(n,m)) 滚动数组优化 
C++#include <bits/stdc++.h>
using namespace std;

int minDistance(string& word1, string& word2) {
    int n = word1.size(), m = word2.size();
    vector<vector<int>> dp(n + 1, vector<int>(m + 1));

    // 初始条件: 空串到目标串的距离
    for (int i = 0; i <= n; i++) dp[i][0] = i;
    for (int j = 0; j <= m; j++) dp[0][j] = j;

    // 填表
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            if (word1[i - 1] == word2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1];    // 字符相同, 无操作
            } else {
                dp[i][j] = 1 + min({
                    dp[i - 1][j],                  // 删除
                    dp[i][j - 1],                  // 插入
                    dp[i - 1][j - 1]               // 替换
                });
            }
        }
    }
    return dp[n][m];
}

4.2 空间优化

与 LCS 类似，可以用滚动数组将空间优化到 O(min(n,m))。由于转移依赖 dp[i-1][j]、dp[i][j-1] 和 dp[i-1][j-1]，从右向左遍历即可只用一行数组。

C++int minDistance_optimized(string& word1, string& word2) {
    if (word1.size() < word2.size()) swap(word1, word2);
    int n = word1.size(), m = word2.size();
    vector<int> dp(m + 1);

    for (int j = 0; j <= m; j++) dp[j] = j;

    for (int i = 1; i <= n; i++) {
        int prev = dp[0];  // dp[i-1][0]
        dp[0] = i;
        for (int j = 1; j <= m; j++) {
            int temp = dp[j];
            if (word1[i - 1] == word2[j - 1]) {
                dp[j] = prev;
            } else {
                dp[j] = 1 + min({dp[j], dp[j - 1], prev});
            }
            prev = temp;
        }
    }
    return dp[m];
}
编辑距离的扩展

实际应用中，不同操作可以有不同的代价。例如 DNA 序列比对中，替换代价可能高于插入/删除。此时只需将转移中的 1 替换为对应代价即可。

相关问题：LeetCode 583（两个字符串的删除操作）、LeetCode 1143（LCS）本质上都是编辑距离的特例。

5. 数字三角形 (Number Triangle)

洛谷 P1216 [IOI1994] 数字三角形

给定一个数字三角形，从顶部出发，在每一节点可以选择向左下或右下走，一直走到底层。要求找出一条路径，使路径上的数字之和最大。

5.1 问题建模

         7
        / \
       3   8
      / \ / \
     8   1   0
    / \ / \ / \
   2   7   4   4
  / \ / \ / \ / \
 4   5   2   6   5

  最优路径: 7 -> 8 -> 7 -> 5 = 27

5.2 自顶向下 DP

状态定义

dp[i][j] 表示从三角形顶部走到第 i 行第 j 个位置时的最大路径和（下标从 0 开始）。

转移方程

// 第 i 行第 j 个数只能从上一行的 j-1 或 j 位置走来
dp[i][j] = a[i][j] + max(dp[i-1][j-1], dp[i-1][j])

// 注意边界: 每行第一个只能从上一行第一个来, 最后一个同理
dp[i][0] = a[i][0] + dp[i-1][0];
dp[i][i] = a[i][i] + dp[i-1][i-1];

填表过程

行原始数据dp 值

28, 1, 018, 16, 15

答案为最后一行的最大值 max(dp[n-1][0..n-1]) = 30。

时间复杂度 O(n^2)
空间复杂度 O(n^2) 或 O(n) 滚动数组 
C++#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<vector<int>> a(n, vector<int>(n, 0));
    for (int i = 0; i < n; i++)
        for (int j = 0; j <= i; j++)
            cin >> a[i][j];

    // dp[i][j]: 走到 (i,j) 的最大路径和
    vector<vector<int>> dp = a;  // 用 a 初始化 dp

    for (int i = 1; i < n; i++) {
        for (int j = 0; j <= i; j++) {
            int best = 0;
            if (j > 0) best = max(best, dp[i-1][j-1]);
            if (j < i) best = max(best, dp[i-1][j]);
            dp[i][j] += best;
        }
    }

    // 答案是最后一行的最大值
    int ans = *max_element(dp[n-1].begin(), dp[n-1].begin() + n);
    cout << ans << endl;
    return 0;
}

5.3 空间优化：从下往上

从底部向上递推可以进一步简化，且只需要一维数组。因为最终答案在 dp[0][0]，无需再取最大值。

C++int main() {
    int n;
    cin >> n;
    vector<int> dp(n);
    for (int i = 0; i < n; i++)
        cin >> dp[i];  // 先读入最后一行

    // 从倒数第二行开始向上
    for (int i = n - 2; i >= 0; i--) {
        int x;
        for (int j = 0; j <= i; j++) {
            cin >> x;
            dp[j] = x + max(dp[j], dp[j + 1]);
        }
    }
    // 注意: 上面的写法需要将输入存储起来或倒序读入
    // 更实用的写法是用二维数组读入后自底向上递推
    cout << dp[0] << endl;
}
技巧：自底向上 vs 自顶向下

数字三角形问题中，自底向上更加简洁：无需处理边界条件（每行首尾的特殊情况），且最终答案直接在 dp[0][0]。这是 DP 设计中的常见技巧——选择递推方向可以显著简化代码。

5.4 路径还原

C++void printPath(vector<vector<int>>& dp, vector<vector<int>>& a) {
    int n = dp.size();
    // 找到最后一行最大值的位置
    int col = 0;
    for (int j = 1; j < n; j++)
        if (dp[n-1][j] > dp[n-1][col])
            col = j;

    // 从底部回溯
    vector<int> path;
    for (int i = n - 1; i >= 0; i--) {
        path.push_back(a[i][col]);
        if (i > 0) {
            // 看是从左上方还是正上方来的
            if (col > 0 && dp[i-1][col-1] > dp[i-1][col])
                col--;
        }
    }
    reverse(path.begin(), path.end());
    for (int x : path) cout << x << " ";
}

总结

算法状态定义时间空间核心技巧
LIS (朴素) dp[i]: 以 i 结尾的 LIS 长度 O(n^2) O(n) 枚举前驱
LIS (优化) tail[]: 长度 k 的最小末尾 O(n log n) O(n) 贪心 + 二分
LCS dp[i][j]: A 前 i 与 B 前 j 的 LCS O(nm) O(nm) / O(min(n,m)) 二维递推 / 滚动数组
编辑距离 dp[i][j]: A 前 i 到 B 前 j 的最少操作数 O(nm) O(nm) / O(min(n,m)) 三种操作取 min
数字三角形 dp[i][j]: 走到 (i,j) 的最大路径和 O(n^2) O(n^2) / O(n) 自底向上简化
DP 设计通用步骤
  1. 定义状态：明确 dp 数组每个元素的含义（这一步最关键）
  2. 写转移方程：当前状态可以从哪些状态转移过来？取 max/min/求和？
  3. 确定初始条件和边界：递推的起点，数组越界的处理
  4. 确定遍历顺序：保证转移时所依赖的状态已经计算完毕
  5. 提取答案：答案在 dp 数组的哪个位置？是否需要遍历取最值？
