背包问题概述

背包问题是组合优化中的一类经典问题：给定一组物品，每件物品有重量和价值，在限定的总重量内，选择若干物品使得总价值最大。背包问题是动态规划的入门经典，也是理解状态转移方程的绝佳模型。

核心要素

  • 物品集合：n 件物品，第 i 件物品的重量为 w[i]，价值为 v[i]
  • 背包容量：背包能承受的最大总重量为 W
  • 决策：对每件物品选择"选"或"不选"（或选几个）
  • 目标：在不超过背包容量的前提下，使所选物品的总价值最大

分类总览

类型 每件物品可选次数 时间复杂度 核心特征
01 背包 最多选 1 次 O(nW) 每件物品只有选/不选两种状态
完全背包 无限次 O(nW) 每件物品可以重复选取
多重背包 最多选 s[i] 次 O(nW log s) / O(nW) 每件物品有数量限制
分组背包 每组最多选 1 件 O(nW) 物品被分为若干组，每组选一件
混合背包 以上混合 视组合而定 不同物品有不同的选取规则

01 背包 (0/1 Knapsack)

最基础的背包模型。有 n 件物品和容量为 W 的背包，第 i 件物品重量为 w[i]，价值为 v[i]。每件物品只能选一次，求背包能装的最大价值。

状态定义

状态定义

dp[i][j] 表示：从前 i 件物品中选取，背包容量为 j 时能获得的最大价值。

  • i：考虑的物品范围，0 到 n
  • j：当前背包剩余容量，0 到 W

状态转移方程

转移方程

对于第 i 件物品，有两种选择：

  • 不选第 i 件：dp[i][j] = dp[i-1][j]
  • 选第 i 件（前提：j >= w[i]）：dp[i][j] = dp[i-1][j - w[i]] + v[i]

取两者较大值：

dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])

转移方程详解

不选第 i 件物品：当前背包容量 j 不变，价值等于"从前 i-1 件物品中选、容量为 j"的最大价值，即 dp[i-1][j]。
选第 i 件物品：需要先预留 w[i] 的容量给第 i 件，剩余容量为 j - w[i]。在"从前 i-1 件物品中选、容量为 j-w[i]"的基础上，加上第 i 件的价值 v[i]，即 dp[i-1][j-w[i]] + v[i]。
取最优：两种决策取较大值，得到 dp[i][j] 的最终答案。

填表示例详解

dp[4][5] 的计算过程

考虑第 4 件物品（w=2, v=5），容量 j=5：

  • 不选物品 4：dp[3][5] = 7
  • 选物品 4（j=5 >= w[4]=2）：dp[3][5-2] + 5 = dp[3][3] + 5 = 5 + 5 = 10
  • dp[4][5] = max(7, 10) = 10

空间优化：一维数组

观察转移方程，dp[i][j] 只依赖于 dp[i-1][...]，即上一行的值。因此可以用一维数组滚动更新。

关键：逆序遍历容量

更新一维数组 dp[j] 时，必须 从 W 到 w[i] 逆序遍历。原因：正序遍历时，dp[j-w[i]] 可能已经在本轮被更新过（即已经考虑了第 i 件物品），导致同一件物品被重复选取——这就变成了完全背包。

逆序保证了 dp[j-w[i]] 仍是上一轮（i-1）的值。

    逆序遍历示意（i=4, w[4]=2, v[4]=5）:

    j:    5     4     3     2     1     0
          |     |     |     |     |     |
          v     v     v     v     v     v
    dp: [ ? ] [ ? ] [ ? ] [ ? ] [ 2 ] [ 0 ]

    从 j=5 到 j=2 逆序更新:
      dp[5] = max(dp[5], dp[5-2] + 5)  -- dp[3] 来自上一轮
      dp[4] = max(dp[4], dp[4-2] + 5)  -- dp[2] 来自上一轮
      dp[3] = max(dp[3], dp[3-2] + 5)  -- dp[1] 来自上一轮
      dp[2] = max(dp[2], dp[2-2] + 5)  -- dp[0] 来自上一轮

完整代码模板

// 01 背包 - 二维写法
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n + 1), v(n + 1);
    for (int i = 1; i <= n; i++)
        cin >> w[i] >> v[i];

    // dp[i][j]: 前i件物品, 容量j的最大价值
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
    for (int i = 1; i <= n; i++) {
        for (int j = 0; j <= W; j++) {
            dp[i][j] = dp[i - 1][j];  // 不选
            if (j >= w[i])
                dp[i][j] = max(dp[i][j], dp[i - 1][j - w[i]] + v[i]);  // 选
        }
    }
    cout << dp[n][W] << endl;
    return 0;
}

// 01 背包 - 一维空间优化（推荐写法）
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n + 1), v(n + 1);
    for (int i = 1; i <= n; i++)
        cin >> w[i] >> v[i];

    vector<int> dp(W + 1, 0);
    for (int i = 1; i <= n; i++) {
        for (int j = W; j >= w[i]; j--) {  // 逆序！
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    cout << dp[W] << endl;
    return 0;
}

完全背包 (Complete Knapsack)

与 01 背包的区别：每件物品可以选取无限次（只要背包容量允许）。

与 01 背包的区别

对比项01 背包完全背包
每件物品选取次数最多 1 次无限次
二维转移dp[i-1][j-w[i]]dp[i][j-w[i]]
一维遍历顺序逆序（W to w[i]）正序（w[i] to W）
原因保证每件物品只用一次允许重复使用同一物品

状态转移方程

完全背包转移方程

二维写法：

dp[i][j] = max(dp[i-1][j], dp[i][j - w[i]] + v[i])

注意第二个分支用的是 dp[i][j-w[i]]（同一行），而非 dp[i-1][j-w[i]]。这表示"在已经考虑过第 i 件物品的基础上，再选一次"。

为什么正序遍历

正序遍历的直觉理解

01 背包逆序是为了"只选一次"——保证更新 dp[j] 时用的是上一行的值。

完全背包正序是为了"可选多次"——更新 dp[j] 时，dp[j-w[i]] 可能已经在本轮被更新（即已经选过第 i 件物品），在此基础上再选一次，等价于选了多次。

    完全背包正序遍历示意（i=1, w[1]=2, v[1]=3）:

    j:    0     1     2     3     4     5
          |     |     |     |     |     |
          v     v     v     v     v     v
    dp: [ 0 ] [ 0 ] [ 0 ] [ 0 ] [ 0 ] [ 0 ]

    正序更新 j=2,3,4,5:
      dp[2] = max(dp[2], dp[0] + 3) = 3   // 选了1次物品1
      dp[3] = max(dp[3], dp[1] + 3) = 3   // dp[1]未更新=0, 选1次
      dp[4] = max(dp[4], dp[2] + 3) = 6   // dp[2]已更新=3, 选2次
      dp[5] = max(dp[5], dp[3] + 3) = 6   // dp[3]已更新=3, 选2次

    dp[4]=6 的含义: 选了2次物品1（重量2+2=4, 价值3+3=6）

完整代码模板

// 完全背包 - 一维空间优化
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n + 1), v(n + 1);
    for (int i = 1; i <= n; i++)
        cin >> w[i] >> v[i];

    vector<int> dp(W + 1, 0);
    for (int i = 1; i <= n; i++) {
        for (int j = w[i]; j <= W; j++) {  // 正序！
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    cout << dp[W] << endl;
    return 0;
}

多重背包 (Multiple Knapsack)

每件物品有数量限制：第 i 件物品最多选 s[i] 次。是 01 背包和完全背包的推广。

朴素做法

将第 i 件物品拆成 s[i] 个独立的 01 背包物品，然后用 01 背包求解。时间复杂度 O(W * sum(s[i]))，当 s[i] 较大时效率低下。

优化一：二进制拆分

二进制拆分思想

将数量 s 拆分为若干组，使得这些组能组合出 0 到 s 的任意整数。

拆分方式：1, 2, 4, 8, ..., 2^k, remainder，其中 remainder = s - (1 + 2 + 4 + ... + 2^k)。

例如 s=10 拆分为 1, 2, 4, 3（因为 1+2+4=7, remainder=10-7=3）。

拆分后每个组作为一个 01 背包物品，总物品数从 s 降到 log(s)。

    二进制拆分示意（s=10）:

    原始: 10个相同物品 ──拆分──> 4个不同物品（大小为1,2,4,3）

    组合验证:
      1 = 1
      2 = 2
      3 = 1+2 或 3
      4 = 4
      5 = 1+4
      6 = 2+4
      7 = 1+2+4
      8 = 1+2+4+1(remainder?)  ← 实际: 3+4+1?
      9 = ...                   ← 需要组合验证
      10 = 1+2+4+3

    每种数量都能由这些组的子集和表示

二进制拆分代码

// 多重背包 - 二进制拆分优化
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n), v(n), s(n);
    for (int i = 0; i < n; i++)
        cin >> w[i] >> v[i] >> s[i];

    // 二进制拆分，生成新的物品列表
    vector<int> nw, nv;
    for (int i = 0; i < n; i++) {
        int k = 1, rem = s[i];
        while (k <= rem) {
            nw.push_back(w[i] * k);
            nv.push_back(v[i] * k);
            rem -= k;
            k *= 2;
        }
        if (rem > 0) {
            nw.push_back(w[i] * rem);
            nv.push_back(v[i] * rem);
        }
    }

    // 01 背包求解
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < (int)nw.size(); i++) {
        for (int j = W; j >= nw[i]; j--) {
            dp[j] = max(dp[j], dp[j - nw[i]] + nv[i]);
        }
    }
    cout << dp[W] << endl;
    return 0;
}

优化二：单调队列优化

单调队列优化（进阶）

对于每件物品 i，将容量 j 按 j % w[i] 分组。同组内的转移可以用单调队列在 O(W) 时间内完成。

总时间复杂度优化为 O(nW)，与二进制拆分的 O(nW log s) 相比更优。

单调队列优化核心思路

令 r = j % w[i]，则 j = k * w[i] + r。转移方程可以改写为：

dp[j] = max{ dp[r + t*w[i]] + (k-t)*v[i] } for t in [max(0, k-s[i]), k]

这是一个滑动窗口最大值问题，可用单调队列在 O(1) 均摊时间内求解。

// 多重背包 - 单调队列优化
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n + 1), v(n + 1), s(n + 1);
    for (int i = 1; i <= n; i++)
        cin >> w[i] >> v[i] >> s[i];

    vector<int> dp(W + 1, 0), ndp(W + 1);
    for (int i = 1; i <= n; i++) {
        ndp = dp;
        for (int r = 0; r < w[i]; r++) {
            // 对余数 r 的分组使用单调队列
            deque<int> q;
            for (int k = 0; r + k * w[i] <= W; k++) {
                int j = r + k * w[i];
                // 维护窗口大小 s[i]
                while (!q.empty() && k - q.front() > s[i])
                    q.pop_front();
                // 维护单调递减
                int val = dp[j] - k * v[i];
                while (!q.empty() &&
                       dp[r + q.back() * w[i]] - q.back() * v[i] <= val)
                    q.pop_back();
                q.push_back(k);
                ndp[j] = dp[r + q.front() * w[i]] + (k - q.front()) * v[i];
            }
        }
        dp = ndp;
    }
    cout << dp[W] << endl;
    return 0;
}

分组背包 (Group Knapsack)

物品被分为若干组，每组内有若干物品。每组最多选择一件物品。目标是使总价值最大。

状态定义与转移

分组背包状态

dp[k][j]：考虑前 k 组物品，容量为 j 时的最大价值。

转移时，对第 k 组的每件物品 i，选择其中一件（或不选）：

dp[k][j] = max(dp[k-1][j], max over i in group k { dp[k-1][j-w[i]] + v[i] })

关键：遍历顺序

三重循环的顺序

分组背包的循环顺序为：先枚举组，再枚举容量，最后枚举组内物品。

即：for k (组) → for j (容量, 逆序) → for i (组内物品)

注意容量 j 必须在组内物品 i 之前枚举，否则同组物品可能被同时选取。

    分组背包循环结构:

    for (k = 1; k <= K; k++)           // 枚举每一组
      for (j = W; j >= 0; j--)         // 枚举容量（逆序）
        for (i in group[k])             // 枚举组内物品
          if (j >= w[i])
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);

    为什么容量在组内物品之前？
    ─ 如果先枚举物品再枚举容量，相当于组内每件物品独立做01背包，
      可能选到多件同组物品，违背"每组最多选一件"的约束。

完整代码模板

// 分组背包
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;

    // 读入分组，group[k] 存储第 k 组的物品索引
    int K;
    cin >> K;
    vector<int> w(n + 1), v(n + 1);
    vector<vector<int>> group(K + 1);
    for (int i = 1; i <= n; i++) {
        int g;
        cin >> w[i] >> v[i] >> g;
        group[g].push_back(i);
    }

    vector<int> dp(W + 1, 0);
    for (int k = 1; k <= K; k++) {
        for (int j = W; j >= 0; j--) {       // 容量逆序
            for (int idx : group[k]) {         // 组内物品
                if (j >= w[idx])
                    dp[j] = max(dp[j], dp[j - w[idx]] + v[idx]);
            }
        }
    }
    cout << dp[W] << endl;
    return 0;
}

复杂度分析

时间复杂度： O(K * W * max_group_size)
空间复杂度： O(W)

经典问题

洛谷 P1757 通天之分组背包

物品被分组，每组只能选一件，求最大价值。分组背包裸题。

背包问题变体

6.1 求方案数

问题变体：不求最大价值，而是求"恰好装满容量 W"的方案数（或使价值恰好为某值的方案数）。

方案数转移

将 max 操作替换为 + 操作：

dp[j] += dp[j - w[i]]

初始条件：dp[0] = 1（容量为 0 时有 1 种方案：什么都不选）

// 01 背包求方案数
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n + 1);
    for (int i = 1; i <= n; i++) cin >> w[i];

    int MOD = 1e9 + 7;
    vector<int> dp(W + 1, 0);
    dp[0] = 1;  // 容量为0有1种方案
    for (int i = 1; i <= n; i++) {
        for (int j = W; j >= w[i]; j--) {
            dp[j] = (dp[j] + dp[j - w[i]]) % MOD;
        }
    }
    cout << dp[W] << endl;
    return 0;
}
洛谷 P1164 小A点菜

6.2 求具体方案（方案回溯）

有时不仅需要最优值，还需要知道具体选了哪些物品。需要记录每一步的决策，然后从终点回溯。

方案回溯思路

使用二维 DP 表，从 dp[n][W] 出发：

  • 如果 dp[i][j] == dp[i-1][j]，说明第 i 件物品没有选
  • 如果 dp[i][j] == dp[i-1][j-w[i]] + v[i]，说明第 i 件物品被选中

逆序回溯 i 从 n 到 1，记录所有被选中的物品。

// 01 背包 - 求具体方案
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, W;
    cin >> n >> W;
    vector<int> w(n + 1), v(n + 1);
    for (int i = 1; i <= n; i++)
        cin >> w[i] >> v[i];

    // 二维 DP
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
    for (int i = 1; i <= n; i++) {
        for (int j = 0; j <= W; j++) {
            dp[i][j] = dp[i - 1][j];
            if (j >= w[i])
                dp[i][j] = max(dp[i][j], dp[i - 1][j - w[i]] + v[i]);
        }
    }

    // 回溯方案
    vector<int> chosen;
    int j = W;
    for (int i = n; i >= 1; i--) {
        if (j >= w[i] && dp[i][j] == dp[i - 1][j - w[i]] + v[i]) {
            chosen.push_back(i);  // 选了第 i 件
            j -= w[i];
        }
    }
    reverse(chosen.begin(), chosen.end());

    cout << dp[n][W] << endl;
    cout << "选择的物品: ";
    for (int idx : chosen) cout << idx << " ";
    cout << endl;
    return 0;
}
注意事项

方案回溯通常需要保留完整的二维 DP 表，空间复杂度为 O(nW)。如果需要在空间优化后回溯，需要额外记录每步决策。

6.3 混合背包

有些物品只能选一次（01 背包），有些能选无限次（完全背包），有些有数量限制（多重背包）。将三种背包混合在一起。

混合背包处理策略

对每件物品根据其类型分别处理：

  • 01 背包物品：容量逆序遍历
  • 完全背包物品：容量正序遍历
  • 多重背包物品：二进制拆分后按 01 背包处理

在同一个 DP 数组上，对不同类型物品使用不同的遍历方向即可。

// 混合背包
#include <bits/stdc++.h>
using namespace std;

struct Item {
    int w, v, s;  // 重量, 价值, 数量 (-1=无限)
};

int main() {
    int n, W;
    cin >> n >> W;

    vector<Item> items;
    for (int i = 0; i < n; i++) {
        int w, v, s;
        cin >> w >> v >> s;
        if (s == -1) {
            // 01 背包：拆成1个物品
            items.push_back({w, v, 1});
        } else if (s == 0) {
            // 完全背包
            items.push_back({w, v, 0});
        } else {
            // 多重背包：二进制拆分
            int k = 1, rem = s;
            while (k <= rem) {
                items.push_back({w * k, v * k, 1});
                rem -= k;
                k *= 2;
            }
            if (rem > 0)
                items.push_back({w * rem, v * rem, 1});
        }
    }

    vector<int> dp(W + 1, 0);
    for (auto& it : items) {
        if (it.s == 0) {
            // 完全背包：正序
            for (int j = it.w; j <= W; j++)
                dp[j] = max(dp[j], dp[j - it.w] + it.v);
        } else {
            // 01背包（含拆分后）：逆序
            for (int j = W; j >= it.w; j--)
                dp[j] = max(dp[j], dp[j - it.w] + it.v);
        }
    }
    cout << dp[W] << endl;
    return 0;
}

记忆口诀

"只有完全背包正序，其余都逆序"。正序意味着同一物品可被多次使用（完全背包特性），逆序保证每件物品只使用一次。

背包问题解题步骤

识别背包类型：判断每件物品的选取次数限制（01 / 完全 / 多重 / 分组）。
确定状态定义：dp[j] 表示容量为 j 时的最优值（一维），或 dp[i][j] 表示考虑前 i 件物品（二维）。
写出转移方程：根据选取次数确定用 max/加法，确定参考上一行还是当前行。
确定遍历顺序：01/多重/分组背包用逆序，完全背包用正序。
初始化与边界：一般初始化 dp 全为 0；若要求"恰好装满"，则 dp[0]=0，其余初始化为 -INF。
输出答案：dp[W] 即为最大价值。若需方案回溯，用二维 DP 并反向追踪。
