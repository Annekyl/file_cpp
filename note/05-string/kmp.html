1. KMP概述

1.1 什么是模式匹配

给定一个 文本串 S（长度为 n）和一个 模式串 P（长度为 m），在 S 中找到 P 出现的所有位置。这是字符串处理中最基本的问题之一。

1.2 暴力匹配的缺陷

朴素算法（暴力匹配）在每次失配时，将模式串右移一位重新从头比较，时间复杂度为 O(nm)。其核心问题在于：已经匹配过的信息被完全丢弃了。

文本串 S:  A B A B C A B A B C A B D
模式串 P:  A B A B C A B D
           ^^^^^^^^x  失配！
暴力做法:  回退到第二位重新开始
           A B A B C A B A B C A B D
             A B A B C A B D
KMP做法:   利用已匹配的"ABAB"信息，直接跳到有意义的位置继续
           A B A B C A B A B C A B D
                 A B A B C A B D

1.3 为什么是 O(n+m)

KMP 算法的核心思想是：当发生失配时，不回退文本串的指针，而是利用模式串自身的结构信息（next 数组），将模式串尽可能多地向右滑动，从而跳过不可能匹配成功的位置。

时间复杂度 O(n + m)
空间复杂度 O(m)
核心思想

KMP 的精髓在于：文本串的指针永远不回退。通过预处理模式串得到 next 数组，在匹配失败时，根据 next 数组决定模式串的滑动距离，从而实现线性时间匹配。

2. next数组

2.1 定义

next[i] 表示模式串 P 的子串 P[0..i] 中，最长的相等前后缀的长度。即：

  • 令 k = next[i]，则 P[0..k-1] = P[i-k+1..i]
  • 且 k 是满足条件的最大值
前后缀的含义

对于字符串 "ABAB"：

  • 前缀："A", "AB", "ABA"（不含自身的所有头部子串）
  • 后缀："B", "AB", "BAB"（不含自身的所有尾部子串）
  • 最长相等前后缀："AB"，长度为 2

2.2 构造过程详细推导

以模式串 P = "ABABABD" 为例，逐步构造 next 数组：

iP[i]子串 P[0..i]最长相等前后缀next[i]
0A"A"无（单字符无前后缀）0
1B"AB"前缀"A" != 后缀"B"0
2A"ABA""A" = "A"，长度11
3B"ABAB""AB" = "AB"，长度22
4A"ABABA""ABA" = "ABA"，长度33
5B"ABABAB""ABAB" = "ABAB"，长度44
6D"ABABABD"无相等前后缀0

2.3 next数组的构造算法

构造 next 数组本身也可以用类似 KMP 匹配的思路来实现，利用已有的 next 值来加速后续的计算：

初始化：next[0] = 0，令 j = 0（j 指向当前最长相等前后缀的下一个位置）
从 i = 1 开始遍历：若 P[i] == P[j]，则 next[i] = j + 1，j++
若 P[i] != P[j]：令 j = next[j-1]（回退到上一个匹配位置的 next 值），继续尝试匹配，直到 j == 0 或匹配成功
若 j 回退到 0 仍不匹配：next[i] = 0 
模式串 P:  A  B  A  B  A  B  D
下标:      0  1  2  3  4  5  6
next:      0  0  1  2  3  4  0

构造过程演示 (关键步骤):
i=2, P[2]='A', j=1 -> P[2]!=P[1], j=next[0]=0 -> P[2]==P[0], next[2]=1, j=1
i=3, P[3]='B', j=1 -> P[3]==P[1], next[3]=2, j=2
i=4, P[4]='A', j=2 -> P[4]==P[2], next[4]=3, j=3
i=5, P[5]='B', j=3 -> P[5]==P[3], next[5]=4, j=4
i=6, P[6]='D', j=4 -> P[6]!=P[4], j=next[3]=2 -> P[6]!=P[2], j=next[1]=0 -> P[6]!=P[0], next[6]=0

2.4 next数组构造代码

// 构造 next 数组（也称为 "前缀函数" 或 "failure function"）
// next[i] = P[0..i] 的最长相等前后缀长度
void buildNext(const string& P, vector<int>& nxt) {
    int m = P.size();
    nxt.resize(m);
    nxt[0] = 0;
    int j = 0;
    for (int i = 1; i < m; i++) {
        while (j > 0 && P[i] != P[j])
            j = nxt[j - 1];   // 回退
        if (P[i] == P[j])
            j++;
        nxt[i] = j;
    }
}

3. KMP匹配过程

3.1 匹配算法

有了 next 数组后，KMP 匹配的核心思想是：

  • 用指针 i 遍历文本串 S，用指针 j 遍历模式串 P
  • 当 S[i] == P[j] 时，两个指针同时前进
  • 当 S[i] != P[j] 时，i 不回退，将 j 跳转到 next[j-1]
  • 当 j == m 时，说明匹配成功，记录位置

3.2 逐步模拟

以 S = "ABABABCA"，P = "ABABC" 为例：

步骤ijS[i]P[j]操作next
100AA匹配，i++, j++next=[0,0,1,2,0]
211BB匹配，i++, j++
322AA匹配，i++, j++
433BB匹配，i++, j++
544AC失配！j=next[3]=2
642AA匹配，i++, j++
753BB匹配，i++, j++
864CC匹配，i++, j++
9-5--j==m，匹配成功！位置=i-m=2
 
第5步失配时的状态:
  S: A B A B A B C A
         i=4
  P:     A B A B C
             j=4 -> 失配
  j 回退到 next[3]=2:
  S: A B A B A B C A
         i=4
  P:       A B A B C
             j=2 -> 继续匹配

3.3 KMP匹配代码

// KMP 模式匹配，返回所有匹配位置（0-indexed）
vector<int> kmpSearch(const string& S, const string& P) {
    int n = S.size(), m = P.size();
    vector<int> nxt;
    buildNext(P, nxt);

    vector<int> pos;   // 存储所有匹配位置
    int j = 0;
    for (int i = 0; i < n; i++) {
        while (j > 0 && S[i] != P[j])
            j = nxt[j - 1];
        if (S[i] == P[j])
            j++;
        if (j == m) {
            pos.push_back(i - m + 1);
            j = nxt[j - 1];  // 继续寻找下一个匹配
        }
    }
    return pos;
}
常见陷阱

匹配成功后不要忘记将 j = nxt[j-1]，而不是将 j 置为 0。这是因为模式串的后缀可能同时也是下一次匹配的前缀。

4. next数组的应用

4.1 最小循环节（最小周期）

对于长度为 n 的字符串 S，若其存在循环节（即 S 可以由某个子串重复若干次得到），则 最小循环节长度 为：

最小循环节公式

len = n - next[n-1]

当且仅当 n % len == 0 时，S 是由长度为 len 的子串重复 n/len 次构成的。

例: S = "ABABAB"  (n=6)
next = [0, 0, 1, 2, 3, 4]

最小循环节长度 = 6 - next[5] = 6 - 4 = 2
6 % 2 == 0，所以 "ABABAB" = "AB" 重复 3 次

例: S = "ABCABC"  (n=6)
next = [0, 0, 0, 1, 2, 3]

最小循环节长度 = 6 - next[5] = 6 - 3 = 3
6 % 3 == 0，所以 "ABCABC" = "ABC" 重复 2 次

例: S = "ABCABD"  (n=6)
next = [0, 0, 0, 1, 2, 0]

最小循环节长度 = 6 - next[5] = 6 - 0 = 6
6 % 6 == 0，循环节就是整个串本身（无更小循环节）

4.2 回文串判断

利用 KMP 可以判断一个串是否为回文串。将原串 S 和它的反转 S' 拼接为 S + "#" + S'（# 为分隔符），然后计算新串的 next 数组。如果 next[last] == n，则 S 是回文串。

// 利用 KMP 判断回文串
bool isPalindrome(const string& s) {
    string rev = s;
    reverse(rev.begin(), rev.end());
    string t = s + "#" + rev;
    vector<int> nxt;
    buildNext(t, nxt);
    return nxt.back() == (int)s.size();
}

4.3 最短循环节（补充）

如果字符串不完全由循环节构成，但想知道"至少添加几个字符能让它成为循环串"，也可以用 next 数组解决：

  • 令 len = n - next[n-1]
  • 若 n % len != 0，则需要添加 len - n % len 个字符

4.4 统计前缀出现次数

利用 next 数组可以快速统计模式串的每个前缀在文本串中出现的次数，这在某些计数类问题中非常有用。

5. 完整C++模板

5.1 标准 KMP 模板

#include <bits/stdc++.h>
using namespace std;

struct KMP {
    string p;
    vector<int> nxt;

    KMP(const string& pattern) : p(pattern) {
        int m = p.size();
        nxt.resize(m);
        nxt[0] = 0;
        int j = 0;
        for (int i = 1; i < m; i++) {
            while (j > 0 && p[i] != p[j])
                j = nxt[j - 1];
            if (p[i] == p[j]) j++;
            nxt[i] = j;
        }
    }

    // 返回所有匹配位置
    vector<int> match(const string& s) {
        vector<int> res;
        int j = 0;
        for (int i = 0; i < (int)s.size(); i++) {
            while (j > 0 && s[i] != p[j])
                j = nxt[j - 1];
            if (s[i] == p[j]) j++;
            if (j == (int)p.size()) {
                res.push_back(i - j + 1);
                j = nxt[j - 1];
            }
        }
        return res;
    }

    // 最小循环节长度
    int minPeriod() {
        int n = p.size();
        int len = n - nxt[n - 1];
        return (n % len == 0) ? len : n;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string text, pattern;
    cin >> text >> pattern;

    KMP kmp(pattern);
    auto pos = kmp.match(text);

    for (int p : pos)
        cout << p + 1 << "\n";  // 1-indexed 输出
    for (int x : kmp.nxt)
        cout << x << " ";

    return 0;
}

5.2 扩展 KMP（Z 函数）

Z 函数（也称为 Z-algorithm）是另一个重要的字符串匹配工具。Z[i] 表示 S 与 S[i..n-1] 的最长公共前缀长度。

// Z 函数（扩展 KMP）
vector<int> zFunction(const string& s) {
    int n = s.size();
    vector<int> z(n, 0);
    z[0] = n;
    int l = 0, r = 0;
    for (int i = 1; i < n; i++) {
        if (i <= r)
            z[i] = min(r - i + 1, z[i - l]);
        while (i + z[i] < n && s[z[i]] == s[i + z[i]])
            z[i]++;
        if (i + z[i] - 1 > r)
            l = i, r = i + z[i] - 1;
    }
    return z;
}
