1. Trie概述

1.1 什么是Trie

Trie（发音同 "try"），又称前缀树或字典树，是一种树形数据结构，用于高效地存储和检索字符串集合中的键。它利用字符串的公共前缀来减少存储空间和查询时间。

1.2 结构特点

  • 根节点不包含字符（为空）
  • 每条边代表一个字符
  • 从根到某个节点的路径上所有字符拼接起来，形成一个字符串前缀
  • 通常用 flag 标记某个节点是否是一个完整字符串的结尾
插入 "abc", "abd", "acd", "b" 后的 Trie 结构:

            (root)
           /     \
          a       b*  <- "b" 在此结束
         / \
        c   c
       /     \
      b*      d*
      |       |
      d*     (end)

"abc" -> root -> a -> c -> b (flag=true)
"abd" -> root -> a -> c -> b -> d (flag=true)
"acd" -> root -> a -> c -> d (flag=true)
"b"   -> root -> b (flag=true)

* 表示该节点是某个字符串的结尾

1.3 复杂度分析

操作时间复杂度空间复杂度
插入O(L)O(L)
查找O(L)O(1)
前缀查询O(L)O(1)

其中 L 为字符串长度。与哈希表相比，Trie 的优势在于：前缀查询天然支持，且不会产生哈希冲突。

Trie vs 哈希表
  • Trie 支持按前缀查找所有匹配项，哈希表无法做到
  • Trie 的查找时间与字符串长度有关，哈希表与集合大小无关
  • Trie 通常比哈希表占用更多空间（每个节点需要子节点指针数组）
  • Trie 没有哈希冲突问题

2. 基本操作

2.1 插入（Insert）

将字符串插入 Trie。从根节点开始，逐个字符沿对应边前进；如果某条边不存在，则创建新节点。到达最后一个字符时，标记该节点为字符串结尾。

初始化：从根节点 p = root 开始
遍历字符串：对于每个字符 c，检查 p 是否有指向 c 的子节点
创建节点：如果子节点不存在，创建一个新节点
移动指针：p = p->child[c]
标记结尾：遍历结束后，p->flag = true

2.2 查找（Search）

查找字符串是否存在于 Trie 中。从根节点开始，逐个字符沿对应边前进。如果某条边不存在，返回 false。到达最后一个字符时，检查该节点是否被标记为字符串结尾。

2.3 前缀查询（StartsWith）

判断是否存在以给定前缀开头的字符串。与查找类似，但不需要检查结尾标记 -- 只要能沿着前缀走完所有字符，就说明存在。

2.4 数组实现（推荐竞赛使用）

竞赛中通常使用静态数组实现 Trie，避免动态内存分配的开销：

// 字符串 Trie - 数组实现（推荐竞赛使用）
struct Trie {
    int nxt[N][26];  // N = 最大节点数, 26 = 字符集大小
    bool flag[N];      // flag[i] = true 表示节点 i 是某个串的结尾
    int cnt;            // 节点计数

    void init() {
        cnt = 0;
        newNode();
    }

    int newNode() {
        cnt++;
        memset(nxt[cnt], 0, sizeof(nxt[cnt]));
        flag[cnt] = false;
        return cnt;
    }

    void insert(const string& s) {
        int p = 0;
        for (char c : s) {
            int d = c - 'a';
            if (!nxt[p][d])
                nxt[p][d] = newNode();
            p = nxt[p][d];
        }
        flag[p] = true;
    }

    bool search(const string& s) {
        int p = 0;
        for (char c : s) {
            int d = c - 'a';
            if (!nxt[p][d]) return false;
            p = nxt[p][d];
        }
        return flag[p];
    }

    bool startsWith(const string& prefix) {
        int p = 0;
        for (char c : prefix) {
            int d = c - 'a';
            if (!nxt[p][d]) return false;
            p = nxt[p][d];
        }
        return true;
    }
};
数组大小计算

最大节点数 N = 所有字符串的总长度之和 + 1（根节点）。例如，如果有 n 个字符串，每个长度最多 L，则 N = n * L + 1。

3. 01-Trie

3.1 什么是01-Trie

01-Trie 是 Trie 的一种特殊形式，用于处理整数的二进制表示。它将每个整数看作一个由 0 和 1 组成的字符串，从高位到低位依次插入 Trie 中。每个节点只有两个子节点：0 和 1。

插入 5 (101), 3 (011), 7 (111), 1 (001) 后的 01-Trie:
(假设用 3 位二进制表示)

              root
             /    \
            0      1
           / \    / \
          0   1  0   1
          |   |  |   |
          1   1  1   1
          |   |  |   |
         (1) (3)(5) (7)

路径 root->0->0->1 = 001 = 1
路径 root->0->1->1 = 011 = 3
路径 root->1->0->1 = 101 = 5
路径 root->1->1->1 = 111 = 7

3.2 01-Trie的用途

01-Trie 最核心的应用是处理异或（XOR）相关的问题：

  • 最大异或对：给定 n 个数，找到两个数使得它们的异或值最大
  • 最大异或子数组：找到一个子数组使得子数组的异或和最大
  • 与已知数的最大异或：在集合中找到与给定数异或值最大的数
  • 异或第 k 小：求所有数对异或值中的第 k 小值

3.3 01-Trie 的插入

将整数的二进制位从高位到低位（通常从第 30 位或第 60 位开始）依次插入：

// 01-Trie 插入（32位整数）
void insert(int x) {
    int p = 0;
    for (int i = 30; i >= 0; i--) {  // 从高位到低位
        int b = (x >> i) & 1;      // 取第 i 位
        if (!nxt[p][b])
            nxt[p][b] = newNode();
        p = nxt[p][b];
    }
}

3.4 01-Trie 的查询

查询与给定数异或值最大的数时，贪心地选择与当前位相反的分支（这样异或结果为 1）：

// 查询与 x 异或值最大的数
int query(int x) {
    int p = 0, res = 0;
    for (int i = 30; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (nxt[p][b ^ 1]) {       // 优先走相反的分支
            res |= (1 << i);        // 这一位异或结果为 1
            p = nxt[p][b ^ 1];
        } else {
            p = nxt[p][b];           // 只能走相同的分支
        }
    }
    return res;
}
贪心策略

异或的性质是：相同为 0，不同为 1。要让异或值最大，就应该让高位尽可能为 1。因此从高位开始，每次优先选择与当前位不同的分支，这就是 01-Trie 的贪心核心。

4. 最大异或对

4.1 问题描述

给定 n 个非负整数 a[0], a[1], ..., a[n-1]，找到两个下标 i, j (i != j) 使得 a[i] XOR a[j] 的值最大。

4.2 解题思路

建 Trie：将所有数依次插入 01-Trie
逐个查询：对于每个数 a[i]，在 Trie 中查找与它异或值最大的数
贪心匹配：从高位到低位，优先走相反方向的分支
取最大值：所有查询结果中的最大值即为答案

4.3 完整实现

#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;     // 最大元素个数
const int MAXB = 31;         // 二进制位数 (32位整数用31)
const int MAXNODES = MAXN * MAXB;

int nxt[MAXNODES][2];
int cnt;

void init() {
    cnt = 0;
    memset(nxt[0], 0, sizeof(nxt[0]));
}

int newNode() {
    cnt++;
    memset(nxt[cnt], 0, sizeof(nxt[cnt]));
    return cnt;
}

void insert(int x) {
    int p = 0;
    for (int i = MAXB - 1; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (!nxt[p][b])
            nxt[p][b] = newNode();
        p = nxt[p][b];
    }
}

int query(int x) {
    int p = 0, res = 0;
    for (int i = MAXB - 1; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (nxt[p][b ^ 1]) {
            res |= (1 << i);
            p = nxt[p][b ^ 1];
        } else {
            p = nxt[p][b];
        }
    }
    return res;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    init();
    vector<int> a(n);
    int ans = 0;

    for (int i = 0; i < n; i++) {
        cin >> a[i];
        if (i > 0)
            ans = max(ans, query(a[i]));
        insert(a[i]);
    }

    cout << ans << endl;
    return 0;
}

4.4 复杂度分析

时间复杂度 O(n * B)
空间复杂度 O(n * B)

其中 B 为二进制位数（通常为 31 或 63），n 为元素个数。

4.5 边插入边查询优化

实际上不需要先把所有数都插入 Trie 再查询。可以在插入第 i 个数之前，先查询它与前 i-1 个数的最大异或值，然后将第 i 个数插入 Trie。这样只需遍历一次数组。

关键观察

最大异或对问题中，如果 A XOR B 最大，那么 B XOR A 也最大。所以只需要考虑每个数与它之前插入的数的异或值，不需要考虑之后的数。这就是边插入边查询的正确性保证。

5.2 01-Trie模板

// 01-Trie 完整模板（支持插入、查询最大异或）
struct BitTrie {
    static const int MAXN = 100005;
    static const int MAXB = 31;    // 32位整数

    int nxt[MAXN * MAXB][2];
    int cnt;

    void init() {
        cnt = 0;
        memset(nxt[0], 0, sizeof(nxt[0]));
    }

    int newNode() {
        cnt++;
        memset(nxt[cnt], 0, sizeof(nxt[cnt]));
        return cnt;
    }

    void insert(int x) {
        int p = 0;
        for (int i = MAXB - 1; i >= 0; i--) {
            int b = (x >> i) & 1;
            if (!nxt[p][b]) nxt[p][b] = newNode();
            p = nxt[p][b];
        }
    }

    // 查询与 x 异或值最大的结果
    int queryMax(int x) {
        int p = 0, res = 0;
        for (int i = MAXB - 1; i >= 0; i--) {
            int b = (x >> i) & 1;
            if (nxt[p][b ^ 1]) {
                res |= (1 << i);
                p = nxt[p][b ^ 1];
            } else {
                p = nxt[p][b];
            }
        }
        return res;
    }

    // 查询与 x 异或值最小的结果
    int queryMin(int x) {
        int p = 0, res = 0;
        for (int i = MAXB - 1; i >= 0; i--) {
            int b = (x >> i) & 1;
            if (nxt[p][b]) {
                p = nxt[p][b];
            } else {
                res |= (1 << i);
                p = nxt[p][b ^ 1];
            }
        }
        return res;
    }
};
使用示例
BitTrie trie;
trie.init();

// 求最大异或对
int ans = 0;
for (int i = 0; i < n; i++) {
    if (i > 0)
        ans = max(ans, trie.queryMax(a[i]));
    trie.insert(a[i]);
}
cout << ans << endl;

5.3 可持久化01-Trie

在某些问题中，需要维护多个版本的 01-Trie（例如，查询某个区间内的最大异或值）。这时可以使用可持久化 01-Trie，利用版本之间的共享节点来节省空间。

// 可持久化 01-Trie（简化版框架）
struct PersistentBitTrie {
    static const int MAXN = 6000005;
    static const int MAXB = 31;

    int nxt[MAXN][2];
    int cnt;

    void init() { cnt = 0; }

    // 从旧版本 root 复制，插入 x 的第 d 位
    int insert(int prev, int x, int d = MAXB - 1) {
        int cur = ++cnt;
        if (d < 0) return cur;
        int b = (x >> d) & 1;
        nxt[cur][b ^ 1] = nxt[prev][b ^ 1];  // 复制另一分支
        nxt[cur][b] = insert(nxt[prev][b], x, d - 1);
        return cur;
    }

    // 在版本 root 中查询与 x 的最大异或
    int query(int root, int x) {
        int p = root, res = 0;
        for (int i = MAXB - 1; i >= 0; i--) {
            int b = (x >> i) & 1;
            if (nxt[p][b ^ 1]) {
                res |= (1 << i);
                p = nxt[p][b ^ 1];
            } else {
                p = nxt[p][b];
            }
        }
        return res;
    }
};
