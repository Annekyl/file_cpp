一、并查集概述

1.1 什么是并查集

并查集（Disjoint Set Union, DSU）是一种用于管理不相交集合（disjoint sets）的数据结构。它支持两种核心操作：

  • 合并（Union）：将两个集合合并为一个集合
  • 查询（Find）：查询某个元素属于哪个集合（通常用"代表元"表示）

并查集常用于解决连通性问题，例如判断两个节点是否在同一连通分量中、动态维护图的连通性等。

1.2 核心思想

并查集用树形结构表示每个集合：同一棵树上的所有节点属于同一集合，树根即为该集合的"代表元"。

  并查集的森林表示：

  集合 A:          集合 B:
      1                4
     / \              /|\
    2   3            5 6 7
    |
    8

  代表元分别为 1 和 4
  Find(8) = 1, Find(6) = 4
  Union(3, 6) 后合并为一棵树

1.3 两种优化策略

朴素实现的并查集在极端情况下会退化为链表，时间复杂度退化为 O(n)。通过以下两种优化，可以将均摊复杂度降至近似 O(1)：

优化策略核心思想效果
路径压缩（Path Compression）在 Find 时将路径上所有节点直接挂到根节点树变得极扁，近似 O(1)
按秩合并（Union by Rank/Size）合并时将较矮的树挂到较高的树下控制树的高度，避免退化
两者结合同时使用路径压缩和按秩合并均摊 O(alpha(n))，alpha 为反阿克曼函数，近似常数
单次操作均摊复杂度 O(alpha(n))
alpha(n) 在实际范围 ≤ 4

二、基本实现

2.1 数据结构

使用一个 parent[] 数组：parent[i] 表示节点 i 的父节点。若 parent[i] == i，则 i 为根节点（即代表元）。

可选地，维护 rank[]（秩）或 size[]（集合大小）用于按秩/大小合并。

2.2 路径压缩图解

  路径压缩前:              路径压缩后（Find(8)）:

      1                        1
     / \                     / | \
    2   3                   2  3  8
    |
    8

  Find(8) 的过程：
  1. 8 -> parent[8] = 2 -> parent[2] = 1 (根)
  2. 路径压缩：将 8 直接挂到 1 下
  结果：parent[8] = 1, parent[2] = 1

2.3 按秩合并图解

  合并前:                    合并后（Union(3, 6), rank 小的挂到大的下面）:

  集合 A (rank=2):           合并后 (rank=2):
      1                          1
     / \                        /|\
    2   3                      2 3 4
                                /|\
  集合 B (rank=1):             5 6 7
      4
     /|\
    5 6 7

  rank[1]=2 > rank[4]=1，所以 4 挂到 1 下

2.4 完整 C++ 模板：路径压缩 + 按秩合并

#include <iostream>
#include <vector>
using namespace std;

struct DSU {
    vector<int> parent, rank, sz;
    int components; // 连通分量个数

    DSU(int n) {
        parent.resize(n + 1);
        rank.resize(n + 1, 0);
        sz.resize(n + 1, 1);
        components = n;
        for (int i = 1; i <= n; i++)
            parent[i] = i;
    }

    // 查找代表元（带路径压缩）
    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);
        return parent[x];
    }

    // 合并两个集合（按秩合并）
    bool unite(int x, int y) {
        x = find(x), y = find(y);
        if (x == y) return false; // 已在同一集合

        // 按秩合并：rank 小的挂到 rank 大的下面
        if (rank[x] < rank[y]) swap(x, y);
        parent[y] = x;
        sz[x] += sz[y];
        if (rank[x] == rank[y]) rank[x]++;
        components--;
        return true;
    }

    // 判断两个元素是否在同一集合
    bool same(int x, int y) {
        return find(x) == find(y);
    }

    // 获取 x 所在集合的大小
    int getSize(int x) {
        return sz[find(x)];
    }
};

int main() {
    int n, m;
    cin >> n >> m;
    DSU dsu(n);

    while (m--) {
        int op, x, y;
        cin >> op >> x >> y;
        if (op == 1) {
            dsu.unite(x, y);
        } else {
            cout << (dsu.same(x, y) ? "Yes" : "No") << endl;
        }
    }
    return 0;
}
递归 vs 迭代路径压缩

在数据量极大（如 n ≥ 10^7）时，递归版路径压缩可能栈溢出。可以改用迭代写法：

// 迭代路径压缩（非递归）
int find(int x) {
    int root = x;
    while (parent[root] != root)
        root = parent[root];
    // 路径压缩
    while (x != root) {
        int nxt = parent[x];
        parent[x] = root;
        x = nxt;
    }
    return root;
}

2.5 只用路径压缩（简化版）

struct DSU {
    vector<int> p;
    DSU(int n) : p(n + 1) { iota(p.begin(), p.end(), 0); }
    int find(int x) { return p[x] == x ? x : p[x] = find(p[x]); }
    bool unite(int x, int y) {
        x = find(x), y = find(y);
        if (x != y) { p[y] = x; return true; }
        return false;
    }
    bool same(int x, int y) { return find(x) == find(y); }
};

三、带权并查集

3.1 概述

带权并查集在普通并查集的基础上，为每条边赋予一个"权值"，用于维护元素之间的相对关系。常见的关系类型包括：

  • 距离关系：weight[x] 表示 x 到其父节点的距离
  • 奇偶关系（种类并查集的基础）：用 0/1 表示同类/异类
  • 倍数关系：weight 表示 x 是父节点的多少倍

3.2 核心思路

维护一个 weight[] 数组，weight[x] 表示 x 到 parent[x] 的"距离"或"权值"。路径压缩时需要同步更新权值：

  Find 路径压缩时的权值更新：

  压缩前:  x --> p --> root
           w1    w2

  压缩后:  x ---------> root
           w1 + w2

  即：weight[x] += weight[parent[x]]
      然后 parent[x] = root

3.3 完整模板：带权并查集（距离型）

struct WeightedDSU {
    vector<int> parent, rank;
    vector<long long> weight; // weight[x] = x 到 parent[x] 的距离

    WeightedDSU(int n) {
        parent.resize(n + 1);
        rank.resize(n + 1, 0);
        weight.resize(n + 1, 0);
        for (int i = 1; i <= n; i++)
            parent[i] = i;
    }

    // 返回 x 的根节点，同时更新 weight[x] 为 x 到根的距离
    int find(int x) {
        if (parent[x] != x) {
            int root = find(parent[x]);
            weight[x] += weight[parent[x]]; // 累加路径上的权值
            parent[x] = root;
        }
        return parent[x];
    }

    // 合并：使得 weight[y] - weight[x] = w（即 y 在 x 右侧距离 w 处）
    bool unite(int x, int y, long long w) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return false;

        // 按秩合并
        if (rank[rx] < rank[ry]) {
            parent[rx] = ry;
            weight[rx] = weight[y] - weight[x] - w;
        } else {
            parent[ry] = rx;
            weight[ry] = weight[x] - weight[y] + w;
            if (rank[rx] == rank[ry]) rank[rx]++;
        }
        return true;
    }

    // 返回 x 到 y 的距离（需在同一集合）
    long long dist(int x, int y) {
        find(x); find(y);
        return weight[y] - weight[x];
    }
};

3.4 经典问题：食物链（POJ 1182）

POJ 1182 食物链

动物王国中有三类动物 A, B, C，A 吃 B，B 吃 C，C 吃 A。现有 N 个动物，用 1~N 编号。给出 K 句话，每句话为以下两种之一：

  • "1 X Y"：表示 X 和 Y 是同类
  • "2 X Y"：表示 X 吃 Y

求假话的总数。（详见第四节：种类并查集的解法）

四、种类并查集

4.1 概述

种类并查集（也叫"扩展域并查集"或"敌人的敌人是朋友"并查集）用于处理元素之间存在多种关系的问题。核心思想是将每个元素拆分成多个"域"，每个域代表一种关系。

4.2 两种实现方式

方法一：偏移量法（带权并查集）

用 weight[x] % 3 表示 x 与根节点的关系：0=同类，1=被根吃，2=吃根。

方法二：扩展域法（推荐，更直观）

将每个节点 x 拆分为多个域：x（自身）、x + n（吃 x 的）、x + 2n（被 x 吃的）。用并查集维护这些域之间的关系。

  扩展域法图解（食物链，N=3）：

  原始节点:    1    2    3
  自身域:      1    2    3    （同类关系）
  天敌域:      4    5    6    （吃该节点的）
  猎物域:      7    8    9    （被该节点吃的）

  "2 1 2" 表示 1 吃 2：
    合并(1, 8)    -- 1 和 "被2吃的" 同类
    合并(2, 4)    -- 2 和 "吃1的" 同类
    合并(7, 5)    -- "被1吃的" 和 "吃2的" 同类

4.3 完整模板：扩展域并查集

struct ExtendedDSU {
    vector<int> parent, rank;
    int n; // 原始节点数

    ExtendedDSU(int n) : n(n) {
        parent.resize(3 * n + 1);
        rank.resize(3 * n + 1, 0);
        for (int i = 1; i <= 3 * n; i++)
            parent[i] = i;
    }

    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);
        return parent[x];
    }

    void unite(int x, int y) {
        x = find(x), y = find(y);
        if (x == y) return;
        if (rank[x] < rank[y]) swap(x, y);
        parent[y] = x;
        if (rank[x] == rank[y]) rank[x]++;
    }

    bool same(int x, int y) { return find(x) == find(y); }

    // x 吃 y
    void eat(int x, int y) {
        unite(x, y + 2 * n);       // x 和 "被y吃的" 同类
        unite(x + n, y);             // "吃x的" 和 y 同类
        unite(x + 2 * n, y + n);   // "被x吃的" 和 "吃y的" 同类
    }

    // 判断 "x 吃 y" 是否与已有信息矛盾
    bool canEat(int x, int y) {
        return !same(x, y) && !same(x, y + n);
    }
};

4.4 带权并查集解法（食物链）

// 食物链 - 带权并查集解法
// weight[x] % 3 的含义：
//   0: x 与根同类
//   1: x 被根吃
//   2: x 吃根
#include <iostream>
#include <vector>
using namespace std;

vector<int> par, wt;

int find(int x) {
    if (par[x] != x) {
        int root = find(par[x]);
        wt[x] += wt[par[x]];
        par[x] = root;
    }
    return par[x];
}

int main() {
    int n, k, ans = 0;
    cin >> n >> k;
    par.resize(n + 1);
    wt.resize(n + 1, 0);
    for (int i = 1; i <= n; i++) par[i] = i;

    while (k--) {
        int d, x, y;
        cin >> d >> x >> y;
        if (x > n || y > n) { ans++; continue; }

        int rx = find(x), ry = find(y);
        if (d == 1) {
            // x 和 y 是同类
            if (rx == ry) {
                if ((wt[x] - wt[y]) % 3 != 0) ans++;
            } else {
                par[ry] = rx;
                wt[ry] = wt[x] - wt[y];
            }
        } else {
            // x 吃 y
            if (rx == ry) {
                if ((wt[x] - wt[y] - 1) % 3 != 0) ans++;
            } else {
                par[ry] = rx;
                wt[ry] = wt[x] - wt[y] - 1;
            }
        }
    }
    cout << ans << endl;
    return 0;
}
扩展域 vs 带权并查集

扩展域法：更直观，将每种关系拆成独立的域，代码容易理解。空间开销较大（3 倍）。

带权并查集：空间更省，实现更紧凑。但权值的计算（特别是合并时的公式推导）容易出错。

两者在时间复杂度上等价，选择自己更熟悉的写法即可。

五、回滚并查集 (可撤销)

5.1 概述

普通并查集的路径压缩是不可逆的：一旦压缩，原始的树结构信息就丢失了。在某些场景下（如线段树分治、时间倒流问题），我们需要能够撤销之前的合并操作。

5.2 实现思路

回滚并查集的关键限制：

  • 不能使用路径压缩（因为路径压缩不可逆）
  • 只使用按秩合并（合并操作可逆：只需记录被合并的节点和之前的秩）
  • 用一个栈记录每次合并操作的信息，撤销时从栈顶弹出并恢复
  回滚并查集的栈记录：

  操作          栈状态                说明
  ─────────────────────────────────────────────
  unite(1,2)   [(2, rank2)]          2 挂到 1 下，记录 2 的旧秩
  unite(3,4)   [(2,r2), (4,rank4)]   4 挂到 3 下
  undo()       [(2,r2)]              弹出栈顶，恢复 4 的父节点和秩
  undo()       []                    弹出栈顶，恢复 2 的父节点和秩

5.3 完整模板

struct RollbackDSU {
    vector<int> parent, rank, sz;
    vector<pair<int<int>> stk; // 记录被修改的节点及其旧值
    int comps;

    RollbackDSU(int n) {
        parent.resize(n + 1);
        rank.resize(n + 1, 0);
        sz.resize(n + 1, 1);
        comps = n;
        for (int i = 1; i <= n; i++) parent[i] = i;
    }

    // 不使用路径压缩的 find
    int find(int x) {
        while (parent[x] != x) x = parent[x];
        return x;
    }

    bool unite(int x, int y) {
        x = find(x), y = find(y);
        if (x == y) {
            stk.push_back({0, 0}); // 无实际合并，仍需压栈保持一致性
            return false;
        }
        if (rank[x] < rank[y]) swap(x, y);
        // 记录被修改节点 y 的旧值
        stk.push_back({y, parent[y]});
        parent[y] = x;
        sz[x] += sz[y];
        if (rank[x] == rank[y]) {
            stk.push_back({-x, rank[x]}); // 负号标记为秩的修改
            rank[x]++;
        }
        comps--;
        return true;
    }

    // 撤销最近一次合并
    void undo() {
        auto [node, val] = stk.back();
        stk.pop_back();
        if (node < 0) {
            rank[-node] = val; // 恢复秩
            undo(); // 继续弹出父节点的修改
        } else if (node != 0) {
            sz[parent[node]] -= sz[node];
            parent[node] = val;
            comps++;
        }
    }

    int snapshot() { return stk.size(); }
    void rollback(int snap) {
        while ((int)stk.size() > snap) undo();
    }
};
使用场景

回滚并查集常配合线段树分治（Segment Tree Divide and Conquer）使用，处理"在某段时间内存在某条边"的动态连通性问题。由于没有路径压缩，单次操作复杂度为 O(log n)。
