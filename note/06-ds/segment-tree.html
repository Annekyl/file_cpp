一、线段树概述

1.1 什么是线段树

线段树（Segment Tree）是一种二叉树结构，用于高效处理区间查询和区间修改问题。每个节点代表一个区间，存储该区间的某种聚合信息（如区间和、区间最大值等）。

  • 叶子节点：代表单个元素（区间长度为 1）
  • 内部节点：代表由其左右子节点合并而来的较大区间
  • 根节点：代表整个区间 [1, n]

1.2 何时使用线段树

需求推荐数据结构
静态区间查询（无修改）前缀和 O(1)
单点修改 + 前缀查询树状数组 O(log n)
单点修改 + 区间查询树状数组 或 线段树 O(log n)
区间修改 + 单点查询树状数组（差分）或 线段树 O(log n)
区间修改 + 区间查询线段树 O(log n)
需要维护复杂信息（如区间 gcd、区间众数）线段树
值域很大（如 10^9）但操作数少动态开点线段树

1.3 复杂度

建树 O(n)
单次操作 O(log n)
空间 O(4n)

1.4 存储方式

线段树通常使用数组模拟二叉树。对于区间 [1, n]，开 4 * n 的数组即可保证不越界：

  • 节点 u 的左子节点：u * 2
  • 节点 u 的右子节点：u * 2 + 1
  • 节点 u 的父节点：u / 2

二、建树

2.1 树结构图解

以数组 a = {1, 3, 5, 2, 4}（维护区间和）为例：

  线段树结构（区间和）：

                       [1, 5] = 15
                      /            \
              [1, 3] = 9          [4, 5] = 6
              /       \           /        \
         [1,2]=4    [3,3]=5   [4,4]=2    [5,5]=4
         /     \
      [1,1]=1 [2,2]=3

  数组存储（下标从 1 开始）：
  tree[1] = 15  (根节点, 区间 [1,5])
  tree[2] = 9   (左子, 区间 [1,3])
  tree[3] = 6   (右子, 区间 [4,5])
  tree[4] = 4   (区间 [1,2])
  tree[5] = 5   (区间 [3,3])
  tree[6] = 2   (区间 [4,4])
  tree[7] = 4   (区间 [5,5])
  tree[8] = 1   (区间 [1,1])
  tree[9] = 3   (区间 [2,2])

2.2 建树代码

#include <iostream>
#include <vector>
using namespace std;

const int MAXN = 100005;
long long tree[4 * MAXN];
long long a[MAXN];
int n;

// 以节点 u 为根，维护区间 [l, r]，构建线段树
void build(int u, int l, int r) {
    if (l == r) {
        tree[u] = a[l]; // 叶子节点
        return;
    }
    int mid = (l + r) / 2;
    build(2 * u, l, mid);          // 左子树
    build(2 * u + 1, mid + 1, r);  // 右子树
    tree[u] = tree[2 * u] + tree[2 * u + 1]; // 合并
}

2.3 建树过程图解

从根节点 [1, 5] 开始
mid = 3，左子区间 [1, 3]，右子区间 [4, 5]。递归构建子树。
递归到叶子节点
当 l == r 时，直接赋值 tree[u] = a[l]。这是递归的终止条件。
回溯合并
从叶子向上，tree[u] = tree[2u] + tree[2u+1]，即左子树和右子树的区间和相加。

三、单点修改 + 区间查询

3.1 单点修改

将 a[pos] 修改为 val。从根节点出发，沿路径更新所有包含 pos 的节点：

  将 a[3] 从 5 改为 7（路径上的节点全部更新）：

                 [1,5]: 15 -> 17
                /              \
        [1,3]: 9 -> 11       [4,5]: 6
        /       \             /      \
    [1,2]=4  [3,3]: 5->7  [4,4]=2  [5,5]=4
    /     \
  [1,1]=1 [2,2]=3

  更新路径：tree[5] -> tree[2] -> tree[1]
// 单点修改：将 a[pos] 改为 val
void update(int u, int l, int r, int pos, long long val) {
    if (l == r) {
        tree[u] = val;
        return;
    }
    int mid = (l + r) / 2;
    if (pos <= mid)
        update(2 * u, l, mid, pos, val);
    else
        update(2 * u + 1, mid + 1, r, pos, val);
    tree[u] = tree[2 * u] + tree[2 * u + 1];
}

3.2 区间查询

查询区间 [ql, qr] 的和。对于当前节点代表的区间 [l, r]，有三种情况：

完全包含：[ql, qr] 完全覆盖 [l, r]，直接返回 tree[u]
不相交：[ql, qr] 与 [l, r] 无交集，返回 0（不影响结果）
部分相交：递归查询左子树和右子树，返回两者的和 
  查询 [2, 4] 的和：

                 [1,5]
                /      \
        [1,3]           [4,5]
        /    \           /    \
    [1,2]  [3,3]     [4,4]  [5,5]
    /   \
  [1,1] [2,2]

  查询过程：
  [1,5] 部分相交 -> 递归左右
  [1,3] 部分相交 -> 递归左右
  [1,2] 部分相交 -> 递归左右
  [1,1] 不相交 (1 ∉ [2,4]) -> 返回 0
  [2,2] 完全包含 (2 ∈ [2,4]) -> 返回 3
  [3,3] 完全包含 (3 ∈ [2,4]) -> 返回 5
  [4,5] 部分相交 -> 递归左右
  [4,4] 完全包含 (4 ∈ [2,4]) -> 返回 2
  [5,5] 不相交 (5 ∉ [2,4]) -> 返回 0

  结果：0 + 3 + 5 + 2 + 0 = 10
// 区间查询：求 [ql, qr] 的和
long long query(int u, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[u]; // 完全包含
    if (qr < l || r < ql) return 0;          // 不相交
    int mid = (l + r) / 2;
    return query(2 * u, l, mid, ql, qr)
         + query(2 * u + 1, mid + 1, r, ql, qr);
}

3.3 完整模板：区间和（单点修改 + 区间查询）

#include <iostream>
#include <vector>
using namespace std;

const int MAXN = 100005;
long long tree[4 * MAXN], a[MAXN];

void build(int u, int l, int r) {
    if (l == r) { tree[u] = a[l]; return; }
    int mid = (l + r) / 2;
    build(2 * u, l, mid);
    build(2 * u + 1, mid + 1, r);
    tree[u] = tree[2 * u] + tree[2 * u + 1];
}

void update(int u, int l, int r, int pos, long long val) {
    if (l == r) { tree[u] = val; return; }
    int mid = (l + r) / 2;
    if (pos <= mid) update(2 * u, l, mid, pos, val);
    else update(2 * u + 1, mid + 1, r, pos, val);
    tree[u] = tree[2 * u] + tree[2 * u + 1];
}

long long query(int u, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[u];
    if (qr < l || r < ql) return 0;
    int mid = (l + r) / 2;
    return query(2 * u, l, mid, ql, qr)
         + query(2 * u + 1, mid + 1, r, ql, qr);
}

int main() {
    int n, m;
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> a[i];
    build(1, 1, n);

    while (m--) {
        int op, x, y;
        cin >> op >> x >> y;
        if (op == 1) {
            update(1, 1, n, x, y);
        } else {
            cout << query(1, 1, n, x, y) << endl;
        }
    }
    return 0;
}

四、区间修改 + 懒标记（Lazy Propagation）

4.1 问题引入

如果要将区间 [ql, qr] 的每个元素都加上 c，朴素做法需要修改 O(n) 个叶子节点。懒标记（Lazy Tag）的核心思想是：

懒标记的核心思想

当一个节点代表的区间完全被修改区间覆盖时，不立即向下传递修改，而是在该节点上打一个"懒标记"，记录"这个区间还需要被修改"。只有当后续操作需要访问子节点时，才将懒标记下传（pushdown）。

4.2 懒标记图解

  对 [1, 3] 每个元素 +2 的过程：

  步骤1: 从根 [1,5] 开始，[1,3] 部分覆盖，递归左右
  步骤2: 左子 [1,3] 完全包含在 [1,3] 中！
         直接更新 tree[2] += 2 * 3 = 6（区间和增加 2*区间长度）
         在节点 [1,3] 上打懒标记 lazy[2] = 2
         不再递归子树！

                  [1,5]: 15 -> 21
                 /              \
         [1,3]: 9 -> 15        [4,5]: 6
         lazy = 2              /      \
         /     \           [4,4]=2  [5,5]=4
     [1,2]=4  [3,3]=5
     (未更新)  (未更新)

  步骤3: 之后如果查询 [1,2]，需要先 pushdown [1,3]
         将 lazy[2] = 2 传给子节点：
         - [1,2]: tree += 2*2 = 4, lazy += 2
         - [3,3]: tree += 2*1 = 2, lazy += 2
         然后清除 lazy[2] = 0

4.3 Pushdown 操作

// 下传懒标记
void pushdown(int u, int l, int r) {
    if (lazy[u] != 0) {
        int mid = (l + r) / 2;
        // 更新左子树
        tree[2 * u] += lazy[u] * (mid - l + 1);
        lazy[2 * u] += lazy[u];
        // 更新右子树
        tree[2 * u + 1] += lazy[u] * (r - mid);
        lazy[2 * u + 1] += lazy[u];
        // 清除当前节点的懒标记
        lazy[u] = 0;
    }
}

4.4 完整模板：区间加 + 区间求和

#include <iostream>
#include <vector>
using namespace std;

const int MAXN = 100005;
long long tree[4 * MAXN], lazy[4 * MAXN], a[MAXN];
int n;

void pushup(int u) {
    tree[u] = tree[2 * u] + tree[2 * u + 1];
}

void pushdown(int u, int l, int r) {
    if (lazy[u]) {
        int mid = (l + r) / 2;
        tree[2 * u] += lazy[u] * (mid - l + 1);
        lazy[2 * u] += lazy[u];
        tree[2 * u + 1] += lazy[u] * (r - mid);
        lazy[2 * u + 1] += lazy[u];
        lazy[u] = 0;
    }
}

void build(int u, int l, int r) {
    if (l == r) { tree[u] = a[l]; return; }
    int mid = (l + r) / 2;
    build(2 * u, l, mid);
    build(2 * u + 1, mid + 1, r);
    pushup(u);
}

// 区间 [ql, qr] 每个元素 + val
void update(int u, int l, int r, int ql, int qr, long long val) {
    if (ql <= l && r <= qr) {
        tree[u] += val * (r - l + 1);
        lazy[u] += val;
        return;
    }
    pushdown(u, l, r);
    int mid = (l + r) / 2;
    if (ql <= mid) update(2 * u, l, mid, ql, qr, val);
    if (qr > mid)  update(2 * u + 1, mid + 1, r, ql, qr, val);
    pushup(u);
}

// 区间查询 [ql, qr] 的和
long long query(int u, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[u];
    pushdown(u, l, r);
    int mid = (l + r) / 2;
    long long res = 0;
    if (ql <= mid) res += query(2 * u, l, mid, ql, qr);
    if (qr > mid)  res += query(2 * u + 1, mid + 1, r, ql, qr);
    return res;
}

int main() {
    int m;
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> a[i];
    build(1, 1, n);

    while (m--) {
        int op, x, y;
        long long k;
        cin >> op;
        if (op == 1) {
            cin >> x >> y >> k;
            update(1, 1, n, x, y, k);
        } else {
            cin >> x >> y;
            cout << query(1, 1, n, x, y) << endl;
        }
    }
    return 0;
}

4.5 区间赋值（覆盖型懒标记）

除了区间加法，另一种常见操作是区间赋值（将区间所有元素设为同一个值）。懒标记的含义变为"这个区间被统一赋值为 lazy[u]"：

// 区间赋值的 pushdown（覆盖型）
void pushdown(int u, int l, int r) {
    if (lazy[u] != -1) { // -1 表示无标记
        int mid = (l + r) / 2;
        tree[2 * u] = lazy[u] * (mid - l + 1);
        lazy[2 * u] = lazy[u];
        tree[2 * u + 1] = lazy[u] * (r - mid);
        lazy[2 * u + 1] = lazy[u];
        lazy[u] = -1;
    }
}
多种懒标记共存

当一个线段树需要同时支持区间加和区间赋值时，需要维护两个懒标记（add 和 set），且规定赋值优先级高于加法。pushdown 时先处理 set 标记，再处理 add 标记。

五、动态开点线段树

5.1 概述

当值域非常大（如 [1, 10^9]）但实际操作数很少（如 n ≤ 10^5）时，预先开 4 * 10^9 的数组会爆内存。动态开点线段树只在需要时才创建节点，空间复杂度降为 O(n log n)。

5.2 核心思想

  • 使用指针或结构体数组 + 下标分配来表示节点
  • 初始时只有根节点（甚至根节点也延迟创建）
  • 每次 update 或 query 需要访问某个子节点时，如果该子节点不存在就新建
  动态开点过程（值域 [1, 8]）：

  初始：只有根节点
  update(3, +5): 沿路径 [1,8] -> [1,4] -> [1,4] -> [3,4] -> [3,3] 创建 5 个节点
  update(7, +3): 沿路径 [1,8] -> [5,8] -> [5,6] -> [7,8] -> [7,7] 复用根和 [5,8]

  最终树结构（只创建了用到的节点）：
              [1,8]
             /      \
        [1,4]        [5,8]
        /    \        /    \
    [1,2]  [3,4]  [5,6]  [7,8]
              |              |
           [3,3]          [7,7]

  共 9 个节点，而非 15 个

5.3 完整模板

struct Node {
    int left, right;   // 左右子节点的下标（0 表示不存在）
    long long val, lazy;
};

const int MAXNODES = 2000005; // n * log(值域) 足够
Node tree[MAXNODES];
int cnt = 1; // 节点分配计数器，1 为根节点

// 创建新节点，返回下标
int newNode() {
    return ++cnt;
}

// 区间 [ql, qr] 每个元素 + val
// u: 当前节点下标, [l, r]: 当前区间
void update(int& u, long long l, long long r,
            long long ql, long long qr, long long val) {
    if (!u) u = newNode(); // 按需创建节点
    if (ql <= l && r <= qr) {
        tree[u].val += val * (r - l + 1);
        tree[u].lazy += val;
        return;
    }
    long long mid = (l + r) / 2;
    // pushdown
    if (tree[u].lazy) {
        update(tree[u].left,  l, mid, l, mid, tree[u].lazy);
        update(tree[u].right, mid+1, r, mid+1, r, tree[u].lazy);
        tree[u].lazy = 0;
    }
    if (ql <= mid) update(tree[u].left,  l, mid, ql, qr, val);
    if (qr > mid)  update(tree[u].right, mid+1, r, ql, qr, val);
    tree[u].val = tree[tree[u].left].val + tree[tree[u].right].val;
}

// 区间查询 [ql, qr]
long long query(int u, long long l, long long r,
                long long ql, long long qr) {
    if (!u) return 0;
    if (ql <= l && r <= qr) return tree[u].val;
    long long mid = (l + r) / 2, res = 0;
    // pushdown
    if (tree[u].lazy) {
        update(tree[u].left,  l, mid, l, mid, tree[u].lazy);
        update(tree[u].right, mid+1, r, mid+1, r, tree[u].lazy);
        tree[u].lazy = 0;
    }
    if (ql <= mid) res += query(tree[u].left,  l, mid, ql, qr);
    if (qr > mid)  res += query(tree[u].right, mid+1, r, ql, qr);
    return res;
}
注意事项

动态开点线段树的 MAXNODES 通常开到 n * log(值域)，即 10^5 * 30 ≈ 3 * 10^6。如果同时有区间修改和区间查询，每个操作会创建 O(log n) 个新节点，总量要乘以操作数。

六、线段树合并（简介）

6.1 概述

线段树合并是将两棵结构相同的线段树合并为一棵的操作。合并时，如果两个节点都存在，递归合并其子节点；如果只有一个存在，直接使用该节点。

6.2 合并代码

// 合并以 u, v 为根的两棵线段树，结果存到 u
int merge(int u, int v, int l, int r) {
    if (!u || !v) return u + v; // 一个为空，返回另一个
    if (l == r) {
        tree[u].val += tree[v].val;
        return u;
    }
    int mid = (l + r) / 2;
    tree[u].left  = merge(tree[u].left,  tree[v].left,  l, mid);
    tree[u].right = merge(tree[u].right, tree[v].right, mid+1, r);
    tree[u].val = tree[tree[u].left].val + tree[tree[u].right].val;
    return u;
}

6.3 应用场景

  • 树上启发式合并（DSU on Tree）：将轻儿子的线段树合并到重儿子上
  • 可持久化线段树的构建
  • 合并权值线段树求第 k 小、区间排名等问题
合并复杂度 O(重叠节点数)
均摊复杂度 O(n log n) 总

七、树状数组 vs 线段树

树状数组（Fenwick Tree）和线段树都能处理区间查询问题，但在实现复杂度、空间和功能上有显著差异：

对比项树状数组 (Fenwick Tree)线段树 (Segment Tree)
代码量极短（10-20 行）较长（50-100 行）
空间O(n)O(4n)
单点修改 + 前缀查询O(log n)O(log n)
区间修改 + 单点查询O(log n)（差分技巧）O(log n)
区间修改 + 区间查询O(log n)（需双树状数组，较复杂）O(log n)（懒标记，直观）
维护复杂信息不支持（只能求和/XOR 等可逆运算）支持（max, min, gcd, 区间合并等）
区间最值不支持支持
二分查找（第 k 小）支持（树状数组上二分）支持（权值线段树）
常数因子极小较大（递归开销）
实现难度低中等
调试难度低中等（懒标记易出错）
选择建议
  • 只需要单点修改 + 前缀/区间查询：优先用树状数组，代码短、常数小
  • 需要区间修改 + 区间查询：用线段树（懒标记），或用双树状数组（代码稍复杂但常数小）
  • 需要维护区间最值、区间 gcd 等复杂信息：必须用线段树
  • 时间卡得很紧：优先考虑树状数组
  • 需要可持久化：线段树（可持久化线段树/主席树）

树状数组 vs 线段树代码对比（区间求和）

// ===== 树状数组（单点修改 + 区间查询）=====
int tree[MAXN], n;

void add(int i, int val) {
    for (; i <= n; i += i & -i)
        tree[i] += val;
}

int sum(int i) {
    int s = 0;
    for (; i > 0; i -= i & -i)
        s += tree[i];
    return s;
}

int rangeSum(int l, int r) { return sum(r) - sum(l - 1); }
