一、ST表概述

1.1 什么是ST表

ST表（Sparse Table，稀疏表）是一种用于高效处理静态区间最值查询（RMQ, Range Minimum/Maximum Query）的数据结构。它基于倍增思想，可以在 O(n log n) 时间内预处理，之后每次查询仅需 O(1) 时间。

1.2 适用场景

  • 数组不会被修改（静态数据）
  • 需要多次进行区间最值查询（最大值、最小值）
  • 也可以用于GCD等其他幂等运算

1.3 核心思想

ST表的核心是预处理所有以每个位置为起点、长度为 2^j 的区间的最值。查询时，将任意区间拆分成两个可能重叠的长度为 2^k 的区间，利用幂等性合并结果。

  ST表核心思想：

  预处理：f[i][j] = min(a[i], a[i+1], ..., a[i + 2^j - 1])
                  = 以 i 为起点，长度为 2^j 的区间的最小值

  查询 [l, r]：
  令 k = floor(log2(r - l + 1))
  则 [l, r] 可以被两个长度为 2^k 的区间覆盖：

  l                                    r
  |------------------------------------|
  |------------|                       |
  |  f[l][k]   |                       |
  |------------|                       |
              |------------|
              |  f[r-2^k+1][k] |
              |------------|

  两个区间有重叠，但对于 min/max 这不影响结果（幂等性）

1.4 复杂度对比

方法预处理单次查询总时间空间
朴素遍历O(1)O(n)O(nq)O(1)
ST表O(n log n)O(1)O(n log n + q)O(n log n)
线段树O(n)O(log n)O(n + q log n)O(n)
树状数组O(n)O(log n)O(n + q log n)O(n)
ST表 vs 线段树

当数据是静态的（不需要修改），且查询的是区间最值或区间GCD时，ST表优于线段树：查询 O(1) 比 O(log n) 快，且代码更简洁。但如果需要支持修改操作，就必须使用线段树或树状数组。

二、预处理

2.1 状态定义

定义 f[i][j] 表示从位置 i 开始，长度为 2^j 的区间的最小值：

f[i][j] = min(a[i], a[i+1], ..., a[i + 2j - 1])

其中 1 ≤ i ≤ n，0 ≤ j ≤ log2(n)。

2.2 递推公式

将长度为 2^j 的区间从中间分成两半，每半长度为 2^(j-1)：

f[i][j] = min(f[i][j-1], f[i + 2j-1][j-1]) 

  递推关系图解：

  f[i][j] 覆盖 [i, i + 2^j - 1]

  i                    i+2^(j-1)              i+2^j-1
  |--------------------|----------------------|
  |    f[i][j-1]       |    f[i+2^(j-1)][j-1] |
  |--------------------|----------------------|

  f[i][j] = min(f[i][j-1], f[i + 2^(j-1)][j-1])

  示例：f[1][2] = min(a[1],a[2],a[3],a[4])
       = min( min(a[1],a[2]), min(a[3],a[4]) )
       = min( f[1][1], f[3][1] )

2.3 初始化

边界条件：f[i][0] = a[i]，即长度为 1 的区间最小值就是元素本身。

2.4 预处理代码

const int MAXN = 100005;
const int LOG = 20;  // log2(MAXN) 向上取整

int f[MAXN][LOG];   // ST表
int a[MAXN];        // 原数组
int n;

void build() {
    // 边界：长度为 1 的区间
    for (int i = 1; i <= n; i++)
        f[i][0] = a[i];

    // 递推：按区间长度从小到大
    for (int j = 1; (1 << j) <= n; j++)
        for (int i = 1; i + (1 << j) - 1 <= n; i++)
            f[i][j] = min(f[i][j-1], f[i + (1 << (j-1))][j-1]);
}

2.5 预处理过程图解

  数组 a = [3, 1, 4, 1, 5, 9, 2, 6]，n=8

  j=0 (长度1):  f[i][0] = a[i]
  i:    1  2  3  4  5  6  7  8
  f[]:  3  1  4  1  5  9  2  6

  j=1 (长度2):  f[i][1] = min(f[i][0], f[i+1][0])
  i:    1  2  3  4  5  6  7
  f[]:  1  1  1  1  5  2  2

  j=2 (长度4):  f[i][2] = min(f[i][1], f[i+2][1])
  i:    1  2  3  4  5
  f[]:  1  1  1  1  2

  j=3 (长度8):  f[i][3] = min(f[i][2], f[i+4][2])
  i:    1
  f[]:  1
预处理时间 O(n log n)
空间复杂度 O(n log n)

三、O(1) 查询

3.1 查询原理

对于查询区间 [l, r] 的最小值，令 k = floor(log2(r - l + 1))，即区间长度对应的最大的 2 的幂次。

然后将 [l, r] 拆分成两个长度为 2^k 的区间：

  • 第一个区间：[l, l + 2^k - 1]，对应 f[l][k]
  • 第二个区间：[r - 2^k + 1, r]，对应 f[r - 2^k + 1][k]

query(l, r) = min(f[l][k], f[r - 2k + 1][k])

3.2 为什么两个区间够用

  查询 [l, r]，令 k = floor(log2(r - l + 1))

  情况1：区间长度恰好是 2^k
  l                           r
  |---------------------------|
  |         f[l][k]           |    (两个区间重合)
  |---------------------------|

  情况2：区间长度大于 2^k
  l                           r
  |---------------------------|
  |------------|              |
  |  f[l][k]   |              |
  |------------|              |
             |------------|
             | f[r-2^k+1][k] |
             |------------|

  两个区间有重叠，但完全覆盖了 [l, r]
  由于 min 是幂等运算，重叠不影响结果

3.3 log2 的预处理

为了 O(1) 计算 floor(log2(x))，可以预先打表：

int lg[MAXN];  // lg[i] = floor(log2(i))

// 预处理 log2
lg[1] = 0;
for (int i = 2; i <= n; i++)
    lg[i] = lg[i / 2] + 1;

3.4 查询代码

// 查询 [l, r] 的最小值，O(1)
int query(int l, int r) {
    int k = lg[r - l + 1];
    return min(f[l][k], f[r - (1 << k) + 1][k]);
}

3.5 查询过程图解

  示例：查询 [2, 7] 的最小值

  a = [3, 1, 4, 1, 5, 9, 2, 6]
       1  2  3  4  5  6  7  8

  区间长度 = 7 - 2 + 1 = 6
  k = floor(log2(6)) = 2  (2^2 = 4)

  两个覆盖区间：
  [2, 5] -> f[2][2] = min(1,4,1,5) = 1
  [4, 7] -> f[4][2] = min(1,5,9,2) = 1

  ans = min(f[2][2], f[4][2]) = min(1, 1) = 1

  验证：a[2..7] = [1,4,1,5,9,2]，最小值确实是 1
查询最大值

查询最大值只需将 min 换成 max，其余完全相同。可以同时维护 fmin[i][j] 和 fmax[i][j] 以支持两种查询。

四、局限性分析

4.1 只适用于幂等运算

ST表要求合并操作满足幂等性（idempotent），即 f(x, x) = x。这是因为查询时两个区间可能重叠，同一个元素可能被计算两次。

运算是否幂等是否适用ST表
min(a, b)min(x, x) = x适用
max(a, b)max(x, x) = x适用
gcd(a, b)gcd(x, x) = x适用
bitwise ORx | x = x适用
bitwise ANDx & x = x适用
a + bx + x = 2x ≠ x不适用
a * bx * x = x² ≠ x不适用

4.2 不支持修改操作

ST表是静态数据结构，预处理完成后不能高效地修改原数组。如果需要支持修改，应使用线段树或树状数组。

4.3 空间复杂度较高

ST表需要 O(n log n) 的空间。当 n = 10^6 时，大约需要 10^6 * 20 = 2 * 10^7 个整数，约 80MB，可能接近内存限制。

常见误区

不要用 ST表求区间和！区间和不是幂等运算，重叠部分会被重复计算。求区间和请使用前缀和（静态）或树状数组/线段树（动态）。

4.4 ST表的适用总结

数据静态（不需要修改），且查询的是区间最值或区间GCD -- 优先使用ST表
需要支持修改操作 -- 使用线段树或树状数组
需要区间求和 -- 使用前缀和（静态）或线段树/树状数组（动态）
空间紧张 -- 考虑线段树（O(n) 空间）或使用四叉树等变体

五、完整 C++ 模板

5.1 区间最小值 ST表

#include <iostream>
#include <algorithm>
using namespace std;

const int MAXN = 100005;
const int LOG = 20;

int f[MAXN][LOG];   // 最小值 ST表
int lg[MAXN];       // log2 预处理
int a[MAXN];
int n;

void build() {
    // 预处理 log2
    lg[1] = 0;
    for (int i = 2; i <= n; i++)
        lg[i] = lg[i / 2] + 1;

    // 初始化 f[i][0]
    for (int i = 1; i <= n; i++)
        f[i][0] = a[i];

    // 递推
    for (int j = 1; j <= lg[n]; j++)
        for (int i = 1; i + (1 << j) - 1 <= n; i++)
            f[i][j] = min(f[i][j-1], f[i + (1 << (j-1))][j-1]);
}

// O(1) 查询 [l, r] 的最小值
int query(int l, int r) {
    int k = lg[r - l + 1];
    return min(f[l][k], f[r - (1 << k) + 1][k]);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int q;
    cin >> n >> q;

    for (int i = 1; i <= n; i++)
        cin >> a[i];

    build();

    while (q--) {
        int l, r;
        cin >> l >> r;
        cout << query(l, r) << endl;
    }
    return 0;
}

5.2 同时支持区间最小值和最大值

int fmin[MAXN][LOG], fmax[MAXN][LOG];

void build() {
    for (int i = 1; i <= n; i++)
        fmin[i][0] = fmax[i][0] = a[i];

    for (int j = 1; (1 << j) <= n; j++) {
        for (int i = 1; i + (1 << j) - 1 <= n; i++) {
            fmin[i][j] = min(fmin[i][j-1], fmin[i + (1 << (j-1))][j-1]);
            fmax[i][j] = max(fmax[i][j-1], fmax[i + (1 << (j-1))][j-1]);
        }
    }
}

int query_min(int l, int r) {
    int k = lg[r - l + 1];
    return min(fmin[l][k], fmin[r - (1 << k) + 1][k]);
}

int query_max(int l, int r) {
    int k = lg[r - l + 1];
    return max(fmax[l][k], fmax[r - (1 << k) + 1][k]);
}

5.3 区间 GCD 查询

int fgcd[MAXN][LOG];

void build_gcd() {
    for (int i = 1; i <= n; i++)
        fgcd[i][0] = a[i];

    for (int j = 1; (1 << j) <= n; j++)
        for (int i = 1; i + (1 << j) - 1 <= n; i++)
            fgcd[i][j] = __gcd(fgcd[i][j-1], fgcd[i + (1 << (j-1))][j-1]);
}

int query_gcd(int l, int r) {
    int k = lg[r - l + 1];
    return __gcd(fgcd[l][k], fgcd[r - (1 << k) + 1][k]);
}
代码优化技巧

1. 使用 ios::sync_with_stdio(false); cin.tie(nullptr); 加速 IO
2. 预处理 lg[] 数组避免每次查询计算 log2
3. 循环条件写成 i + (1 << j) - 1 <= n 而非 i <= n - (1 << j) + 1，避免溢出
