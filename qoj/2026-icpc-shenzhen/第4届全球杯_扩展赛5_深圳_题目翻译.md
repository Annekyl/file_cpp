## 问题B. 全明星对决

**时间限制：** 2秒 | **内存限制：** 1024MB

本届大赛将以全明星对决作为特别展示赛：红队与蓝队之间的编程对决。与标准赛制不同，本次比赛允许灵活的队伍规模，但每位选手必须恰好被分配到某一支队伍中。

共有 $n$ 位全明星选手。每位选手 $i$ 在三个核心维度上被评估：算法洞察力、实现速度和调试韧性，这些属性被量化为正整数 $(x_i, y_i, z_i)$，构成三维能力向量。此外，每位选手 $i$ 被赋予一个正整数协同值 $w_i$，反映其放大队伍表现的能力。队伍的整体价值定义为其队员协同值的乘积。

**多样性规则：** 同一队伍中任意两位选手 $i$、$j$ 的欧氏距离必须 $\ge d$，即：

### 输出

输出 $n-1$ 行，第 $i$ 行输出红队恰好包含 $i$ 名选手时的价值总和模 $998244353$。

## 问题C. 差一件物品

**时间限制：** 1.5秒 | **内存限制：** 1024MB

**本题为交互题。** 每次输出后请刷新缓冲区。刷新方式：
- C/C++：`fflush(stdout)` 或 `cout.flush()`
- Java/Kotlin：`System.out.flush()`
- Python：`sys.stdout.flush()`

共有 $n$ 个人和 $m$ 件物品，每件物品必须分配给恰好一个人。

不同的人对物品的评价方式不同。对于任意物品集合，一个人可以评估其对它的满意度。你不知道这些评估是如何计算的，但可以通过询问查询其值。你最多可以询问 $n \times m$ 次。

形式上，对每个人 $i$，存在一个函数 $u_i(S)$，将任意物品子集 $S$ 映射到一个非负整数。这些函数满足：
- $u_i(\emptyset) = 0$
- 若 $S \subseteq T$，则 $u_i(S) \le u_i(T)$（即给某人更多物品不会让其满意度降低）

然而，人是敏感的：若人 $i$ 观察到另一人 $j$ 拥有严格更好的物品集合，他/她可能会抱怨。

幸运的是，人也是宽容的：若从 $j$ 的物品集合中移除一件物品就足以消除这种优势，则不会产生抱怨。

形式上，设 $s_i$ 为分配给人 $i$ 的物品集合。目标是将每件物品恰好分配给一个人，使得对任意一对人 $i$ 和 $j$：若 $u_i(s_j) > u_i(s_i)$，则存在 $x \in s_j$ 使得 $u_i(s_j \setminus \{x\}) \le u_i(s_i)$。

### 输入

每个测试文件只有一组测试数据。

第一行含两个整数 $n$ 和 $m$（$1 \le n \le 100$，$1 \le m \le 500$）。

### 交互协议

**询问格式：** 若要查询人 $i$ 对物品子集 $\{x_1, x_2, \ldots, x_k\}$ 的评价，输出：

```
? i k x1 x2 ... xk
```

其中 $1 \le i \le n$，$0 \le k \le m$，$x_1, \ldots, x_k$ 为两两不同的整数且 $1 \le x_t \le m$。

刷新输出后，程序应读取一个整数 $v$（$0 \le v \le 10^9$），即 $u_i(\{x_1, \ldots, x_k\})$。

至多可以询问 $n \times m$ 次。注意交互方是自适应的，即回答可能依赖于之前的询问，但始终与满足所有约束条件的某组有效函数 $u_i(\cdot)$ 一致。

**提交分配时，输出：**

```
! a1 a2 ... am
```

其中 $a_j$ 为物品 $j$ 分配的人的编号（$1 \le a_j \le n$）。分配物品不计入询问次数。刷新输出后程序应立即终止。

### 样例

| 标准输入 | 标准输出 |
|---------|---------|
| 2 3 | ? 1 1 1 |
| 10 | ? 1 2 2 3 |
| 20 | ? 2 1 1 |
| 10 | ? 2 2 2 3 |
| 5  | ! 2 1 1 |

## 问题D. 城市管理

**时间限制：** 3秒 | **内存限制：** 1024MB

你在管理一座城市。若干工厂将在你的城市建立，若干工人将来到城市寻找工作。你需要合理安排以最大化总利润。

每座工厂有产能 $x_i$，意味着在一轮生产中：
- 若有一名工人在其中工作，产生利润 $x_i$。
- 若没有工人，不产生利润。
- 每座工厂至多有一名工人。

在接下来的 $n$ 天中，每天发生以下两种事件之一：

- **W（工人）**：一名工人来找工作。此后可重新分配所有工人（包括正在工厂工作的工人）——可将每名工人分配到任意工厂或让其空闲。
- **F x（工厂）**：建立一座产能为 $x_i$ 的工厂。建立后，若有空闲工人，可选择将一名空闲工人分配到该新工厂，也可不分配。若没有空闲工人，该工厂只能空置。

每天事件执行后，所有工厂进行一轮生产并按上述规则产生利润。

求 $n$ 天内的最大总利润。

### 输入

多组测试数据。第一行含整数 $T$（$1 \le T \le 10^6$）。每组测试数据：

第一行含整数 $n$（$1 \le n \le 2 \times 10^5$），表示天数。

接下来 $n$ 行，第 $i$ 行首先是字符 $op_i$（$op_i \in \{W, F\}$），表示第 $i$ 天的事件类型。若 $op_i = F$，则后跟整数 $x_i$（$1 \le x_i \le 10^7$）。

保证所有测试数据的 $n$ 之和不超过 $10^6$。

### 输出

每组测试数据输出一行，表示最大总利润。

| F 100 | |
| W | |
| W | |

| W | |
| 9 | |
| W | |

| F 1 | |

### 注释

对第一个样例的解释：

## 问题E. 斗地主验牌

**时间限制：** 2秒 | **内存限制：** 1024MB

本题与扑克游戏"斗地主"有关。

### 游戏介绍

"斗地主"是一种三人轮流出牌的卡牌游戏。三位玩家分别为 A、B 和 C。每位玩家手持若干张牌，游戏持续到某位玩家出完手牌为止。

玩家 A 和 B 属于农民队。若 A 或 B 中任意一人出完手牌，农民获胜。玩家 C 单独构成地主队，若 C 出完手牌，地主获胜。

### 出牌规则

本题中，每张牌写有一个 $[1, n]$ 之间的整数，称为该牌的点数，点数从小到大排列：$1, 2, \ldots, n$。

本题仅允许以下两种出牌组合：
- **单张**：任意单张牌（仅当该玩家手中该点数恰好只有一张时可出）。**注意：手中是一对的牌不能拆开当单张出。**
- **对子**：两张相同点数的牌。

每轮中，一名玩家作为该轮的起手玩家先出牌，可出任意合法组合。然后从下一位玩家开始，依次轮流。每位玩家可以做以下两种选择之一：
- 出与本轮最后出牌相同类型且点数严格更高的组合。
- 选择不出（过）。

一名玩家出牌后，若其余两名玩家连续都选择不出，则本轮结束。此时，最后一次成功出牌的玩家成为下一轮的起手玩家。

### 题目

本题考虑以下情形：
- 地主 C 只剩一张牌，点数为 $p_c$。
- 两位农民 A 和 B 的手牌分别由长度为 $n$ 的字符串 $s_a$ 和 $s_b$ 描述，仅含字符 `0`、`1` 和 `2`。其中第 $i$ 个字符表示对应玩家持有点数为 $i$ 的牌的数量。

所有玩家的手牌为公开信息，即所有玩家都知道对方手中的牌。玩家 A 是第一轮的起手玩家，三位玩家按 $A \to B \to C \to A \to \cdots$ 的顺序轮流出牌。假设所有玩家均采用最优策略，判断农民队能否获胜。

### 输入

多组测试数据。第一行含整数 $T$（$1 \le T \le 10^5$）。每组测试数据：

第一行含整数 $n$（$1 \le n \le 2.5 \times 10^5$）。

第二行含长度为 $n$ 的字符串 $s_a$，仅含 `0`、`1`、`2`，表示农民 A 的手牌。

第三行含长度为 $n$ 的字符串 $s_b$，仅含 `0`、`1`、`2`，表示农民 B 的手牌。

第四行含整数 $p_c$（$1 \le p_c \le n$），表示地主 C 手中唯一一张牌的点数。

保证每位玩家至少有一张牌。保证所有测试数据的 $n$ 之和不超过 $10^6$。

### 输出

每组测试数据输出一行。若农民队能获胜，输出 `Yes`；否则输出 `No`。

### 样例

| 标准输入 | 标准输出 |
|---------|---------|
| 4 | Yes |
| 9 | No |
| 001110201 | Yes |
| 002110211 | No |
| 7 | |
| 9 | |
| 110200000 | |
| 222000000 | |
| 7 | |
| 9 | |
| 222000000 | |
| 110000000 | |
| 7 | |
| 9 | |
| 111000002 | |
| 111000210 | |
| 7 | |

### 注释

第一个样例的解释如下：

| 轮 | A手牌 | B手牌 | C | 出牌过程 |
|----|-------|-------|---|---------|
| 1 | 3,4,5,7,7,9 | 3,3,4,5,7,7,8,9 | 7 | A出(3)→B出(8)→C过→A出(9)→B过→C过 |
| 2 | 4,5,7,7 | 3,3,4,5,7,7,9 | 7 | A出(4)→B出(9)→C过→A过 |
| 3 | 5,7,7 | 3,3,4,5,7,7 | 7 | B出(3,3)→C过→A出(7,7)→B过→C过 |
| 4 | 5 | 4,5,7,7 | 7 | A出(5) |

## 问题F. 星座

**时间限制：** 1秒 | **内存限制：** 1024MB

"Astra"是一款美丽的桌游，玩家轮流标记星星来发现星座。

考虑 Alice 和 Bob 的双人博弈。游戏中有一个树形星座，可视为由 $n$ 个星星（编号 $1$ 到 $n$）和 $(n-1)$ 条边组成的连通图。初始时只有一个星星 $s$ 被标记。Alice 和 Bob 按以下规则轮流标记剩余星星：

- 每次必须标记至少 1 个、至多 $k$ 个星星。
- 每个星星只能被标记一次。
- 设本次标记的星星依次为 $a_1, a_2, \ldots, a_p$（按标记顺序）：
  - 标记 $a_1$ 时，它必须与某个已标记的星星相邻。
  - 对所有 $2 \le i \le p$，标记 $a_i$ 时，它必须与 $a_{i-1}$ 相邻。

两个星星 $u$ 和 $v$ 相邻当且仅当它们直接由一条边相连。

标记最后一个星星的玩家获得星座（胜利）。给定 Alice 先手且双方均采用最优策略，对每个 $1 \le s \le n$，判断谁会获胜。

### 输入

多组测试数据。第一行含整数 $T$（$1 \le T \le 100$）。每组测试数据：

第一行含两个整数 $n$ 和 $k$（$2 \le n \le 2 \times 10^3$，$1 \le k < n$），表示星星数量和每次最多标记的星星数。

接下来 $(n-1)$ 行，第 $i$ 行含两个整数 $u_i$ 和 $v_i$（$1 \le u_i, v_i \le n$），表示星星 $u_i$ 和 $v_i$ 之间有一条边。

保证所有测试数据的 $n$ 之和不超过 $2 \times 10^3$。

### 输出

每组测试数据输出一行，包含字符串 $d_1 d_2 \cdots d_n$（$d_i \in \{0, 1\}$），其中 $d_i$ 表示 $s=i$ 时谁会获胜。若 Alice 获胜则 $d_i = 1$；若 Bob 获胜则 $d_i = 0$。

### 样例

| 标准输入 | 标准输出 |
|---------|---------|
| 3 | 011000 |
| 6 2 | 000 |
| 1 2 | 101 |
| 3 2 | |
| 3 4 | |
| 5 3 | |
| 2 6 | |
| 3 1 | |
| 1 2 | |
| 2 3 | |
| 3 2 | |
| 1 2 | |
| 2 3 | |

### 注释

第一个样例中（树的结构：1-2，2-3，3-4，3-5，2-6）：

- **$s=1$**：Alice 的第一次可选择：标记 $\{2\}$、$\{2,3\}$ 或 $\{2,6\}$。若 Alice 选 $\{2,6\}$，Bob 可标记 $\{3\}$，剩余2个分离的星星。之后每人每次只能标记一个，Bob 会获胜。Alice 的其他选择可类似分析。
- **$s=2$**：Alice 可选择标记 $\{3\}$，剩余4个分离的星星，之后每人每次只能标记一个，Alice 会获胜。

第二个样例中，$k=1$，恰好有 $3-1=2$ 次出牌，Bob 始终获胜。

第三个样例中，若 $s=1$ 或 $s=3$，Alice 可直接标记剩余2个星星获胜。

## 问题G. 贪吃蛇

**时间限制：** 1秒 | **内存限制：** 1024MB

**本题为交互题。** 每次输出后请刷新缓冲区。刷新方式：
- C/C++：`fflush(stdout)` 或 `cout.flush()`
- Java/Kotlin：`System.out.flush()`
- Python：`sys.stdout.flush()`

你在一个 $n \times m$ 的网格上玩经典贪吃蛇游戏。行从上到下编号为 $1$ 到 $n$，列从左到右编号为 $1$ 到 $m$，位于第 $i$ 行第 $j$ 列的格子记为 $(i, j)$。

蛇可用一系列坐标对表示：$(x_1, y_1), (x_2, y_2), \ldots, (x_k, y_k)$，其中 $k$ 为蛇的长度。蛇头在 $(x_1, y_1)$，蛇尾在 $(x_k, y_k)$，相邻部分位于共享一条边的格子中。

初始时，蛇长度为 1，位于给定格子 $(r_s, c_s)$。

有4种移动命令：
- **U**：蛇向上移动一步，蛇头移动到 $(x_1 - 1, y_1)$。
- **D**：蛇向下移动一步，蛇头移动到 $(x_1 + 1, y_1)$。
- **L**：蛇向左移动一步，蛇头移动到 $(x_1, y_1 - 1)$。
- **R**：蛇向右移动一步，蛇头移动到 $(x_1, y_1 + 1)$。

蛇头移动时，身体每部分也相应移动：第 $i$（$2 \le i \le k$）部分移动到第 $(i-1)$ 部分之前所在的位置。同时，蛇尾之前占据的格子变为空格。

蛇不能移出网格，也不能与自身碰撞（任意命令后身体任意两部分不能在同一格子）。特别地：若蛇头正在移动到 $(x'_1, y'_1)$，允许 $(x'_1, y'_1) = (x_k, y_k)$（蛇头进入格子的同时蛇尾离开）。当 $k=2$ 时，允许用单个命令交换蛇头和蛇尾。

共有 $(nm-1)$ 个苹果将依次出现。每次，一个苹果出现在当前蛇未占据的某格 $(r, c)$。你需要输出一串命令，将蛇头从当前位置移动到 $(r, c)$。**蛇头必须在最后一条命令时到达 $(r, c)$，且在此之前不能进入 $(r, c)$。**

命令序列中，除最后一条外均为普通移动（蛇尾离开其格子）。最后一条为**进食移动**：蛇吃掉苹果，蛇尾不离开，蛇的长度加 1。

你的任务是成功吃掉所有 $(nm-1)$ 个苹果。命令序列长度至多为 $nm$。可以证明答案始终存在。

### 输入

多组测试数据。第一行含整数 $T$（$1 \le T \le 100$）。每组测试数据：

第一行含四个整数 $n, m, r_s, c_s$（$2 \le n, m \le 50$，$nm \le 100$，$1 \le r_s \le n$，$1 \le c_s \le m$）。
